<Info>
  **Voraussetzungen**:
  - Node.js Version 19 oder höher
  - Ein Dokumentations-Repository mit einer `docs.json`-Datei
</Info>

Befolgen Sie diese Schritte, um Mintlify auf Ihrem Betriebssystem zu installieren und auszuführen.

<Steps>
<Step title="Mintlify CLI installieren">

```bash
npm i -g mint
```
</Step>

<Step title="Lokale Vorschau">

Navigieren Sie zu Ihrem Dokumentationsverzeichnis, in dem sich die Datei `docs.json` befindet, und führen Sie den folgenden Befehl aus:

```bash
mint dev
```

Eine lokale Vorschau Ihrer Dokumentation ist unter `http://localhost:3000` verfügbar.

</Step>
</Steps>

## Benutzerdefinierte Ports

Standardmäßig verwendet Mintlify den Port 3000. Sie können den Port, auf dem Mintlify läuft, mit dem Flag `--port` anpassen. Um Mintlify beispielsweise auf Port 3333 auszuführen, verwenden Sie diesen Befehl:

```bash
mint dev --port 3333
```

Wenn Sie versuchen, Mintlify auf einem bereits belegten Port auszuführen, wird der nächste verfügbare Port verwendet:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify-Versionen

Bitte beachten Sie, dass jede CLI-Version mit einer bestimmten Version von Mintlify verknüpft ist. Wenn Ihre lokale Vorschau nicht mit der Produktionsversion übereinstimmt, aktualisieren Sie bitte die CLI:

```bash
npm mint update
```

## Links validieren

Die CLI kann Sie bei der Validierung von Links in Ihrer Dokumentation unterstützen. Um defekte Links zu identifizieren, verwenden Sie den folgenden Befehl:

```bash
mint broken-links
```

## Fehlerbehebung

<AccordionGroup>
  <Accordion title='Fehler: Das Modul "sharp" konnte nicht mit der darwin-arm64-Laufzeit geladen werden'>

    Dies kann an einer veralteten Node-Version liegen. Versuchen Sie Folgendes:
    1. Entfernen Sie die aktuell installierte Version der CLI: `npm remove -g mint`
    2. Aktualisieren Sie auf Node v19 oder höher.
    3. Installieren Sie die CLI erneut: `npm i -g mint`
  </Accordion>

  <Accordion title="Problem: Unbekannter Fehler tritt auf">
  
    Lösung: Gehen Sie zum Stammverzeichnis Ihres Geräts und löschen Sie den Ordner `~/.mintlify`. Führen Sie dann `mint dev` erneut aus.
  </Accordion>
</AccordionGroup>

Sind Sie neugierig, was sich in der neuesten CLI-Version geändert hat? Schauen Sie sich das [CLI-Änderungsprotokoll](https://www.npmjs.com/package/mintlify?activeTab=versions) an.
