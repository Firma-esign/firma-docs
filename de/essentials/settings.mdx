---
title: 'Globale Einstellungen'
description: 'Mintlify gibt Ihnen mit der Datei docs.json die vollständige Kontrolle über das Aussehen und Verhalten Ihrer Dokumentation'
icon: 'gear'
---

Jede Mintlify-Site benötigt eine `docs.json`-Datei mit den Kernkonfigurationseinstellungen. Erfahren Sie mehr über die [Eigenschaften](#properties) weiter unten.

## Eigenschaften

<ResponseField name="name" type="string" required>
Der Name Ihres Projekts. Wird für den globalen Titel verwendet.

Beispiel: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Ein Array von Gruppen mit allen Seiten innerhalb dieser Gruppe
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    Der Name der Gruppe.

    Beispiel: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    Die relativen Pfade zu den Markdown-Dateien, die als Seiten dienen.

    Beispiel: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Pfad zum Logo-Bild oder Objekt mit Pfad zu Logo-Bildern für „light"- und „dark"-Modus
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Pfad zum Logo im hellen Modus
    </ResponseField>
    <ResponseField name="dark" type="string">
      Pfad zum Logo im dunklen Modus
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Wohin der Klick auf das Logo führt
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Pfad zum Favicon-Bild
</ResponseField>

<ResponseField name="colors" type="Colors">
  Hex-Farbcodes für Ihr globales Theme
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      Die Primärfarbe. Wird am häufigsten für hervorgehobene Inhalte, Abschnittsüberschriften
      und Akzente im hellen Modus verwendet
    </ResponseField>
    <ResponseField name="light" type="string">
      Die Primärfarbe für den dunklen Modus. Wird am häufigsten für hervorgehobene
      Inhalte, Abschnittsüberschriften und Akzente im dunklen Modus verwendet
    </ResponseField>
    <ResponseField name="dark" type="string">
      Die Primärfarbe für wichtige Schaltflächen
    </ResponseField>
    <ResponseField name="background" type="object">
      Die Farbe des Hintergrunds im hellen und dunklen Modus
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          Der Hex-Farbcode des Hintergrunds im hellen Modus
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          Der Hex-Farbcode des Hintergrunds im dunklen Modus
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array von `name`s und `url`s der Links, die Sie in der Topbar einfügen möchten
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    Der Name der Schaltfläche.

    Beispiel: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    Die URL, zu der Sie beim Klick auf die Schaltfläche gelangen. Beispiel: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link zeigt eine Schaltfläche an. GitHub zeigt die Repo-Informationen unter der angegebenen URL, einschließlich der Anzahl der GitHub-Sterne.
    </ResponseField>
    <ResponseField name="url" type="string">
    Wenn `link`: Wohin die Schaltfläche verlinkt.
    
    Wenn `github`: Link zum Repository, aus dem GitHub-Informationen geladen werden sollen.
    </ResponseField>
    <ResponseField name="name" type="string">
    Text innerhalb der Schaltfläche. Nur erforderlich, wenn `type` ein `link` ist.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array von Versionsnamen. Verwenden Sie dies nur, wenn Sie verschiedene Versionen
  der Dokumentation mit einem Dropdown in der Navigationsleiste anzeigen möchten.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Ein Array der Anker, einschließlich `icon`, `color` und `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    Das [Font Awesome](https://fontawesome.com/search?q=heart) Symbol, das den Anker kennzeichnet.

    Beispiel: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    Der Name des Anker-Labels.

    Beispiel: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      Der Anfang der URL, der markiert, welche Seiten in den Anker gehören. In der Regel ist dies der Name des Ordners, in dem Sie Ihre Seiten abgelegt haben.
    </ResponseField>
    <ResponseField name="color" type="string">
      Die Hex-Farbe des Anker-Icon-Hintergrunds. Kann auch ein Verlauf sein, wenn Sie ein Objekt mit den Eigenschaften `from` und `to` übergeben, die jeweils eine Hex-Farbe sind.
    </ResponseField>
    <ResponseField name="version" type="string">
      Wird verwendet, wenn Sie einen Anker ausblenden möchten, bis die richtige Dokumentationsversion ausgewählt ist.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Übergeben Sie `true`, wenn Sie den Anker ausblenden möchten, bis Sie jemanden direkt auf Docs darin verlinken.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Eines von: "brands", "duotone", "light", "sharp-solid", "solid" oder "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Überschreiben Sie die Standardkonfigurationen für den obersten Anker.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      Der Name des obersten Ankers
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Font Awesome Icon.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Eines von: "brands", "duotone", "light", "sharp-solid", "solid" oder "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Ein Array von Navigationstabs.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      Der Name des Tab-Labels.
    </ResponseField>
    <ResponseField name="url" type="string">
      Der Anfang der URL, der markiert, welche Seiten in den Tab gehören. In der
      Regel ist dies der Name des Ordners, in dem Sie Ihre Seiten abgelegt haben.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Konfiguration für API-Einstellungen. Erfahren Sie mehr über API-Seiten unter [API-Komponenten](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      Die Basis-URL für alle API-Endpunkte. Wenn `baseUrl` ein Array ist, werden mehrere Basis-URL-Optionen aktiviert,
      zwischen denen der Benutzer wechseln kann.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          Die für alle API-Endpunkte verwendete Authentifizierungsstrategie.
        </ResponseField>
        <ResponseField name="name" type="string">
        Der Name des im API-Playground verwendeten Authentifizierungsparameters.

        Wenn die Methode `basic` ist, sollte das Format `[usernameName]:[passwordName]` sein
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        Der Standardwert, der als Präfix für das Authentifizierungs-Eingabefeld dienen soll.

        Beispiel: Ein `inputPrefix` von `AuthKey` würde das Standardergebnis der Authentifizierungseingabe als `AuthKey` erben.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Konfigurationen für den API-Playground

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Ob der Playground angezeigt, ausgeblendet oder nur der Endpunkt ohne zusätzliche Benutzerinteraktivität angezeigt wird `simple`

          Erfahren Sie mehr in den [Playground-Anleitungen](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Durch Aktivieren dieses Flags wird sichergestellt, dass die Schlüsselreihenfolge auf OpenAPI-Seiten mit der in der OpenAPI-Datei definierten Schlüsselreihenfolge übereinstimmt.

      <Warning>Dieses Verhalten wird bald standardmäßig aktiviert, zu diesem Zeitpunkt wird dieses Feld veraltet.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Eine Zeichenkette oder ein Array von Zeichenketten mit URL(s) oder relativen Pfaden,
  die auf Ihre OpenAPI-Datei verweisen.
  
  Beispiele:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Ein Objekt von Social-Media-Konten, bei dem das Schlüssel-Wert-Paar die Social-Media-Plattform und die Konto-URL darstellt.
  
  Beispiel: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Einer der folgenden Werte: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Beispiel: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    Die URL zur Social-Plattform.
    
    Beispiel: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Konfigurationen zum Aktivieren von Feedback-Schaltflächen

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Aktiviert eine Schaltfläche, mit der Benutzer Änderungen über Pull-Requests vorschlagen können
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Aktiviert eine Schaltfläche, mit der Benutzer ein Issue zur Dokumentation eröffnen können
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Passen Sie den Umschalter für den dunklen Modus an.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Legen Sie fest, ob Sie neuen Benutzern immer den hellen oder dunklen Modus anzeigen möchten.
      Wenn nicht festgelegt, verwenden wir standardmäßig denselben Modus wie das Betriebssystem des Benutzers.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Auf true setzen, um den Umschalter für Hell-/Dunkelmodus auszublenden. Sie können `isHidden` mit `default` kombinieren, um Ihre Docs zu zwingen, nur den hellen oder dunklen Modus zu verwenden. Beispiel:
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Ein Hintergrundbild, das hinter jeder Seite angezeigt wird. Siehe Beispiele bei
  [Infisical](https://infisical.com/docs) und [FRPC](https://frpc.io).
</ResponseField>
