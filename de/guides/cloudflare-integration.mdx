Mit Firma können Sie jeder auf Cloudflare laufenden Anwendung rechtsverbindliche E-Signaturen hinzufügen. Rufen Sie die Firma-API aus Workers oder Pages Functions auf, um Templates zu erstellen, Signaturanfragen zu senden und Abschlüsse per Webhook zu verfolgen.

Dieser Leitfaden behandelt zwei Integrationswege:

1. **Workers** – Eigenständige serverlose Funktionen, die die Firma-REST-API aufrufen. Am besten für APIs, Microservices oder Apps, die Cloudflare Pages nicht nutzen.
2. **Pages Functions** – Dateibasierte serverlose Funktionen, die zusammen mit Ihrer Cloudflare-Pages-Site bereitgestellt werden. Am besten, wenn Sie bereits ein Pages-Projekt haben und alles in einem Repo halten möchten.

Beide Wege nutzen dieselben Firma-API-Endpunkte. Der Unterschied liegt in der Strukturierung und Bereitstellung Ihres Codes.

## Voraussetzungen

- Ein [Firma-Konto](https://app.firma.dev) mit einem API-Schlüssel
- Die [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/install-and-update/) installiert (`npm install -g wrangler`)
- Mindestens ein Firma-Template mit konfigurierten Signaturfeldern

<Note>
  Firma verwendet den rohen API-Schlüssel als Wert des `Authorization`-Headers. Setzen Sie ihm kein `Bearer` voran. Dies unterscheidet sich von vielen anderen APIs.
</Note>

## Wählen Sie Ihren Weg

<Tabs>
  <Tab title="Workers">

### Schritt 1: Ein Worker-Projekt erstellen

Wenn Sie noch kein Workers-Projekt haben, legen Sie eines an:

```bash
npm create cloudflare@latest -- firma-signing
```

Wählen Sie bei Aufforderung **„Hello World" Worker** und anschließend JavaScript oder TypeScript. Nach dem Scaffolding:

```bash
cd firma-signing
```

### Schritt 2: API-Schlüssel als Secret speichern

Fügen Sie Ihren Firma-API-Schlüssel als verschlüsseltes Secret hinzu. Wrangler fordert Sie zum Einfügen des Wertes auf:

```bash
wrangler secret put FIRMA_API_KEY
```

Codieren Sie Ihren API-Schlüssel niemals fest in Quelldateien. Secrets werden verschlüsselt gespeichert und zur Laufzeit in das `env`-Binding Ihres Workers eingefügt.

### Schritt 3: Worker zum Senden von Signaturanfragen erstellen

Ersetzen Sie den Inhalt von `src/index.js` (oder `src/index.ts`, falls Sie TypeScript gewählt haben) durch Folgendes. Dieser Worker akzeptiert eine POST-Anfrage mit Signer-Details, ruft den Firma-Endpunkt `create-and-send` auf und gibt die Signing-Request-ID zurück.

```javascript
const FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api";

export default {
  async fetch(request, env) {
    if (request.method === "OPTIONS") {
      return new Response(null, {
        headers: {
          "Access-Control-Allow-Origin": "*",
          "Access-Control-Allow-Methods": "POST, OPTIONS",
          "Access-Control-Allow-Headers": "Content-Type",
        },
      });
    }

    if (request.method !== "POST") {
      return Response.json({ error: "Method not allowed" }, { status: 405 });
    }

    const { name, template_id, signer_email, signer_first_name, signer_last_name } =
      await request.json();

    const response = await fetch(
      `${FIRMA_API}/signing-requests/create-and-send`,
      {
        method: "POST",
        headers: {
          Authorization: env.FIRMA_API_KEY,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          name,
          template_id,
          recipients: [
            {
              first_name: signer_first_name,
              last_name: signer_last_name,
              email: signer_email,
              designation: "Signer",
              order: 1,
            },
          ],
        }),
      }
    );

    const data = await response.json();

    if (!response.ok) {
      return Response.json({ error: data }, { status: response.status });
    }

    return Response.json({
      signing_request_id: data.id,
      status: "sent",
    });
  },
};
```

<Tip>
  Der `create-and-send`-Endpunkt erstellt die Signaturanfrage und versendet sie in einem einzigen Aufruf an die Empfänger. Wenn Sie die Anfrage vor dem Versand prüfen oder ändern möchten, verwenden Sie `POST /signing-requests`, um einen Entwurf zu erstellen, und anschließend separat `POST /signing-requests/{id}/send`.
</Tip>

### Schritt 4: Bereitstellen

```bash
wrangler deploy
```

Wrangler gibt die URL Ihres Workers aus (etwa `https://firma-signing.<your-subdomain>.workers.dev`). Ihr Frontend kann jetzt Signer-Details per POST an diese URL senden.

### Webhook-Verarbeitung

Um zu verfolgen, wann Dokumente unterzeichnet werden, erstellen Sie einen zweiten Worker, der Firma-Webhook-Ereignisse empfängt. Registrieren Sie im Firma-Dashboard unter **Settings → Webhooks** einen Webhook, der auf die URL dieses Workers zeigt.

```javascript
export default {
  async fetch(request, env) {
    if (request.method !== "POST") {
      return new Response("Method not allowed", { status: 405 });
    }

    const payload = await request.json();
    const { type, data } = payload;

    if (type === "signing_request.completed") {
      const signingRequestId = data.signing_request.id;
      console.log(`Signing request ${signingRequestId} completed`);
    }

    if (type === "signing_request.recipient.signed") {
      const recipientEmail = data.recipient?.email;
      console.log(`${recipientEmail} signed the document`);
    }

    return Response.json({ received: true });
  },
};
```

<Warning>
  Verifizieren Sie in der Produktion stets die Webhook-Signatur mit Ihrem Firma-Webhook-Signing-Secret. Implementierungsdetails finden Sie im [Webhooks-Leitfaden](/guides/webhooks).
</Warning>

### Schritt 1: Pages-Projekt einrichten

Wenn Sie bereits ein Cloudflare-Pages-Projekt haben, überspringen Sie den nächsten Schritt. Andernfalls erstellen Sie eines:

```bash
mkdir firma-app && cd firma-app
npm init -y
mkdir public functions
echo "<h1>Firma App</h1>" > public/index.html
```

Pages Functions liegen im Verzeichnis `/functions`. Jede Datei wird automatisch einem URL-Pfad zugeordnet, sodass `functions/api/send-signing-request.js` zu `/api/send-signing-request` wird.

### Schritt 2: API-Schlüssel als Secret speichern

Fügen Sie Ihren Firma-API-Schlüssel als verschlüsseltes Pages-Secret hinzu:

```bash
wrangler pages secret put FIRMA_API_KEY --project-name your-project-name
```

Alternativ gehen Sie zu **Cloudflare Dashboard → Pages → Ihr Projekt → Settings → Environment variables** und fügen `FIRMA_API_KEY` als verschlüsselte Variable hinzu.

### Schritt 3: Pages-Funktion zum Senden von Signaturanfragen erstellen

Erstellen Sie eine neue Datei unter `functions/api/send-signing-request.js`:

```javascript
const FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api";

export async function onRequestPost(context) {
  const { name, template_id, signer_email, signer_first_name, signer_last_name } =
    await context.request.json();

  const response = await fetch(
    `${FIRMA_API}/signing-requests/create-and-send`,
    {
      method: "POST",
      headers: {
        Authorization: context.env.FIRMA_API_KEY,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        name,
        template_id,
        recipients: [
          {
            first_name: signer_first_name,
            last_name: signer_last_name,
            email: signer_email,
            designation: "Signer",
            order: 1,
          },
        ],
      }),
    }
  );

  const data = await response.json();

  if (!response.ok) {
    return Response.json({ error: data }, { status: response.status });
  }

  return Response.json({
    signing_request_id: data.id,
    status: "sent",
  });
}
```

<Tip>
  Pages Functions exportieren benannte Handler pro HTTP-Methode: `onRequestPost`, `onRequestGet` usw. Wenn Sie mehrere Methoden verarbeiten müssen, exportieren Sie `onRequest` und prüfen Sie `context.request.method` darin.
</Tip>

### Schritt 4: Bereitstellen

Wenn Ihr Pages-Projekt mit einem Git-Repo verbunden ist, pushen Sie Ihre Änderungen und Cloudflare deployt automatisch. Für manuelle Deployments:

```bash
wrangler pages deploy public --project-name your-project-name
```

Ihre Funktion ist jetzt unter `https://your-project.pages.dev/api/send-signing-request` live.

### Webhook-Verarbeitung

Erstellen Sie eine zweite Pages-Funktion unter `functions/api/firma-webhook.js`, um eingehende Firma-Webhook-Ereignisse zu verarbeiten. Registrieren Sie diese URL im Firma-Dashboard unter **Settings → Webhooks**.

```javascript
export async function onRequestPost(context) {
  const payload = await context.request.json();

    console.log(`${recipientEmail} signed the document`);

```

## Embedded Signing

Für Apps, in denen Unterzeichner Dokumente direkt in Ihrer Oberfläche ausfüllen, bietet Firma ein einbettbares Signaturerlebnis. Die `create-and-send`-Antwort enthält eine `first_signer.id` (die `signing_request_user_id`) und einen fertigen `first_signer.signing_link`. Laden Sie ihn in einem iframe:

```html
<iframe
  src="https://app.firma.dev/signing/{signing_request_user_id}"
  style="width:100%;height:900px;border:0;"
  allow="camera;microphone;clipboard-write"
  title="Document Signing"
></iframe>
```

Vollständige Einrichtungsanweisungen einschließlich Sicherheits-Best-Practices finden Sie im [Embedded-Signing-Leitfaden](/guides/embeddable-signing).

## Bonus: MCP-Verbindung für KI-gestütztes Bauen

Firma bietet einen [Docs-MCP-Server](/guides/mcp), mit dem sich KI-Coding-Tools verbinden können. Wenn Sie beim Bauen Ihrer Cloudflare-Integration einen KI-Assistenten nutzen, kann dieser die Firma-Dokumentation durchsuchen und korrekte API-Aufrufe generieren.

Fügen Sie die MCP-Server-URL zur Konfiguration Ihres Tools hinzu:

```text
https://docs.firma.dev/mcp
```

Dies gilt nur für die Entwicklungsumgebung und wirkt sich nicht auf Ihre bereitgestellte App aus.
