Betten Sie den Template-Editor von Firma in Ihre Anwendung ein und nutzen Sie JWT-Authentifizierung für sicheren, zeitlich begrenzten Zugriff. Ideal für White-Label-Integrationen und Multi-Tenant-Anwendungen.

## Anwendungsfälle

- **White-Label-Template-Bearbeitung**: Ermöglichen Sie Benutzern das Bearbeiten von Templates unter Ihrer Marke
- **Multi-Tenant-Anwendungen**: Sicherer Template-Zugriff pro Benutzer, ohne API-Schlüssel offenzulegen
- **Eingebettete Workflows**: Nahtlose Template-Erstellung innerhalb Ihres Produkts
- **Zeitlich begrenzter Zugriff**: Tokens laufen aus Sicherheitsgründen automatisch ab

## So funktioniert es

1. Ihr Server fordert mit Ihrem API-Schlüssel ein JWT-Token von der Firma-API an
2. Firma gibt ein kurzlebiges JWT-Token mit der Template-ID zurück
3. Ihr Frontend bettet den Editor mit dem JWT-Token ein
4. Das Token läuft automatisch ab (konfigurierbare Gültigkeit)

<Note>
  **Rate-Limit**: JWT-Endpunkte unterstützen **120 Anfragen pro Minute pro API-Schlüssel** für Anwendungen mit hohem Volumen.
</Note>

## JWT-Authentifizierung

### JWT-Token generieren

Generieren Sie ein JWT-Token für ein bestimmtes Template über den Endpunkt `/generate-template-token`.

**Endpunkt**: `POST /generate-template-token`

**Request-Body**:

```json
{
  "companies_workspaces_templates_id": "123e4567-e89b-12d3-a456-426614174000"
}
```

**Antwort (201 Created)**:

```json
{
  "token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_at": "2024-04-20T10:00:00Z",
  "jwt_record_id": "jwt123-e89b-12d3-a456-426614174000"
}
```

**Rate-Limit-Header**:

```
X-RateLimit-Limit: 120
X-RateLimit-Remaining: 119
X-RateLimit-Reset: 1729180800
```

## Implementierungsleitfaden

<Warning>
  **Sicherheit**: Legen Sie Ihren API-Schlüssel niemals im clientseitigen Code offen. Generieren Sie JWT-Tokens stets von Ihrem sicheren Backend aus.
</Warning>

### Backend: JWT-Token generieren

Rufen Sie die Firma-API aus Ihrem Backend auf, um ein JWT-Token zu generieren. Ihr Backend-Endpunkt sollte eine Template-ID akzeptieren und das JWT an Ihr Frontend zurückgeben.

**Node.js-Beispiel**:

```js
// Example: Simple backend function to generate JWT
async function generateTemplateToken(templateId) {
  const response = await fetch('https://api.firma.dev/functions/v1/signing-request-api/generate-template-token', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.FIRMA_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      companies_workspaces_templates_id: templateId
    })
  })
  
  const data = await response.json()
  return data.token
}
```

**Python-Beispiel**:

```py
import os
import requests

def generate_template_token(template_id):
    response = requests.post(
        'https://api.firma.dev/functions/v1/signing-request-api/generate-template-token',
        headers={
            'Authorization': f'Bearer {os.getenv("FIRMA_API_KEY")}',
            'Content-Type': 'application/json'
        },
        json={
            'companies_workspaces_templates_id': template_id
        }
    )
    data = response.json()
    return data['token']
```

### Frontend-Implementierung – HTML / Vanilla-JavaScript

```html
<!-- Load the Firma Template Editor library -->
<script src="https://api.firma.dev/functions/v1/embed-proxy/template-editor.js"></script>

<!-- Create a container for the editor -->
<div id="template-editor-container"></div>

<script>
  // Assuming you've already fetched the JWT from your backend
  const jwt = 'eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...';
  const templateId = '08f72e3b-79a8-4eac-a268-b3f9efaf6573';
  
  // Initialize the template editor
  const editor = new FirmaTemplateEditor({
    container: '#template-editor-container',
    jwt: jwt,
    templateId: templateId,
    
    // Optional configuration
    width: '100%',        // Custom width (default: '100%')
    height: '100vh',      // Custom height (default: '100vh')
    showCloseButton: true, // Show close button in header
    
    // Optional callbacks
    onSave: (data) => {
      console.log('Template saved:', data);
    },
    onClose: (data) => {
      if (data.hasUnsavedChanges) {
        // Prompt user to save before leaving
      }
      console.log('Editor closed, unsaved changes:', data.hasUnsavedChanges);
    },
    onError: (error) => {
      console.error('Editor error:', error);
    },
    onLoad: (template) => {
      console.log('Editor loaded successfully:', template);
    }
  });
</script>
```

### Frontend-Implementierung – React

```jsx
import { useEffect, useRef, useState } from 'react';

function TemplateEditorComponent({ templateId, jwt }) {
  const containerRef = useRef<HTMLDivElement>(null);
  const editorRef = useRef<any>(null);
  const [isLoaded, setIsLoaded] = useState(false);

  // Load the Firma Template Editor library
  useEffect(() => {
    const script = document.createElement('script');
    script.src = 'https://api.firma.dev/functions/v1/embed-proxy/template-editor.js';
    script.async = true;
    
    script.onload = () => setIsLoaded(true);
    document.body.appendChild(script);
    
    return () => {
      if (document.body.contains(script)) {
        document.body.removeChild(script);
      }
    };
  }, []);

  // Initialize the editor when library and JWT are ready
  useEffect(() => {
    if (!isLoaded || !containerRef.current || !jwt) return;

    editorRef.current = new window.FirmaTemplateEditor({
      container: containerRef.current,
      jwt: jwt,
      templateId: templateId,
      
      // Optional configuration
      theme: 'dark',        // 'light' | 'dark' (default: 'dark')
      readOnly: false,      // Disable editing (default: false)
      width: '100%',        // Custom width (default: '100%')
      height: '100vh',      // Custom height (default: '100vh')
      showCloseButton: true, // Show close button in header
      
      // Optional callbacks
      onSave: (data) => console.log('Saved:', data),
      onClose: (data) => {
        if (data.hasUnsavedChanges) {
          // Prompt user to save before leaving
        }
      },
      onError: (error) => console.error('Error:', error),
      onLoad: (template) => console.log('Loaded:', template)
    });

    return () => {
      if (editorRef.current?.destroy) {
        editorRef.current.destroy();
      }
    };
  }, [isLoaded, jwt, templateId]);

  return <div ref={containerRef} className="w-full h-full" />;
}

export default TemplateEditorComponent;
```

### Konfigurationsoptionen

| Option       | Typ                   | Beschreibung                                                             |
| ------------ | --------------------- | ------------------------------------------------------------------------ |
| `container`  | HTMLElement \| string | DOM-Element oder CSS-Selektor, in dem der Editor gemountet wird (erforderlich) |
| `jwt`        | string                | Authentifizierungstoken von Ihrem Backend (erforderlich)                 |
| `templateId` | string                | Template-Kennung (erforderlich)                                          |
| `theme`      | `'dark'` \| `'light'` | Editor-Theme (Standard: `'dark'`)                                        |
| `readOnly`   | boolean               | Nur-Lese-Modus aktivieren (Standard: `false`)                            |
| `height`     | string                | Container-Höhe (Standard: `'100vh'`)                                     |
| `width`      | string                | Container-Breite (Standard: `'100%'`)                                    |
| `showCloseButton` | boolean          | Schließen-Button im Editor-Header anzeigen (Standard: `false`)           |
| `onSave`     | function              | Callback, wenn das Template gespeichert wird                             |
| `onClose`    | function              | Callback, wenn der Schließen-Button geklickt oder `triggerClose()` aufgerufen wird. Erhält `{ hasUnsavedChanges: boolean }` |
| `onError`    | function              | Callback für Fehlerbehandlung                                            |
| `onLoad`     | function              | Callback, wenn der Editor mit den Template-Daten geladen wird            |

### Instanzmethoden

| Methode | Beschreibung |
|---------|--------------|
| `triggerClose()` | Löst den Schließen-Ablauf programmatisch aus. Prüft auf ungespeicherte Änderungen und ruft `onClose` mit `{ hasUnsavedChanges: boolean }` auf |

Verwenden Sie `triggerClose()`, wenn Ihre Host-Anwendung den Editor aus ihrer eigenen Oberfläche schließen muss – zum Beispiel bei einem übergeordneten Navigationsereignis oder einem „Zurück"-Button:

```js
// Close the editor from your app's UI
document.getElementById('my-back-button').addEventListener('click', () => {
  editor.triggerClose();
});
```

## postMessage-Ereignisse (Editor → Host)

Der Editor von Firma sendet postMessage-Ereignisse für wichtige Lebenszyklus-Aktionen. Nachfolgend ein empfohlenes, minimales Ereignisschema, das Sie implementieren können, um auf Speichern- und Publish-Ereignisse des Editors zu reagieren. Falls Sie ein kanonisches Schema in Ihrer Plattform haben, ersetzen Sie diese durch Ihre offiziellen Ereignisnamen.

<Tip>
  Nutzen Sie diese Ereignisse, um die Editor-Aktivität zu verfolgen, ohne zusätzliche API-Aufrufe zu machen – so bleiben Sie innerhalb der Rate-Limits.
</Tip>

Ereignis-Envelope (window.postMessage-Payload):

```json
{
	"type": "editor.event",
	"event": "editor.saved", // or editor.published, editor.closed
	"payload": {
		"template_id": "tmpl_123",
		"updated_at": "2025-09-04T12:34:56Z",
		"draft": false
	}
}
```

### Beispiel: Client-Listener (Plain JS)

```js
window.addEventListener('message', (ev) => {
	// 1) Validate origin
	if (ev.origin !== 'https://app.firma.dev') return
	// 2) Validate shape
	const data = ev.data || {}
	if (data.type !== 'editor.event') return

	switch (data.event) {
		case 'editor.saved':
			console.log('Template saved', data.payload)
			// Optionally fetch the latest template via API
			break
		case 'editor.published':
			console.log('Template published', data.payload)
			break
		case 'editor.closed':
			console.log('Editor closed')
			break
		default:
			break
	}
})
```

## Verwaltung des Token-Lebenszyklus

JWT-Tokens werden mit ausreichender Gültigkeit für typische Bearbeitungssitzungen generiert. Der Editor verarbeitet Token-Ablauf automatisch.

### Automatischer Ablauf

JWT-Tokens laufen basierend auf dem `expires_at`-Zeitstempel automatisch ab. Nach Ablauf:

- Der eingebettete Editor lehnt das Token ab
- Benutzer müssen ein neues Token anfordern, um fortzufahren
- Kein API-Aufruf nötig – Tokens laufen passiv ab

## Sicherheits-Best-Practices

<Warning>
  **Legen Sie Ihren API-Schlüssel niemals im clientseitigen Code offen.** Generieren Sie JWT-Tokens stets über einen sicheren Server-Endpunkt.
</Warning>

### ✅ Do's

- ✅ Tokens vom Backend-Server generieren
- ✅ Rate-Limits überwachen
- ✅ HTTPS für alle API-Anfragen verwenden

### ❌ Don'ts

- ❌ API-Schlüssel nicht im Frontend-Code offenlegen
- ❌ Tokens nicht über Benutzer hinweg wiederverwenden
- ❌ JWT-Tokens nicht loggen (Sicherheitsrisiko)
- ❌ Tokens nicht zwischen verschiedenen Templates teilen

## Fehlerbehebung

### Fehler: Token abgelaufen

**Symptom**: Editor zeigt „Token expired" oder Authentifizierungsfehler an

**Lösung**:

- Token-Erneuerung vor Ablauf implementieren
- Neues Token generieren und das iframe neu laden
- Synchronisation der Systemuhr prüfen

### 401 Unauthorized

**Symptom**: JWT-Generierung schlägt mit 401 fehl

**Mögliche Ursachen**:

- Ungültiger oder fehlender API-Schlüssel
- API-Schlüssel hat nicht die erforderlichen Berechtigungen
- API-Schlüssel ist deaktiviert

**Lösung**: API-Schlüssel im Dashboard überprüfen und Berechtigungen prüfen

### 404 Not Found

**Symptom**: Template nicht gefunden beim Generieren eines JWTs

**Mögliche Ursachen**:

- Template-ID existiert nicht
- Template gehört zu einem anderen Workspace
- Template wurde gelöscht

**Lösung**: Template-ID und Workspace-Zugriff überprüfen

### Rate-Limit überschritten

**Symptom**: 429 Too Many Requests

**Lösung**:

- Token-Caching implementieren
- Token-Ablaufzeit erhöhen
- Auf Rate-Limit-Reset warten (Header `X-RateLimit-Reset` prüfen)
- Retry-Logik mit Backoff implementieren
