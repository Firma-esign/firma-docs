Firma bietet zwei MCP-Server, mit denen KI-Assistenten auf unterschiedliche Weise mit Firma arbeiten können:

- **Data MCP** (`https://mcp.firma.dev/mcp`) – Gewährt KI-Assistenten direkten Zugriff auf die Firma-API. Verwalten Sie Vorlagen, erstellen Sie Signaturanfragen, konfigurieren Sie Arbeitsbereiche und mehr über natürlichsprachliche Konversationen. Erfordert OAuth-Authentifizierung.
- **Docs MCP** (`https://docs.firma.dev/mcp`) – Gewährt KI-Assistenten Zugriff auf die vollständige Firma-Dokumentation und API-Referenz. Nutzen Sie ihn, um präzisen Integrationscode mit korrekten Endpunkten, Request-Strukturen und Auth-Mustern zu erzeugen. Keine Authentifizierung erforderlich.

Die meisten Entwickler möchten beide verbunden haben: Data MCP für Operationen auf Firma-Daten und Docs MCP zum Erstellen von Firma-Integrationen.

## Was ist MCP?

Das [Model Context Protocol](https://modelcontextprotocol.io) (MCP) ist ein offener Standard, der KI-Assistenten mit externen Tools und Datenquellen verbindet. Anstatt API-Aufrufe manuell zu schreiben, beschreiben Sie in natürlicher Sprache, was Sie benötigen, und der KI-Assistent nutzt die MCP-Server von Firma, um die richtigen API-Aufrufe auszuführen oder die richtige Dokumentation für Sie nachzuschlagen.

## Einrichtung

<Tabs>

<Tab title="Data MCP">

Der Data-MCP-Server authentifiziert sich über OAuth 2.1 mit PKCE anhand Ihrer Firma-Anmeldedaten. Wenn Sie sich über einen unterstützten Client verbinden, wird der OAuth-Ablauf automatisch abgewickelt.

**Unterstützte Clients:**

- **Claude** – Claude Code, Claude Desktop, Claude.ai
- **ChatGPT** – Über Developer-Mode-Connectors
- **Cursor** – Native MCP-Unterstützung
- **GitHub Copilot** – VS Code 1.101+
- **OpenAI Codex** – Über CLI oder Konfiguration
- **Andere MCP-Clients** – Jeder Client, der den [Streamable HTTP-Transport](https://modelcontextprotocol.io/specification/2025-03-26/basic/transports#streamable-http) implementiert

<Tabs>

<Tab title="Claude Code">

Fügen Sie den Firma-Data-MCP-Server über die CLI hinzu:

```bash
claude mcp add --transport http firma-api https://mcp.firma.dev/mcp
```

Oder fügen Sie ihn zu Ihrer Claude-Code-Konfiguration (`.claude/settings.json`) hinzu:

```json
{
  "mcpServers": {
    "firma-api": {
      "type": "url",
      "url": "https://mcp.firma.dev/mcp"
    }
  }
}
```

Bei der ersten Nutzung werden Sie aufgefordert, sich mit Ihrem Firma-Konto anzumelden.

### Was der Docs MCP bereitstellt

Der Docs-MCP-Server gibt KI-Assistenten Zugriff auf:

- **API-Referenz** – Endpunkt-URLs, Request-/Response-Strukturen, Authentifizierungsdetails
- **Integrationsleitfäden** – Schritt-für-Schritt-Anleitungen für jede unterstützte Plattform
- **Funktionsleitfäden** – Webhooks, eingebettetes Signieren, White-Labeling, E-Mail-Vorlagen und mehr
- **Codebeispiele** – Funktionierende Beispiele in Node.js, Python und weiteren Sprachen

### Wann Sie ihn verwenden sollten

Verweisen Sie in Ihren Prompts auf „die Firma-Dokumentation", damit der Assistent diesen Server abfragt:

```text
Using the Firma docs, create an Express route that sends a signing request
using the create-and-send endpoint. Use process.env.FIRMA_API_KEY for auth.
```

Ohne diesen Verweis greift Ihr Assistent möglicherweise auf allgemeines Wissen zurück und übersieht Endpunkt- oder Authentifizierungsdetails.

## Arbeitsbereich auswählen

Nach der Anmeldung am Data MCP starten Sie ohne ausgewählten Arbeitsbereich. Sie können entweder:

- **den Assistenten bitten, einen Arbeitsbereich auszuwählen** – z. B. „Wechsle in den Marketing-Arbeitsbereich"
- **den Assistenten pro Anfrage eine Arbeitsbereichs-ID übergeben lassen** – der Assistent kann einen `workspace_id`-Parameter bei einzelnen Tool-Aufrufen mitgeben

Der Assistent listet typischerweise zuerst Ihre Arbeitsbereiche auf und fragt, welcher verwendet werden soll.

## Beispielkonversationen

### Alle Signaturanfragen auflisten

> „Zeige mir alle ausstehenden Signaturanfragen in meinem Marketing-Arbeitsbereich"

Der Assistent wählt den Arbeitsbereich aus, ruft den Endpunkt zum Auflisten von Signaturanfragen auf und präsentiert die Ergebnisse.

### Signaturanfrage erstellen und senden

> „Erstelle eine Signaturanfrage aus der NDA-Vorlage und sende sie an jane@example.com"

Der Assistent findet die Vorlage, dupliziert sie in eine Signaturanfrage, setzt den Empfänger und sendet sie.

### Signaturstatus prüfen

> „Wie ist der Status der Signaturanfrage, die ich letzte Woche an John geschickt habe?"

Der Assistent durchsucht kürzliche Signaturanfragen und gibt den aktuellen Status zurück, inklusive der Information, welche Empfänger unterschrieben haben.

### Integrationscode erzeugen

> „Verwende die Firma-Dokumentation und erstelle eine Next.js-API-Route, die eine Signaturanfrage über den create-and-send-Endpunkt versendet"

Der Assistent fragt den Docs MCP nach dem korrekten Endpunkt, der Request-Struktur und dem Auth-Muster ab und erzeugt dann funktionierenden Code.

## Ratenbegrenzungen

MCP-Anfragen unterliegen denselben [Ratenbegrenzungen](rate-limits) wie direkte API-Aufrufe. Jeder Tool-Aufruf entspricht einer API-Anfrage. Der Docs-MCP-Server hat keine Ratenbegrenzungen.

## Fehlerbehebung

### Tools werden nicht geladen

Wenn Ihr Client den Server als verbunden anzeigt, aber keine Tools erscheinen:

- **Claude.ai / ChatGPT**: Trennen Sie den Connector und verbinden Sie ihn in den Einstellungen erneut
- **Cursor / VS Code**: Starten Sie die Anwendung nach dem Hinzufügen der Konfiguration neu
- **Claude Desktop**: Starten Sie die App neu – Änderungen an `claude_desktop_config.json` erfordern einen Neustart

### Arbeitsbereich nicht gefunden

Stellen Sie sicher, dass Sie nach der Verbindung mit dem Data MCP einen Arbeitsbereich ausgewählt haben. Verwenden Sie „Liste meine Arbeitsbereiche auf", um die verfügbaren Optionen zu sehen.

### Authentifizierungsfehler

Melden Sie sich ab und wieder an, um Ihre Sitzung zu aktualisieren. Wenn das Problem weiterhin besteht, trennen Sie den Server und verbinden Sie ihn in den Einstellungen Ihres Clients erneut. Der Docs MCP erfordert keine Authentifizierung.
