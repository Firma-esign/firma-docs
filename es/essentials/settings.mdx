---
title: 'Configuración global'
description: 'Mintlify te da control total sobre el aspecto y la sensación de tu documentación mediante el archivo docs.json'
icon: 'gear'
---

Todo sitio de Mintlify necesita un archivo `docs.json` con los ajustes básicos de configuración. Obtén más información sobre las [propiedades](#properties) a continuación.

## Propiedades

<ResponseField name="name" type="string" required>
Nombre de tu proyecto. Se utiliza para el título global.

Ejemplo: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un array de grupos con todas las páginas dentro de cada grupo
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    El nombre del grupo.

    Ejemplo: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    Las rutas relativas a los archivos markdown que actuarán como páginas.

    Ejemplo: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Ruta a la imagen del logotipo o un objeto con la ruta a las imágenes del logotipo para el modo "light" y "dark"
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Ruta al logotipo en modo claro
    </ResponseField>
    <ResponseField name="dark" type="string">
      Ruta al logotipo en modo oscuro
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      A dónde te lleva al hacer clic en el logotipo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Ruta a la imagen del favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de color hexadecimales para tu tema global
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      El color principal. Se usa con más frecuencia para el contenido destacado, los encabezados de sección y los acentos en modo claro
    </ResponseField>
    <ResponseField name="light" type="string">
      El color principal para el modo oscuro. Se usa con más frecuencia para el contenido destacado, los encabezados de sección y los acentos en modo oscuro
    </ResponseField>
    <ResponseField name="dark" type="string">
      El color principal para los botones importantes
    </ResponseField>
    <ResponseField name="background" type="object">
      El color del fondo tanto en modo claro como oscuro
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          El código de color hexadecimal del fondo en modo claro
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          El código de color hexadecimal del fondo en modo oscuro
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array de `name`s y `url`s de los enlaces que quieras incluir en la barra superior
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    El nombre del botón.

    Ejemplo: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    La URL a la que se accede al hacer clic en el botón. Ejemplo: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link muestra un botón. GitHub muestra la información del repositorio en la URL proporcionada, incluido el número de estrellas de GitHub.
    </ResponseField>
    <ResponseField name="url" type="string">
    Si es `link`: a dónde enlaza el botón.
    
    Si es `github`: enlace al repositorio del que cargar la información de GitHub.
    </ResponseField>
    <ResponseField name="name" type="string">
    Texto dentro del botón. Solo es obligatorio si `type` es `link`.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array de nombres de versión. Úsalo solo si quieres mostrar diferentes versiones
  de la documentación con un desplegable en la barra de navegación.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Un array de los anclajes; incluye `icon`, `color` y `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    El icono de [Font Awesome](https://fontawesome.com/search?q=heart) que se usa para destacar el anclaje.

    Ejemplo: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    El nombre de la etiqueta del anclaje.

    Ejemplo: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      El comienzo de la URL que indica qué páginas entran en el anclaje. Normalmente, este es el nombre de la carpeta donde has puesto tus páginas.
    </ResponseField>
    <ResponseField name="color" type="string">
      El color hexadecimal del fondo del icono del anclaje. También puede ser un degradado si pasas un objeto con las propiedades `from` y `to`, cada una con un color hexadecimal.
    </ResponseField>
    <ResponseField name="version" type="string">
      Se usa si quieres ocultar un anclaje hasta que se seleccione la versión correcta de la documentación.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Pasa `true` si quieres ocultar el anclaje hasta que enlaces directamente a alguien a documentación dentro de él.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: "brands", "duotone", "light", "sharp-solid", "solid" o "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Sobrescribe las configuraciones por defecto del anclaje superior.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      El nombre del anclaje superior
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Icono de Font Awesome.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: "brands", "duotone", "light", "sharp-solid", "solid" o "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Un array de pestañas de navegación.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      El nombre de la etiqueta de la pestaña.
    </ResponseField>
    <ResponseField name="url" type="string">
      El comienzo de la URL que indica qué páginas entran en la pestaña. Normalmente,
      este es el nombre de la carpeta donde has puesto tus páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuración para los ajustes de la API. Obtén más información sobre las páginas de API en [Componentes de API](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      La URL base para todos los endpoints de la API. Si `baseUrl` es un array, habilitará varias opciones de URL base
      que el usuario podrá alternar.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          La estrategia de autenticación utilizada para todos los endpoints de la API.
        </ResponseField>
        <ResponseField name="name" type="string">
        El nombre del parámetro de autenticación utilizado en el playground de API.

        Si el método es `basic`, el formato debe ser `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        El valor por defecto diseñado para ser un prefijo del campo de entrada de autenticación.

        Por ejemplo, un `inputPrefix` de `AuthKey` heredaría el resultado de entrada por defecto del campo de autenticación como `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configuraciones del playground de la API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Si el playground se muestra, se oculta o solo muestra el endpoint sin interactividad adicional del usuario `simple`

          Obtén más información en las [guías del playground](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Al activar esta opción, se garantiza que el orden de las claves en las páginas de OpenAPI coincida con el orden de claves definido en el archivo OpenAPI.

      <Warning>Este comportamiento se habilitará pronto de forma predeterminada, momento en el cual este campo quedará obsoleto.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Una cadena o un array de cadenas con las URL o las rutas relativas que apuntan a tu
  archivo OpenAPI.
  
  Ejemplos:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un objeto de cuentas de redes sociales donde el par clave:propiedad representa la plataforma de redes sociales y la URL de la cuenta.
  
  Ejemplo: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Uno de los siguientes valores: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Ejemplo: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    La URL de la plataforma social.
    
    Ejemplo: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configuraciones para habilitar los botones de comentarios

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Habilita un botón que permite a los usuarios sugerir ediciones mediante pull requests
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Habilita un botón que permite a los usuarios abrir una incidencia sobre la documentación
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personaliza el interruptor de modo oscuro.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Establece si siempre quieres mostrar el modo claro u oscuro a los usuarios nuevos. Cuando no
      está establecido, usamos por defecto el mismo modo que el sistema operativo del usuario.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Establece en true para ocultar el interruptor de modo oscuro/claro. Puedes combinar `isHidden` con `default` para forzar que tu documentación use solo modo claro u oscuro. Por ejemplo:
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Una imagen de fondo que se mostrará detrás de cada página. Consulta el ejemplo con
  [Infisical](https://infisical.com/docs) y [FRPC](https://frpc.io).
</ResponseField>
