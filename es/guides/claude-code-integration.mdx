Firma te permite añadir firmas electrónicas con validez legal a cualquier cosa que construyas con Claude Code. Conecta los servidores MCP de Firma y Claude Code podrá generar código de integración con Firma preciso, gestionar tus solicitudes de firma y operar sobre tus datos de Firma directamente desde la terminal o tu IDE.

## Requisitos previos

- Una [cuenta de Firma](https://app.firma.dev) con una clave de API
- Claude Code instalado ([CLI](https://docs.anthropic.com/en/docs/claude-code), [extensión de VS Code](https://marketplace.visualstudio.com/items?itemName=anthropic.claude-code) o [plugin de JetBrains](https://plugins.jetbrains.com/plugin/claude-code))
- Al menos una plantilla de Firma con campos de firma configurados

<Note>
  Firma utiliza la clave de API sin procesar como valor del encabezado `Authorization`; no le antepongas `Bearer`. Esto difiere de otras muchas APIs.
</Note>

## Primeros pasos

### Paso 1: añade los servidores MCP de Firma

Añade ambos servidores MCP de Firma a la configuración de tu proyecto. En la raíz de tu proyecto, crea o edita `.claude/settings.json`:

```json
{
  "mcpServers": {
    "firma-api": {
      "type": "url",
      "url": "https://mcp.firma.dev/mcp"
    },
    "firma-docs": {
      "type": "url",
      "url": "https://docs.firma.dev/mcp"
    }
  }
}
```

Como alternativa, añádelos a la configuración de tu usuario en `~/.claude/settings.json` para que estén disponibles en todos los proyectos.

- **firma-api** da a Claude Code acceso directo a la API de Firma: 84 herramientas para solicitudes de firma, plantillas, espacios de trabajo y webhooks.
- **firma-docs** da a Claude Code acceso a la documentación completa de Firma para que genere código de integración preciso.

Reinicia Claude Code después de guardar la configuración. En el primer uso de una herramienta de la API de Firma, Claude Code te guiará para iniciar sesión con tu cuenta de Firma mediante OAuth.

<Note>
  Cuándo usar cada servidor: `firma-api` es para hacer cosas (enviar solicitudes de firma, gestionar plantillas). `firma-docs` es para construir cosas (generar código de integración con detalles precisos de la API). La mayoría de los desarrolladores querrán tener ambos conectados.
</Note>

### Paso 2: usa Firma desde Claude Code

Una vez conectado, puedes pedir a Claude Code que construya integraciones con Firma u opere sobre tus datos:

**Generar una integración de backend:**

```text
Using the Firma API docs, create an Express route that:
1. Accepts a name, template_id, signer email, and signer name
2. Uses the create-and-send endpoint to send a signing request
3. Returns the signing request ID and the signer's signing link
Store the API key in process.env.FIRMA_API_KEY.
```

**Listar tus solicitudes de firma:**

```text
List all pending signing requests in my default workspace.
```

**Enviar una solicitud de firma:**

```text
Create a signing request from the NDA template and send it to jane@example.com
```

**Añadir firma integrada a una aplicación React:**

```text
Using the Firma docs, add an embedded signing view to this React app.
After a signing request is created, render the signer's signing_link
in an iframe.
```

**Configurar webhooks:**

```text
Using the Firma docs, create a webhook handler that listens for
signing_request.completed events and updates the contract status
in our Postgres database.
```

Referenciar explícitamente "los docs de Firma" indica a Claude Code que consulte el servidor MCP `firma-docs` antes de escribir código. Sin eso, puede recurrir al conocimiento general y omitir detalles de endpoints o autenticación.

## Lo que genera Claude Code

Cuando pides a Claude Code que construya una integración con Firma, el código de backend generado debería tener este aspecto:

```typescript
import express from "express";

const FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api";
const app = express();
app.use(express.json());

app.post("/api/signing-requests", async (req, res) => {
  const { name, template_id, signer_email, signer_first_name, signer_last_name } =
    req.body;

  const response = await fetch(
    `${FIRMA_API}/signing-requests/create-and-send`,
    {
      method: "POST",
      headers: {
        Authorization: process.env.FIRMA_API_KEY,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        name,
        template_id,
        recipients: [
          {
            first_name: signer_first_name,
            last_name: signer_last_name,
            email: signer_email,
            designation: "Signer",
            order: 1,
          },
        ],
      }),
    }
  );

  const data = await response.json();

  if (!response.ok) return res.status(response.status).json({ error: data });
  res.json({
    signing_request_id: data.id,
    signing_request_user_id: data.first_signer.id,
    signing_link: data.first_signer.signing_link,
  });
});

app.listen(3000);
```

El endpoint `create-and-send` crea la solicitud de firma y la envía a los destinatarios de forma atómica. Si necesitas revisar o modificar la solicitud antes de enviarla, usa `POST /signing-requests` para crear un borrador y luego `POST /signing-requests/{id}/send` por separado.

<Warning>
  Nunca expongas tu clave de API en el código de frontend. Llama siempre a la API de Firma desde un backend donde los secretos se mantengan seguros.
</Warning>

## Integración de webhooks

Para hacer un seguimiento de los eventos de firma en tiempo real, pide a Claude Code que añada un gestor de webhook:

```text
Using the Firma docs, create a webhook handler that receives Firma events.
When signing_request.completed fires, update the contract record in the database.
```

El agente generará algo como esto:

```typescript
app.post("/api/firma-webhook", async (req, res) => {
  const { type, data } = req.body;

  if (type === "signing_request.completed") {
    const signingRequestId = data.signing_request.id;
    // Update your database or trigger the next workflow step.
  }

  res.json({ received: true });
});
```

En el panel de Firma, en **Settings > Webhooks**, registra la URL de tu endpoint. Firma envía eventos para todos los cambios de estado importantes. Consulta la [guía de webhooks](/guides/webhooks) para ver la lista completa de eventos y la verificación de firma.

<Warning>
  Verifica siempre la firma del webhook usando tu secreto de firma de webhook de Firma en producción. Consulta la [guía de webhooks](/guides/webhooks) para conocer los detalles de implementación.
</Warning>

## Firma integrada

Para aplicaciones donde los firmantes completan los documentos dentro de tu UI en lugar de abrir una página alojada en Firma, la respuesta de `create-and-send` incluye `first_signer.id` (el `signing_request_user_id`) y un `first_signer.signing_link` listo para usar. Carga la URL del firmante en un iframe:

```html
<iframe
  src="https://app.firma.dev/signing/{signing_request_user_id}"
  style="width:100%;height:900px;border:0;"
  allow="camera;microphone;clipboard-write"
  title="Document Signing"
></iframe>
```

Consulta la [guía de firma integrada](/guides/embeddable-signing) para conocer la configuración completa, incluyendo las mejores prácticas de seguridad.

## Consejos

- **Usa `firma-docs` para construir y `firma-api` para operar.** El servidor de documentación ayuda a Claude Code a escribir código de integración correcto. El servidor de API le permite gestionar solicitudes de firma, plantillas y espacios de trabajo directamente.
- **Pasa `template_id` explícitamente.** Las plantillas son la forma más segura de restringir lo que el agente puede enviar.
- **Valida antes de enviar.** Para documentos más críticos, pide a Claude Code que cree un borrador con `POST /signing-requests`, y luego confirma antes de llamar a `POST /signing-requests/{id}/send`.
- **Espacios de trabajo para aplicaciones multitenant.** Si estás construyendo un producto SaaS, dale a cada cliente final su propio espacio de trabajo de Firma para que las plantillas y el uso queden aislados.
