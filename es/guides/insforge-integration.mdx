Firma te permite añadir firmas electrónicas con validez legal a cualquier aplicación construida sobre InsForge. Como InsForge le da a tu agente de código compute, alojamiento, autenticación y almacenamiento de fábrica, puedes añadir Firma a una aplicación existente construida por un agente añadiendo una función de compute y un gestor de webhook.

Esta guía cubre dos rutas de integración:

1. **Función de compute (por aplicación)**: escribe una función de TypeScript que llame directamente a la API REST de Firma. Ideal para aplicaciones individuales o lógica personalizada.
2. **Generación asistida por MCP (a nivel del agente)**: conecta el servidor MCP de documentación de Firma a tu agente de código para que pueda escribir la integración por ti con referencias precisas de la API.

## Requisitos previos

- Una [cuenta de Firma](https://app.firma.dev) con una clave de API
- Un proyecto de InsForge con compute y almacenamiento habilitados
- Al menos una plantilla de Firma con campos de firma configurados

<Note>
  Firma utiliza la clave de API sin procesar como valor del encabezado `Authorization`; no le antepongas `Bearer`. Esto difiere de otras muchas APIs.
</Note>

## Primeros pasos

<Tabs>

<Tab title="Función de compute">

Este es el enfoque más directo. Escribes una función de compute de TypeScript que llama a la API REST de Firma.

### Paso 1: guarda tu clave de API como un secreto

En el panel de tu proyecto de InsForge, añade un secreto llamado `FIRMA_API_KEY` con tu clave de API de Firma como valor. Consulta la [documentación de secretos de InsForge](https://docs.insforge.dev/api-reference/admin/create-a-new-secret) para más detalles.

El secreto se cifra y está automáticamente disponible en tus funciones de compute a través de `Deno.env.get("FIRMA_API_KEY")`.

<Warning>
  Nunca expongas tu clave de API en el código del frontend. Llama siempre a la API de Firma desde funciones de compute donde los secretos se mantienen seguros.
</Warning>

### Paso 2: crea una función de compute para enviar solicitudes de firma

Crea una nueva función en tu proyecto de InsForge, o pídele a tu agente de código que genere una. Esta función crea una solicitud de firma a partir de una plantilla y la envía en una sola llamada a la API usando el endpoint `create-and-send`:

```typescript
import { createClient } from "npm:@insforge/sdk";

const FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api";

const corsHeaders = {
  "Access-Control-Allow-Origin": "*",
  "Access-Control-Allow-Headers": "authorization, content-type",
};

export default async function handler(req: Request) {
  if (req.method === "OPTIONS") {
    return new Response("ok", { headers: corsHeaders });
  }

  const insforge = createClient({
    baseUrl: Deno.env.get("INSFORGE_BASE_URL")!,
    anonKey: Deno.env.get("ANON_KEY")!,
  });

  const authHeader = req.headers.get("Authorization");
  if (!authHeader) {
    return new Response(JSON.stringify({ error: "Unauthorized" }), {
      status: 401,
      headers: { ...corsHeaders, "Content-Type": "application/json" },
    });
  }

  const token = authHeader.replace("Bearer ", "");
  const client = createClient({
    baseUrl: Deno.env.get("INSFORGE_BASE_URL")!,
    edgeFunctionToken: token,
  });

  const { data: userData } = await client.auth.getCurrentUser();
  if (!userData?.user) {
    return new Response(JSON.stringify({ error: "Unauthorized" }), {
      status: 401,
      headers: { ...corsHeaders, "Content-Type": "application/json" },
    });
  }

  const { template_id, signer_email, signer_first_name, signer_last_name } =
    await req.json();

  const apiKey = Deno.env.get("FIRMA_API_KEY")!;

  const response = await fetch(
    `${FIRMA_API}/signing-requests/create-and-send`,
    {
      method: "POST",
      headers: {
        Authorization: apiKey,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        template_id,
        recipients: [
          {
            first_name: signer_first_name,
            last_name: signer_last_name,
            email: signer_email,
            designation: "Signer",
            order: 1,
          },
        ],
      }),
    }
  );

  const data = await response.json();

  if (!response.ok) {
    return new Response(JSON.stringify({ error: data }), {
      status: response.status,
      headers: { ...corsHeaders, "Content-Type": "application/json" },
    });
  }

  return new Response(
    JSON.stringify({
      signing_request_id: data.id,
      first_signer_id: data.first_signer.id,
      signing_link: data.first_signer.signing_link,
      status: "sent",
    }),
    {
      status: 201,
      headers: { ...corsHeaders, "Content-Type": "application/json" },
    }
  );
}
```

<Note>
  El endpoint `create-and-send` crea la solicitud de firma y la envía a los destinatarios de forma atómica. Si necesitas revisar o modificar la solicitud antes de enviarla, usa `POST /signing-requests` para crear un borrador y luego `POST /signing-requests/{id}/send` por separado.
</Note>

La respuesta incluye `first_signer.id` (el `signing_request_user_id`) y `first_signer.signing_link`, que necesitarás si quieres integrar la experiencia de firma directamente en tu aplicación.

### Paso 3: llama a la función desde la UI de tu aplicación

Invoca la función de compute desde tu frontend usando el SDK de InsForge:

```typescript
import { insforge } from "@/lib/insforge";

const { data, error } = await insforge.functions.invoke(
  "sendSigningRequest",
  {
    body: {
      template_id: "your-template-id",
      signer_email: "alice@example.com",
      signer_first_name: "Alice",
      signer_last_name: "Johnson",
    },
  }
);
```

También puedes pedir a tu agente de código que lo conecte: "Cuando el usuario haga clic en Send Contract, llama a la función sendSigningRequest con el ID de la plantilla y los datos del firmante del formulario."

### Paso 3: da instrucciones a tu agente

Como InsForge está diseñado en torno a la programación agéntica, esta es la ruta más eficiente: tu agente lee los docs de Firma a través del servidor MCP, genera la función de compute y despacha la integración en una sola pasada.

Ejemplo de prompt:

```text
Create an InsForge compute function that sends a Firma signing request
using the create-and-send endpoint. Use the FIRMA_API_KEY secret for
authentication. Look up the Firma API docs via the MCP server for the
correct request and response format.
```

El agente usará el servidor MCP para consultar la URL correcta del endpoint, la forma del cuerpo de la petición y los campos de respuesta, y luego generará la función de compute con detalles precisos de la API.

## Integración de webhooks

Para hacer un seguimiento de cuándo se firman los documentos, configura un webhook de Firma que apunte a una función de compute de InsForge.

1. Crea una nueva función de compute para gestionar los eventos entrantes de webhook.
2. En el panel de Firma, en **Settings -> Webhooks**, registra un webhook que apunte a la URL de tu función: `https://<your-app>.functions.insforge.app/<function-name>`.
3. Selecciona los eventos que quieras recibir, como `signing_request.completed` y `signing_request.recipient.signed`.

```typescript
import { createClient } from "npm:@insforge/sdk";

export default async function handler(req: Request) {
  const insforge = createClient({
    baseUrl: Deno.env.get("INSFORGE_BASE_URL")!,
    anonKey: Deno.env.get("ANON_KEY")!,
  });

  const payload = await req.json();
  const { type, data } = payload;

  if (type === "signing_request.completed") {
    const signingRequestId = data.signing_request.id;

    await insforge.database
      .from("contracts")
      .update({
        status: "signed",
        signed_at: new Date().toISOString(),
      })
      .eq("firma_signing_request_id", signingRequestId);
  }

  return Response.json({ received: true });
}
```

Firma envía eventos para todos los cambios de estado clave. Consulta la [guía de webhooks](/guides/webhooks) para ver la lista completa de tipos de eventos y estructuras de carga útil.

<Warning>
  En producción, verifica la firma del webhook usando tu secreto de firma de webhook de Firma. Consulta la [guía de webhooks](/guides/webhooks) para conocer los detalles sobre la verificación de firmas HMAC.
</Warning>

## Firma integrada

Para aplicaciones donde los firmantes completan documentos directamente en tu UI, Firma proporciona una experiencia de firma integrable. La respuesta de `create-and-send` incluye un `first_signer.id` (el `signing_request_user_id`) y un `first_signer.signing_link` listo para usar. Carga la URL del firmante en un iframe dentro de tu aplicación de InsForge:

```html
<iframe
  src="https://app.firma.dev/signing/{signing_request_user_id}"
  style="width:100%;height:900px;border:0;"
  allow="camera;microphone;clipboard-write"
  title="Document Signing"
></iframe>
```

Consulta la [guía de firma integrada](/guides/embeddable-signing) para obtener instrucciones de configuración completas, incluyendo las mejores prácticas de seguridad y la gestión de eventos postMessage.
