Firma proporciona dos servidores MCP que permiten a los asistentes de IA trabajar con Firma de distintas formas:

- **MCP de datos** (`https://mcp.firma.dev/mcp`): da a los asistentes de IA acceso directo a la API de Firma. Gestiona plantillas, crea solicitudes de firma, configura espacios de trabajo y más mediante conversaciones en lenguaje natural. Requiere autenticación OAuth.
- **MCP de documentación** (`https://docs.firma.dev/mcp`): da a los asistentes de IA acceso a la documentación completa y a la referencia de la API de Firma. Úsalo para generar código de integración preciso con endpoints, formas de peticiones y patrones de autenticación correctos. No requiere autenticación.

La mayoría de los desarrolladores querrán tener ambos conectados: el MCP de datos para operar sobre los datos de Firma, y el MCP de documentación para construir integraciones con Firma.

## ¿Qué es MCP?

El [Model Context Protocol](https://modelcontextprotocol.io) (MCP) es un estándar abierto que conecta los asistentes de IA a herramientas y fuentes de datos externas. En lugar de escribir llamadas a la API manualmente, describes lo que necesitas en lenguaje natural, y el asistente de IA utiliza los servidores MCP de Firma para ejecutar las llamadas correctas a la API o consultar la documentación adecuada en tu nombre.

## Configuración

<Tabs>

<Tab title="MCP de datos">

El servidor MCP de datos se autentica mediante OAuth 2.1 con PKCE usando las credenciales de tu cuenta de Firma. Cuando te conectas a través de cualquier cliente admitido, el flujo OAuth se gestiona automáticamente.

**Clientes admitidos:**

- **Claude**: Claude Code, Claude Desktop, Claude.ai
- **ChatGPT**: a través de los conectores del modo desarrollador
- **Cursor**: soporte nativo de MCP
- **GitHub Copilot**: VS Code 1.101+
- **OpenAI Codex**: a través de la CLI o configuración
- **Otros clientes MCP**: cualquier cliente que implemente el [transporte Streamable HTTP](https://modelcontextprotocol.io/specification/2025-03-26/basic/transports#streamable-http)

<Tabs>

<Tab title="Claude Code">

Añade el servidor MCP de datos de Firma a través de la CLI:

```bash
claude mcp add --transport http firma-api https://mcp.firma.dev/mcp
```

O añádelo a tu configuración de Claude Code (`.claude/settings.json`):

```json
{
  "mcpServers": {
    "firma-api": {
      "type": "url",
      "url": "https://mcp.firma.dev/mcp"
    }
  }
}
```

Se te pedirá que inicies sesión con tu cuenta de Firma en el primer uso.

### Qué proporciona el MCP de documentación

El servidor MCP de documentación da a los asistentes de IA acceso a:

- **Referencia de la API**: URLs de endpoints, formas de peticiones/respuestas, detalles de autenticación
- **Guías de integración**: instrucciones paso a paso para cada plataforma admitida
- **Guías de funciones**: webhooks, firma integrada, marca blanca, plantillas de correo y más
- **Ejemplos de código**: ejemplos funcionales en Node.js, Python y otros lenguajes

### Cuándo usarlo

Referencia "los docs de Firma" en tus prompts para indicar al asistente que consulte este servidor:

```text
Using the Firma docs, create an Express route that sends a signing request
using the create-and-send endpoint. Use process.env.FIRMA_API_KEY for auth.
```

Sin esta referencia, tu asistente puede recurrir al conocimiento general y omitir detalles de endpoints o autenticación.

## Selección de espacio de trabajo

Tras iniciar sesión en el MCP de datos, empiezas sin un espacio de trabajo seleccionado. Puedes:

- **Pedir al asistente que seleccione un espacio de trabajo**: por ejemplo, "Cambia al espacio de trabajo Marketing"
- **Dejar que el asistente pase un ID de espacio de trabajo por petición**: el asistente puede incluir un parámetro `workspace_id` en las llamadas de herramientas individuales

El asistente normalmente listará primero tus espacios de trabajo y preguntará cuál usar.

## Ejemplos de conversaciones

### Listar todas las solicitudes de firma

> "Muéstrame todas las solicitudes de firma pendientes en mi espacio de trabajo Marketing"

El asistente seleccionará el espacio de trabajo, llamará al endpoint de listado de solicitudes de firma y presentará los resultados.

### Crear y enviar una solicitud de firma

> "Crea una solicitud de firma a partir de la plantilla de NDA y envíala a jane@example.com"

El asistente encontrará la plantilla, la duplicará en una solicitud de firma, establecerá el destinatario y la enviará.

### Comprobar el estado de firma

> "¿Cuál es el estado de la solicitud de firma que envié a John la semana pasada?"

El asistente buscará las solicitudes de firma recientes y devolverá el estado actual, incluyendo qué destinatarios han firmado.

### Generar código de integración

> "Using the Firma docs, create a Next.js API route that sends a signing request using the create-and-send endpoint"

El asistente consultará al MCP de documentación el endpoint correcto, la forma de la petición y el patrón de autenticación, y luego generará código funcional.

## Límites de tasa

Las peticiones MCP están sujetas a los mismos [límites de tasa](rate-limits) que las llamadas directas a la API. Cada llamada a una herramienta se corresponde con una petición a la API. El servidor MCP de documentación no tiene límites de tasa.

## Resolución de problemas

### Las herramientas no cargan

Si tu cliente muestra el servidor como conectado pero no aparecen herramientas:

- **Claude.ai / ChatGPT**: intenta desconectar y volver a conectar el conector desde Ajustes
- **Cursor / VS Code**: reinicia la aplicación tras añadir la configuración
- **Claude Desktop**: reinicia la aplicación; los cambios en `claude_desktop_config.json` requieren un reinicio

### Espacio de trabajo no encontrado

Asegúrate de haber seleccionado un espacio de trabajo tras conectar al MCP de datos. Usa "Lista mis espacios de trabajo" para ver las opciones disponibles.

### Errores de autenticación

Prueba a cerrar sesión y volver a iniciarla para refrescar tu sesión. Si el problema persiste, desconecta y vuelve a conectar el servidor en la configuración de tu cliente. El MCP de documentación no requiere autenticación.
