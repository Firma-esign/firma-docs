Firma ofrece dos servidores MCP que permiten a los asistentes de IA trabajar con Firma de distintas maneras:

- **Data MCP** (`https://mcp.firma.dev/mcp`) - Da a los asistentes de IA acceso directo a la API de Firma. Gestiona plantillas, crea solicitudes de firma, configura espacios de trabajo y más mediante conversaciones en lenguaje natural. Requiere autenticación OAuth.
- **Docs MCP** (`https://docs.firma.dev/mcp`) - Da a los asistentes de IA acceso a la documentación completa y la referencia de la API de Firma. Úsalo para generar código de integración preciso con los endpoints, formatos de solicitud y patrones de autenticación correctos. No requiere autenticación.

La mayoría de los desarrolladores quieren tener ambos conectados: el Data MCP para operar sobre los datos de Firma y el Docs MCP para construir integraciones con Firma.

## ¿Qué es MCP?

El [Model Context Protocol](https://modelcontextprotocol.io) (MCP) es un estándar abierto que conecta asistentes de IA a herramientas y fuentes de datos externas. En lugar de escribir llamadas a la API manualmente, describes lo que necesitas en lenguaje natural, y el asistente de IA usa los servidores MCP de Firma para ejecutar las llamadas a la API correctas o consultar la documentación adecuada en tu nombre.

## Configuración

<Tabs>

<Tab title="Data MCP">

El servidor Data MCP se autentica mediante OAuth 2.1 con PKCE usando las credenciales de tu cuenta de Firma. Cuando te conectas a través de cualquier cliente compatible, el flujo de OAuth se gestiona automáticamente.

**Clientes compatibles:**

- **Claude** - Claude Code, Claude Desktop, Claude.ai
- **ChatGPT** - A través del Modo Desarrollador para conectores
- **Cursor** - Soporte nativo de MCP
- **GitHub Copilot** - VS Code 1.101+
- **OpenAI Codex** - A través de la CLI o config
- **Otros clientes MCP** - Cualquier cliente que implemente el [transporte Streamable HTTP](https://modelcontextprotocol.io/specification/2025-03-26/basic/transports#streamable-http)

<Tabs>

<Tab title="Claude Code">

Agrega el servidor Data MCP de Firma mediante la CLI:

```bash
claude mcp add --transport http firma-api https://mcp.firma.dev/mcp
```

O añádelo a tu configuración de Claude Code (`.claude/settings.json`):

```json
{
  "mcpServers": {
    "firma-api": {
      "type": "url",
      "url": "https://mcp.firma.dev/mcp"
    }
  }
}
```

Se te pedirá que inicies sesión con tu cuenta de Firma la primera vez que lo uses.

### Qué ofrece el Docs MCP

El servidor Docs MCP da a los asistentes de IA acceso a:

- **Referencia de la API** - URLs de endpoints, formatos de solicitud/respuesta, detalles de autenticación
- **Guías de integración** - Instrucciones paso a paso para cada plataforma compatible
- **Guías de funciones** - Webhooks, firma incrustada, marca blanca, plantillas de correo y más
- **Ejemplos de código** - Ejemplos funcionales en Node.js, Python y otros lenguajes

### Cuándo usarlo

Menciona "los docs de Firma" en tus prompts para indicarle al asistente que consulte este servidor:

```text
Using the Firma docs, create an Express route that sends a signing request
using the create-and-send endpoint. Use process.env.FIRMA_API_KEY for auth.
```

Sin esta referencia, tu asistente puede recurrir a conocimiento general y pasar por alto detalles de endpoints o autenticación.

## Selección de espacio de trabajo

Después de iniciar sesión en el Data MCP, empiezas sin ningún espacio de trabajo seleccionado. Puedes:

- **Pedirle al asistente que seleccione un espacio de trabajo** - por ejemplo, "Cambia al espacio de trabajo de Marketing"
- **Dejar que el asistente pase un ID de espacio de trabajo por solicitud** - el asistente puede incluir un parámetro `workspace_id` en las llamadas a herramientas individuales

El asistente normalmente listará primero tus espacios de trabajo y te preguntará cuál usar.

## Ejemplos de conversación

### Listar todas las solicitudes de firma

> "Muéstrame todas las solicitudes de firma pendientes en mi espacio de trabajo de Marketing"

El asistente seleccionará el espacio de trabajo, llamará al endpoint de listado de solicitudes de firma y presentará los resultados.

### Crear y enviar una solicitud de firma

> "Crea una solicitud de firma a partir de la plantilla de NDA y envíala a jane@example.com"

El asistente encontrará la plantilla, la duplicará en una solicitud de firma, establecerá el destinatario y la enviará.

### Consultar el estado de la firma

> "¿Cuál es el estado de la solicitud de firma que le envié a John la semana pasada?"

El asistente buscará entre las solicitudes de firma recientes y devolverá el estado actual, incluyendo qué destinatarios ya firmaron.

### Generar código de integración

> "Usando los docs de Firma, crea una ruta de API en Next.js que envíe una solicitud de firma usando el endpoint create-and-send"

El asistente consultará el Docs MCP para obtener el endpoint, el formato de solicitud y el patrón de autenticación correctos, y luego generará código funcional.

## Límites de tasa

Las solicitudes MCP están sujetas a los mismos [límites de tasa](rate-limits) que las llamadas directas a la API. Cada llamada a herramienta corresponde a una solicitud de API. El servidor Docs MCP no tiene límites de tasa.

## Solución de problemas

### Las herramientas no cargan

Si tu cliente muestra el servidor como conectado pero no aparece ninguna herramienta:

- **Claude.ai / ChatGPT**: Intenta desconectar y volver a conectar el conector desde Configuración
- **Cursor / VS Code**: Reinicia la aplicación después de agregar la configuración
- **Claude Desktop**: Reinicia la app - los cambios en `claude_desktop_config.json` requieren reinicio

### Espacio de trabajo no encontrado

Asegúrate de haber seleccionado un espacio de trabajo después de conectarte al Data MCP. Usa "Lista mis espacios de trabajo" para ver las opciones disponibles.

### Errores de autenticación

Intenta cerrar sesión y volver a iniciarla para refrescar tu sesión. Si el problema persiste, desconecta y vuelve a conectar el servidor en la configuración de tu cliente. El Docs MCP no requiere autenticación.
