Chaque site Mintlify a besoin d'un fichier `docs.json` avec les paramètres de configuration principaux. Découvrez les [propriétés](#properties) ci-dessous.

## Propriétés

<ResponseField name="name" type="string" required>
Nom de votre projet. Utilisé pour le titre global.

Exemple : `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un tableau de groupes contenant toutes les pages de chaque groupe.
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    Le nom du groupe.

    Exemple : `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    Les chemins relatifs vers les fichiers markdown qui serviront de pages.

    Exemple : `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Chemin vers l'image du logo ou objet avec chemin vers les images du logo en mode « clair » et « sombre »
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Chemin du logo en mode clair
    </ResponseField>
    <ResponseField name="dark" type="string">
      Chemin du logo en mode sombre
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Où le clic sur le logo redirige l'utilisateur
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Chemin de l'image du favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Codes couleur hexadécimaux pour votre thème global
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      La couleur principale. Utilisée le plus souvent pour les contenus mis en avant,
      les en-têtes de section et les accents, en mode clair.
    </ResponseField>
    <ResponseField name="light" type="string">
      La couleur principale en mode sombre. Utilisée le plus souvent pour les
      contenus mis en avant, les en-têtes de section et les accents, en mode sombre.
    </ResponseField>
    <ResponseField name="dark" type="string">
      La couleur principale pour les boutons importants
    </ResponseField>
    <ResponseField name="background" type="object">
      La couleur de l'arrière-plan en modes clair et sombre
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          Le code couleur hexadécimal de l'arrière-plan en mode clair
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          Le code couleur hexadécimal de l'arrière-plan en mode sombre
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Tableau des `name` et `url` des liens que vous souhaitez inclure dans la barre supérieure
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    Le nom du bouton.

    Exemple : `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    L'URL vers laquelle vous êtes dirigé après avoir cliqué sur le bouton. Exemple : `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link affiche un bouton. GitHub affiche les informations du dépôt à l'URL indiquée, y compris le nombre d'étoiles GitHub.
    </ResponseField>
    <ResponseField name="url" type="string">
    Si `link` : la cible du bouton.
    
    Si `github` : lien vers le dépôt pour charger les informations GitHub.
    </ResponseField>
    <ResponseField name="name" type="string">
    Texte à l'intérieur du bouton. Requis uniquement si `type` vaut `link`.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Tableau des noms de versions. Utilisez-le uniquement si vous souhaitez afficher
  différentes versions de la documentation via un menu déroulant dans la barre de navigation.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Un tableau des ancres, incluant `icon`, `color` et `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    L'icône [Font Awesome](https://fontawesome.com/search?q=heart) utilisée pour mettre en avant l'ancre.

    Exemple : `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    Le nom du libellé de l'ancre.

    Exemple : `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      Le début de l'URL qui indique quelles pages appartiennent à l'ancre. Généralement, il s'agit du nom du dossier dans lequel vous placez vos pages.
    </ResponseField>
    <ResponseField name="color" type="string">
      La couleur hexadécimale de l'arrière-plan de l'icône de l'ancre. Peut aussi être un dégradé si vous passez un objet avec les propriétés `from` et `to`, chacune étant une couleur hexadécimale.
    </ResponseField>
    <ResponseField name="version" type="string">
      À utiliser si vous voulez masquer une ancre jusqu'à ce que la bonne version de la documentation soit sélectionnée.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Passez `true` si vous voulez masquer l'ancre jusqu'à ce que vous redirigiez directement quelqu'un vers la documentation qu'elle contient.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      L'une des valeurs suivantes : "brands", "duotone", "light", "sharp-solid", "solid" ou "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Redéfinit les configurations par défaut de l'ancre la plus en haut.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      Le nom de l'ancre la plus en haut
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Icône Font Awesome.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      L'une des valeurs suivantes : "brands", "duotone", "light", "sharp-solid", "solid" ou "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Un tableau des onglets de navigation.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      Le nom du libellé de l'onglet.
    </ResponseField>
    <ResponseField name="url" type="string">
      Le début de l'URL qui indique quelles pages appartiennent à l'onglet. Généralement,
      il s'agit du nom du dossier dans lequel vous placez vos pages.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuration des paramètres d'API. Découvrez-en plus sur les pages API dans [Composants d'API](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      L'URL de base pour tous les endpoints d'API. Si `baseUrl` est un tableau, cela active plusieurs
      options d'URL de base entre lesquelles l'utilisateur peut basculer.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          La stratégie d'authentification utilisée pour tous les endpoints d'API.
        </ResponseField>
        <ResponseField name="name" type="string">
        Le nom du paramètre d'authentification utilisé dans l'API playground.

        Si method vaut `basic`, le format doit être `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        La valeur par défaut destinée à être un préfixe pour le champ de saisie d'authentification.

        Ex. Un `inputPrefix` de `AuthKey` hériterait du résultat de saisie par défaut du champ d'authentification comme `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configurations pour l'API playground

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Si le playground est affiché, masqué ou uniquement en train d'afficher l'endpoint sans interactivité utilisateur supplémentaire `simple`

          Découvrez-en plus dans les [guides du playground](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      L'activation de ce drapeau garantit que l'ordre des clés dans les pages OpenAPI correspond à l'ordre des clés défini dans le fichier OpenAPI.

      <Warning>Ce comportement sera bientôt activé par défaut, moment auquel ce champ sera déprécié.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Une chaîne ou un tableau de chaînes contenant l'URL ou le chemin relatif vers
  votre fichier OpenAPI.
  
  Exemples :
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un objet des comptes de réseaux sociaux où la paire clé:propriété représente la plateforme de réseau social et l'URL du compte.
  
  Exemple : 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    L'une des valeurs suivantes : `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Exemple : `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    L'URL vers la plateforme sociale.
    
    Exemple : `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configurations pour activer les boutons de retour d'expérience

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Active un bouton permettant aux utilisateurs de suggérer des modifications via des pull requests
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Active un bouton permettant aux utilisateurs de signaler un problème concernant la documentation
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personnalisez le bouton de bascule du mode sombre.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      À définir si vous voulez toujours afficher le mode clair ou sombre pour les
      nouveaux utilisateurs. Lorsqu'il n'est pas défini, nous utilisons par défaut le
      même mode que celui du système d'exploitation de l'utilisateur.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Définissez à true pour masquer le bouton de bascule mode clair/sombre. Vous pouvez combiner `isHidden` avec `default` pour forcer votre documentation à n'utiliser que le mode clair ou sombre. Par exemple :
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Une image d'arrière-plan à afficher derrière chaque page. Voir des exemples avec
  [Infisical](https://infisical.com/docs) et [FRPC](https://frpc.io).
</ResponseField>
