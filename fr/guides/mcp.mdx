Firma fournit deux serveurs MCP qui permettent aux assistants IA de travailler avec Firma de différentes manières :

- **Data MCP** (`https://mcp.firma.dev/mcp`) - Donne aux assistants IA un accès direct à l'API Firma. Gérez des modèles, créez des demandes de signature, configurez des espaces de travail et plus encore via des conversations en langage naturel. Nécessite une authentification OAuth.
- **Docs MCP** (`https://docs.firma.dev/mcp`) - Donne aux assistants IA l'accès à la documentation complète et à la référence de l'API de Firma. Utilisez-le pour générer du code d'intégration précis avec des endpoints, formes de requête et motifs d'authentification corrects. Aucune authentification requise.

La plupart des développeurs veulent que les deux soient connectés : le Data MCP pour opérer sur les données Firma, et le Docs MCP pour construire les intégrations Firma.

## Qu'est-ce que MCP ?

Le [Model Context Protocol](https://modelcontextprotocol.io) (MCP) est un standard ouvert qui connecte les assistants IA à des outils et sources de données externes. Au lieu d'écrire manuellement des appels d'API, vous décrivez ce dont vous avez besoin en langage naturel, et l'assistant IA utilise les serveurs MCP de Firma pour exécuter les bons appels d'API ou consulter la bonne documentation en votre nom.

## Configuration

<Tabs>

<Tab title="Data MCP">

Le serveur Data MCP s'authentifie via OAuth 2.1 avec PKCE en utilisant les identifiants de votre compte Firma. Lorsque vous vous connectez via un client pris en charge, le flux OAuth est géré automatiquement.

**Clients pris en charge :**

- **Claude** - Claude Code, Claude Desktop, Claude.ai
- **ChatGPT** - Via les connecteurs du mode développeur
- **Cursor** - Support MCP natif
- **GitHub Copilot** - VS Code 1.101+
- **OpenAI Codex** - Via CLI ou config
- **Autres clients MCP** - Tout client implémentant le [transport Streamable HTTP](https://modelcontextprotocol.io/specification/2025-03-26/basic/transports#streamable-http)

<Tabs>

<Tab title="Claude Code">

Ajoutez le serveur Firma Data MCP via la CLI :

```bash
claude mcp add --transport http firma-api https://mcp.firma.dev/mcp
```

Ou ajoutez-le à votre configuration Claude Code (`.claude/settings.json`) :

```json
{
  "mcpServers": {
    "firma-api": {
      "type": "url",
      "url": "https://mcp.firma.dev/mcp"
    }
  }
}
```

Vous serez invité à vous connecter avec votre compte Firma lors de la première utilisation.

### Ce que le Docs MCP fournit

Le serveur Docs MCP donne aux assistants IA l'accès à :

- **Référence de l'API** - URL d'endpoint, formes de requête/réponse, détails d'authentification
- **Guides d'intégration** - Instructions pas à pas pour chaque plateforme prise en charge
- **Guides de fonctionnalités** - Webhooks, signature intégrée, white labeling, modèles d'e-mail et plus
- **Exemples de code** - Exemples fonctionnels en Node.js, Python et autres langages

### Quand l'utiliser

Référencez « the Firma docs » dans vos prompts pour indiquer à l'assistant d'interroger ce serveur :

```text
Using the Firma docs, create an Express route that sends a signing request
using the create-and-send endpoint. Use process.env.FIRMA_API_KEY for auth.
```

Sans cette référence, votre assistant peut se rabattre sur des connaissances générales et manquer des détails d'endpoint ou d'authentification.

## Sélection d'espace de travail

Après vous être connecté au Data MCP, vous commencez sans espace de travail sélectionné. Vous pouvez soit :

- **Demander à l'assistant de sélectionner un espace de travail** - par ex. « Passe à l'espace de travail Marketing »
- **Laisser l'assistant passer un ID d'espace de travail par requête** - l'assistant peut inclure un paramètre `workspace_id` sur les appels d'outil individuels

L'assistant listera généralement d'abord vos espaces de travail et demandera lequel utiliser.

## Exemples de conversations

### Lister toutes les demandes de signature

> « Montre-moi toutes les demandes de signature en attente dans mon espace de travail Marketing »

L'assistant sélectionnera l'espace de travail, appellera l'endpoint de liste des demandes de signature et présentera les résultats.

### Créer et envoyer une demande de signature

> « Crée une demande de signature à partir du modèle NDA et envoie-la à jane@example.com »

L'assistant trouvera le modèle, le dupliquera en une demande de signature, définira le destinataire et l'enverra.

### Vérifier le statut de signature

> « Quel est le statut de la demande de signature que j'ai envoyée à John la semaine dernière ? »

L'assistant recherchera les demandes de signature récentes et retournera le statut actuel, y compris quels destinataires ont signé.

### Générer du code d'intégration

> « Using the Firma docs, crée une route API Next.js qui envoie une demande de signature en utilisant l'endpoint create-and-send »

L'assistant interrogera le Docs MCP pour l'endpoint correct, la forme de requête et le motif d'authentification, puis générera du code fonctionnel.

## Limites de débit

Les requêtes MCP sont soumises aux mêmes [limites de débit](rate-limits) que les appels d'API directs. Chaque appel d'outil correspond à une requête d'API. Le serveur Docs MCP n'a pas de limites de débit.

## Dépannage

### Les outils ne se chargent pas

Si votre client affiche le serveur comme connecté mais qu'aucun outil n'apparaît :

- **Claude.ai / ChatGPT** : essayez de déconnecter et reconnecter le connecteur depuis les paramètres
- **Cursor / VS Code** : redémarrez l'application après avoir ajouté la configuration
- **Claude Desktop** : redémarrez l'application - les modifications de `claude_desktop_config.json` nécessitent un redémarrage

### Espace de travail introuvable

Assurez-vous d'avoir sélectionné un espace de travail après vous être connecté au Data MCP. Utilisez « Liste mes espaces de travail » pour voir les options disponibles.

### Erreurs d'authentification

Essayez de vous déconnecter et de vous reconnecter pour rafraîchir votre session. Si le problème persiste, déconnectez et reconnectez le serveur dans les paramètres de votre client. Le Docs MCP ne nécessite pas d'authentification.
