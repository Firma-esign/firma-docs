Ogni sito Mintlify ha bisogno di un file `docs.json` con le impostazioni di configurazione principali. Scopri di più sulle [proprietà](#properties) qui sotto.

## Proprietà

<ResponseField name="name" type="string" required>
Nome del tuo progetto. Utilizzato per il titolo globale.

Esempio: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un array di gruppi con tutte le pagine all'interno di ciascun gruppo
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    Il nome del gruppo.

    Esempio: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    I percorsi relativi ai file markdown che serviranno come pagine.

    Esempio: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Percorso all'immagine del logo o oggetto con il percorso alle immagini del logo in modalità "light" e "dark"
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Percorso al logo in modalità chiara
    </ResponseField>
    <ResponseField name="dark" type="string">
      Percorso al logo in modalità scura
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Dove ti porta il clic sul logo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Percorso all'immagine della favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Codici colore esadecimali per il tuo tema globale
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      Il colore primario. Usato più spesso per contenuti evidenziati, intestazioni di sezione
      e accenti, in modalità chiara
    </ResponseField>
    <ResponseField name="light" type="string">
      Il colore primario per la modalità scura. Usato più spesso per contenuti evidenziati,
      intestazioni di sezione e accenti, in modalità scura
    </ResponseField>
    <ResponseField name="dark" type="string">
      Il colore primario per i pulsanti importanti
    </ResponseField>
    <ResponseField name="background" type="object">
      Il colore dello sfondo sia in modalità chiara che scura
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          Il codice colore esadecimale dello sfondo in modalità chiara
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          Il codice colore esadecimale dello sfondo in modalità scura
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array di `name` e `url` dei link che vuoi includere nella barra superiore
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    Il nome del pulsante.

    Esempio: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    L'URL a cui punta il pulsante. Esempio: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link mostra un pulsante. GitHub mostra le informazioni del repository all'URL fornito, incluso il numero di stelle GitHub.
    </ResponseField>
    <ResponseField name="url" type="string">
    Se `link`: dove punta il pulsante.
    
    Se `github`: link al repository da cui caricare le informazioni di GitHub.
    </ResponseField>
    <ResponseField name="name" type="string">
    Testo all'interno del pulsante. Obbligatorio solo se `type` è `link`.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array di nomi di versione. Usalo solo se vuoi mostrare versioni diverse
  della documentazione tramite un menu a discesa nella barra di navigazione.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Un array delle ancore, include `icon`, `color` e `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    L'icona [Font Awesome](https://fontawesome.com/search?q=heart) usata per caratterizzare l'ancora.

    Esempio: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    Il nome dell'etichetta dell'ancora.

    Esempio: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      L'inizio dell'URL che indica quali pagine finiscono nell'ancora. Generalmente è il nome della cartella in cui hai messo le tue pagine.
    </ResponseField>
    <ResponseField name="color" type="string">
      Il colore esadecimale dello sfondo dell'icona dell'ancora. Può anche essere un gradiente se passi un oggetto con le proprietà `from` e `to`, ognuna delle quali è un colore esadecimale.
    </ResponseField>
    <ResponseField name="version" type="string">
      Usato se vuoi nascondere un'ancora finché non è selezionata la versione corretta della documentazione.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Imposta `true` se vuoi nascondere l'ancora finché non colleghi direttamente qualcuno alla documentazione al suo interno.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Uno tra: "brands", "duotone", "light", "sharp-solid", "solid" o "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Sovrascrivi le configurazioni predefinite per l'ancora principale in alto.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      Il nome dell'ancora principale in alto
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Icona Font Awesome.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Uno tra: "brands", "duotone", "light", "sharp-solid", "solid" o "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Un array di tab di navigazione.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      Il nome dell'etichetta della tab.
    </ResponseField>
    <ResponseField name="url" type="string">
      L'inizio dell'URL che indica quali pagine finiscono nella tab. Generalmente
      è il nome della cartella in cui hai messo le tue pagine.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configurazione per le impostazioni dell'API. Scopri di più sulle pagine API in [API Components](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      L'URL di base per tutti gli endpoint API. Se `baseUrl` è un array, saranno abilitate più opzioni di URL di base
      tra cui l'utente potrà scegliere.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          La strategia di autenticazione utilizzata per tutti gli endpoint API.
        </ResponseField>
        <ResponseField name="name" type="string">
        Il nome del parametro di autenticazione utilizzato nell'API playground.

        Se il metodo è `basic`, il formato deve essere `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        Il valore predefinito pensato come prefisso per il campo di input dell'autenticazione.

        Es.: un `inputPrefix` di `AuthKey` produrrà nel campo di autenticazione il valore predefinito `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configurazioni per l'API playground

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Se il playground viene mostrato, nascosto o mostrato solo con l'endpoint senza alcuna interattività aggiuntiva `simple`

          Scopri di più nelle [guide al playground](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      L'abilitazione di questo flag garantisce che l'ordine delle chiavi nelle pagine OpenAPI corrisponda a quello definito nel file OpenAPI.

      <Warning>Questo comportamento sarà presto abilitato per impostazione predefinita, momento in cui questo campo verrà deprecato.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Una stringa o un array di stringhe di URL o percorsi relativi che puntano al
  tuo file OpenAPI.
  
  Esempi:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un oggetto di account di social media in cui la coppia chiave:proprietà rappresenta la piattaforma social e l'URL dell'account.
  
  Esempio: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Uno dei seguenti valori `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Esempio: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    L'URL della piattaforma social.
    
    Esempio: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configurazioni per abilitare i pulsanti di feedback

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Abilita un pulsante che consente agli utenti di suggerire modifiche tramite pull request
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Abilita un pulsante che consente agli utenti di segnalare un problema relativo alla documentazione
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personalizza il toggle della modalità scura.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Imposta se vuoi mostrare sempre la modalità chiara o scura ai nuovi utenti. Se non è
      impostato, di default utilizziamo la stessa modalità del sistema operativo dell'utente.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Imposta su true per nascondere il toggle chiaro/scuro. Puoi combinare `isHidden` con `default` per forzare la tua documentazione a usare solo la modalità chiara o scura. Ad esempio:
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Un'immagine di sfondo da mostrare dietro ogni pagina. Vedi l'esempio con
  [Infisical](https://infisical.com/docs) e [FRPC](https://frpc.io).
</ResponseField>
