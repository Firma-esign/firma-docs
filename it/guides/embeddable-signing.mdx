# Firma incorporabile

Incorpora il flusso di firma direttamente nella tua applicazione in modo che i destinatari possano firmare i documenti senza lasciare il tuo prodotto. Questo crea un'esperienza di firma continua e white-label.

## Pattern dell'URL di firma

L'interfaccia di firma è disponibile all'indirizzo:

`https://app.firma.dev/signing/{signing_request_user_id}`

## Incorporamento base tramite iframe

```html
<iframe 
  src="https://app.firma.dev/signing/{signing_request_user_id}" 
  style="width:100%;height:900px;border:0;" 
  allow="camera;microphone;clipboard-write"
  title="Document Signing"
></iframe>
```

### Permessi iframe richiesti

- `camera` - Per la verifica dell'identità (se abilitata)
- `microphone` - Per la verifica video (se abilitata)
- `clipboard-write` - Per copiare/incollare contenuti

## Ottenere l'URL di firma

Il `signing_request_user_id` viene restituito quando recuperi gli utenti della richiesta di firma tramite API.

### Esempio: ottenere gli URL di firma dei destinatari

```js
const response = await fetch(
  `https://api.firma.dev/functions/v1/signing-request-api/signing-requests/${signingRequestId}/users`,
  {
    headers: {
      'Authorization': `Bearer ${API_KEY}`
    }
  }
)

const users = await response.json()

// Ogni utente ha il proprio URL di firma
users.forEach(user => {
  const signingUrl = `https://app.firma.dev/signing/${user.id}`
  console.log(`${user.email}: ${signingUrl}`)
})
```

## Esempio di implementazione completa

### Componente React

```jsx
import { useState, useEffect } from 'react'

function SigningView({ signingRequestId, recipientEmail }) {
  const [signingUrl, setSigningUrl] = useState(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)
  
  useEffect(() => {
    async function loadSigningUrl() {
      try {
        // Recupera gli utenti della richiesta di firma dal tuo backend
        const response = await fetch(
          `/api/signing-requests/${signingRequestId}/users`,
          { credentials: 'include' }
        )
        
        if (!response.ok) {
          throw new Error('Failed to load signing users')
        }
        
        const users = await response.json()
        
        // Trova l'utente per email
        const user = users.find(
          u => u.email === recipientEmail
        )
        
        if (!user) {
          throw new Error('User not found')
        }
        
        const url = `https://app.firma.dev/signing/${user.id}`
        setSigningUrl(url)
        setLoading(false)
      } catch (err) {
        setError(err.message)
        setLoading(false)
      }
    }
    
    loadSigningUrl()
  }, [signingRequestId, recipientEmail])
  
  if (loading) return <div>Loading signing interface...</div>
  if (error) return <div>Error: {error}</div>
  
  return (
    <iframe
      src={signingUrl}
      style={{
        width: '100%',
        height: '900px',
        border: 0
      }}
      allow="camera;microphone;clipboard-write"
      title="Sign Document"
    />
  )
}
```

### Endpoint lato server (Node.js)

```js
import express from 'express'
import fetch from 'node-fetch'

const app = express()
const API_KEY = process.env.FIRMA_API_KEY

app.get('/api/signing-requests/:id/users', async (req, res) => {
  try {
    const response = await fetch(
      `https://api.firma.dev/functions/v1/signing-request-api/signing-requests/${req.params.id}/users`,
      {
        headers: {
          'Authorization': `Bearer ${API_KEY}`
        }
      }
    )
    
    if (!response.ok) {
      const error = await response.text()
      return res.status(response.status).json({ error })
    }
    
    const users = await response.json()
    res.json(users)
  } catch (error) {
    console.error('Failed to fetch signing users:', error)
    res.status(500).json({ error: 'Internal server error' })
  }
})

app.listen(3000)
```

## Eventi postMessage

L'iframe di firma emette eventi postMessage per tracciare il progresso della firma:

```js
window.addEventListener('message', (event) => {
  // Convalida l'origine
  if (event.origin !== 'https://app.firma.dev') return
  
  const data = event.data
  
  switch (data.type) {
    case 'signing.started':
      console.log('User started signing')
      break
    case 'signing.completed':
      console.log('Document signed successfully')
      // Reindirizza o mostra un messaggio di successo
      break
    case 'signing.declined':
      console.log('User declined to sign')
      break
    case 'signing.error':
      console.error('Signing error:', data.error)
      break
  }
})
```

## Best practice di sicurezza

<Warning>
  **Non esporre mai chiavi API nel codice frontend.** Recupera sempre gli ID utente di firma tramite un endpoint backend sicuro.
</Warning>

### ✅ Cosa fare

- ✅ Recupera gli ID utente di firma tramite il tuo backend
- ✅ Convalida le origini dei postMessage (`https://app.firma.dev`)
- ✅ Usa HTTPS per tutte le richieste API
- ✅ Monitora gli eventi di firma tramite webhook (60 req/min)

### ❌ Cosa non fare

- ❌ Non esporre chiavi API nel codice client
- ❌ Non fidarti dei dati postMessage senza convalida dell'origine

## Integrazione dei webhook

Usa i webhook per tracciare gli eventi di firma in tempo reale invece di fare polling:

```js
// Gestore webhook
app.post('/webhooks/firma', async (req, res) => {
  const event = req.body
  
  switch (event.event_type) {
    case 'signing_request.viewed':
      await notifyUser(event.data.signing_request_id, 'viewed')
      break
    case 'signing_request.recipient.signed':
      await notifyUser(event.data.signing_request_id, 'signed')
      break
    case 'signing_request.completed':
      await markComplete(event.data.signing_request_id)
      break
  }
  
  res.json({ received: true })
})
```

Consulta la [guida ai webhook](webhooks) per i dettagli completi di implementazione.

## Risoluzione dei problemi

### L'iframe non si carica

**Cause possibili**:

- `signing_request_user_id` non valido
- Il destinatario ha già completato la firma
- La richiesta di firma è stata annullata o è scaduta

**Soluzione**: verifica lo stato della richiesta di firma tramite API

### Gli eventi postMessage non vengono ricevuti

**Cause possibili**:

- La convalida dell'origine blocca i messaggi
- Il listener degli eventi non è collegato prima che l'iframe si carichi

**Soluzione**:

- Verifica che l'origine sia esattamente `https://app.firma.dev`
- Collega il listener prima di creare l'iframe

### 403 Forbidden quando si recupera la richiesta di firma

**Cause possibili**:

- L'utente non ha accesso alla richiesta di firma
- La chiave API non ha i permessi richiesti

**Soluzione**: verifica la logica di autorizzazione e i permessi della chiave API
