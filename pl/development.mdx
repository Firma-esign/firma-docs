<Info>
  **Wymagania wstępne**:
  - Node.js w wersji 19 lub wyższej
  - Repozytorium dokumentacji z plikiem `docs.json`
</Info>

Wykonaj poniższe kroki, aby zainstalować i uruchomić Mintlify w swoim systemie operacyjnym.

<Steps>
<Step title="Zainstaluj Mintlify CLI">

```bash
npm i -g mint
```
</Step>

<Step title="Podgląd lokalny">

Przejdź do katalogu z dokumentacją, w którym znajduje się plik `docs.json`, i uruchom następujące polecenie:

```bash
mint dev
```

Lokalny podgląd dokumentacji będzie dostępny pod adresem `http://localhost:3000`.

</Step>
</Steps>

## Niestandardowe porty

Domyślnie Mintlify używa portu 3000. Możesz dostosować port, na którym działa Mintlify, używając flagi `--port`. Na przykład, aby uruchomić Mintlify na porcie 3333, użyj tego polecenia:

```bash
mint dev --port 3333
```

Jeśli spróbujesz uruchomić Mintlify na porcie, który jest już zajęty, zostanie użyty kolejny dostępny port:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Wersje Mintlify

Pamiętaj, że każde wydanie CLI jest powiązane z konkretną wersją Mintlify. Jeśli lokalny podgląd nie odpowiada wersji produkcyjnej, zaktualizuj CLI:

```bash
npm mint update
```

## Weryfikacja linków

CLI może pomóc w weryfikacji linków w Twojej dokumentacji. Aby zidentyfikować uszkodzone linki, użyj następującego polecenia:

```bash
mint broken-links
```

Zalecamy używanie rozszerzeń w Twoim IDE do rozpoznawania i formatowania MDX. Jeśli korzystasz z VSCode, rozważ [rozszerzenie MDX VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) do podświetlania składni oraz [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) do formatowania kodu.

## Rozwiązywanie problemów

<AccordionGroup>
  <Accordion title='Błąd: Could not load the "sharp" module using the darwin-arm64 runtime'>

    Może to być spowodowane nieaktualną wersją node. Spróbuj następujących kroków:
    1. Usuń aktualnie zainstalowaną wersję CLI: `npm remove -g mint`
    2. Zaktualizuj Node do wersji v19 lub wyższej.
    3. Zainstaluj ponownie CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Problem: Napotkano nieznany błąd">
  
    Rozwiązanie: Przejdź do katalogu głównego swojego urządzenia i usuń folder `~/.mintlify`. Następnie uruchom ponownie `mint dev`.
  </Accordion>
</AccordionGroup>

Ciekawi Cię, co zmieniło się w najnowszej wersji CLI? Sprawdź [dziennik zmian CLI](https://www.npmjs.com/package/mintlify?activeTab=versions).
