Menu nawigacji to lista linków widoczna na każdej stronie serwisu.

Prawdopodobnie będziesz aktualizować `docs.json` za każdym razem, gdy dodasz nową stronę. Strony nie pojawiają się automatycznie.

## Składnia nawigacji

Nasza składnia nawigacji jest rekurencyjna, co oznacza, że możesz tworzyć zagnieżdżone grupy nawigacji. Nie musisz uwzględniać `.mdx` w nazwach stron.

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": [
            "quickstart",
            {
              "group": "Nested Reference Pages",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Foldery

Po prostu umieść pliki MDX w folderach i zaktualizuj ścieżki w `docs.json`.

Na przykład, aby mieć stronę pod adresem `https://yoursite.com/your-folder/your-page`, utwórz folder o nazwie `your-folder` zawierający plik MDX o nazwie `your-page.mdx`.

<Warning>

Nie możesz użyć `api` jako nazwy folderu, chyba że zagnieździsz go w innym folderze. Mintlify używa Next.js, który rezerwuje folder `api` na najwyższym poziomie dla wewnętrznych wywołań serwerowych. Nazwa folderu taka jak `api-reference` zostanie zaakceptowana.

</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## Ukryte strony

Pliki MDX niedołączone do `docs.json` nie pojawią się na pasku bocznym, ale są dostępne za pomocą paska wyszukiwania i przez bezpośrednie linkowanie.
