Każda witryna Mintlify wymaga pliku `docs.json` z podstawowymi ustawieniami konfiguracji. Dowiedz się więcej o [właściwościach](#properties) poniżej.

## Właściwości

<ResponseField name="name" type="string" required>
Nazwa Twojego projektu. Używana jako tytuł globalny.

Przykład: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Tablica grup ze wszystkimi stronami w danej grupie
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    Nazwa grupy.

    Przykład: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    Względne ścieżki do plików markdown, które będą pełnić rolę stron.

    Przykład: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Ścieżka do obrazu logo lub obiekt ze ścieżką do logo dla trybu „light” i „dark”
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Ścieżka do logo w trybie jasnym
    </ResponseField>
    <ResponseField name="dark" type="string">
      Ścieżka do logo w trybie ciemnym
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Miejsce docelowe, do którego prowadzi kliknięcie w logo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Ścieżka do obrazu favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Kody kolorów heksadecymalne dla globalnego motywu
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      Kolor podstawowy. Używany najczęściej dla wyróżnionej treści, nagłówków
      sekcji, akcentów, w trybie jasnym
    </ResponseField>
    <ResponseField name="light" type="string">
      Kolor podstawowy dla trybu ciemnego. Używany najczęściej dla wyróżnionej
      treści, nagłówków sekcji, akcentów, w trybie ciemnym
    </ResponseField>
    <ResponseField name="dark" type="string">
      Kolor podstawowy dla ważnych przycisków
    </ResponseField>
    <ResponseField name="background" type="object">
      Kolor tła zarówno w trybie jasnym, jak i ciemnym
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          Kod heksadecymalny koloru tła w trybie jasnym
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          Kod heksadecymalny koloru tła w trybie ciemnym
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Tablica pól `name` i `url` linków, które chcesz umieścić w górnym pasku
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    Nazwa przycisku.

    Przykład: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    Adres URL, do którego prowadzi kliknięcie przycisku. Przykład: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link wyświetla przycisk. GitHub wyświetla informacje o repozytorium spod podanego adresu URL, w tym liczbę gwiazdek na GitHub.
    </ResponseField>
    <ResponseField name="url" type="string">
    Jeśli `link`: cel, do którego prowadzi przycisk.
    
    Jeśli `github`: link do repozytorium, z którego mają być załadowane informacje z GitHub.
    </ResponseField>
    <ResponseField name="name" type="string">
    Tekst wewnątrz przycisku. Wymagane tylko, jeśli `type` to `link`.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Tablica nazw wersji. Używaj tego tylko wtedy, gdy chcesz pokazać różne wersje
  dokumentacji z listy rozwijanej na pasku nawigacji.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Tablica kotwic, zawiera pola `icon`, `color` i `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    Ikona [Font Awesome](https://fontawesome.com/search?q=heart) używana do wyróżnienia kotwicy.

    Przykład: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    Nazwa etykiety kotwicy.

    Przykład: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      Początek adresu URL wskazujący, które strony trafiają do kotwicy. Zazwyczaj jest to nazwa folderu, w którym umieściłeś strony.
    </ResponseField>
    <ResponseField name="color" type="string">
      Heksadecymalny kolor tła ikony kotwicy. Może być również gradientem, jeśli przekażesz obiekt z właściwościami `from` i `to`, z których każda jest kolorem heksadecymalnym.
    </ResponseField>
    <ResponseField name="version" type="string">
      Używane, jeśli chcesz ukryć kotwicę, dopóki nie zostanie wybrana odpowiednia wersja dokumentacji.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Przekaż `true`, jeśli chcesz ukryć kotwicę, dopóki nie skierujesz kogoś bezpośrednio do dokumentacji w niej zawartej.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Jedno z: "brands", "duotone", "light", "sharp-solid", "solid" lub "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Nadpisz domyślne konfiguracje dla najwyżej położonej kotwicy.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      Nazwa najwyższej kotwicy
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Ikona Font Awesome.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Jedno z: "brands", "duotone", "light", "sharp-solid", "solid" lub "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Tablica zakładek nawigacyjnych.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      Nazwa etykiety zakładki.
    </ResponseField>
    <ResponseField name="url" type="string">
      Początek adresu URL wskazujący, które strony trafiają do zakładki. Zazwyczaj
      jest to nazwa folderu, w którym umieściłeś strony.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Konfiguracja ustawień API. Dowiedz się więcej o stronach API w [Komponentach API](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      Bazowy adres URL dla wszystkich endpointów API. Jeśli `baseUrl` jest tablicą, umożliwi wiele opcji
      bazowych adresów URL, między którymi użytkownik może się przełączać.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          Strategia uwierzytelniania używana dla wszystkich endpointów API.
        </ResponseField>
        <ResponseField name="name" type="string">
        Nazwa parametru uwierzytelniania używanego w API playground.

        Jeśli metoda to `basic`, format powinien być następujący: `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        Domyślna wartość, która ma być prefiksem dla pola wejściowego uwierzytelniania.

        Np. `inputPrefix` `AuthKey` sprawi, że domyślnym wynikiem pola uwierzytelniania będzie `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Konfiguracje dla API playground

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Czy playground jest widoczny, ukryty, czy tylko wyświetla endpoint bez dodatkowej interaktywności użytkownika `simple`

          Dowiedz się więcej w [przewodnikach po playground](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Włączenie tej flagi zapewnia, że kolejność kluczy na stronach OpenAPI odpowiada kolejności kluczy zdefiniowanej w pliku OpenAPI.

      <Warning>To zachowanie zostanie wkrótce włączone domyślnie, w tym momencie to pole zostanie oznaczone jako przestarzałe.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Ciąg znaków lub tablica ciągów znaków z adresem(-ami) URL lub ścieżką(-ami) względną(-ymi) wskazującą(-ymi) na Twój
  plik OpenAPI.
  
  Przykłady:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Obiekt kont mediów społecznościowych, w którym para klucz:właściwość reprezentuje platformę mediów społecznościowych i adres URL konta.
  
  Przykład: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Jedna z następujących wartości: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Przykład: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    Adres URL do platformy społecznościowej.
    
    Przykład: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Konfiguracje włączające przyciski opinii

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Włącza przycisk umożliwiający użytkownikom sugerowanie zmian za pomocą pull requestów
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Włącza przycisk umożliwiający użytkownikom zgłaszanie problemów dotyczących dokumentacji
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Dostosuj przełącznik trybu ciemnego.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Ustaw, jeśli zawsze chcesz pokazywać nowym użytkownikom tryb jasny lub ciemny. Jeśli nie zostanie
      ustawiony, domyślnie używamy tego samego trybu, co system operacyjny użytkownika.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Ustaw na true, aby ukryć przełącznik trybu ciemnego/jasnego. Możesz połączyć `isHidden` z `default`, aby wymusić w dokumentacji użycie tylko trybu jasnego lub tylko ciemnego. Na przykład:
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Obraz tła wyświetlany za każdą stroną. Zobacz przykład z
  [Infisical](https://infisical.com/docs) i [FRPC](https://frpc.io).
</ResponseField>
