Firma pozwala dodać prawnie wiążące podpisy elektroniczne do dowolnej aplikacji działającej na Cloudflare. Wywołuj API Firma z Workers lub Pages Functions, aby tworzyć szablony, wysyłać żądania podpisu i śledzić zakończenia za pomocą webhooków.

Ten przewodnik obejmuje dwie ścieżki integracji:

1. **Workers** — Samodzielne funkcje serverless, które wywołują REST API Firma. Najlepsze dla API, mikroserwisów lub aplikacji, które nie używają Cloudflare Pages.
2. **Pages Functions** — Funkcje serverless oparte na plikach, które są wdrażane wraz z Twoją stroną Cloudflare Pages. Najlepsze, jeśli masz już projekt Pages i chcesz mieć wszystko w jednym repozytorium.

Obie ścieżki używają tych samych endpointów API Firma. Różnica polega na sposobie strukturyzacji i wdrażania kodu.

## Wymagania wstępne

- [Konto Firma](https://app.firma.dev) z kluczem API
- Zainstalowany [CLI Wrangler](https://developers.cloudflare.com/workers/wrangler/install-and-update/) (`npm install -g wrangler`)
- Co najmniej jeden szablon Firma ze skonfigurowanymi polami podpisu

<Note>
  Firma używa surowego klucza API jako wartości nagłówka `Authorization`. Nie poprzedzaj go prefiksem `Bearer`. Różni się to od wielu innych API.
</Note>

### Krok 1: Utwórz projekt Workers

Jeśli nie masz jeszcze projektu Workers, utwórz jego szkielet:

```bash
npm create cloudflare@latest -- firma-signing
```

Wybierz **„Hello World" Worker**, gdy pojawi się monit, a następnie wybierz JavaScript lub TypeScript. Po zakończeniu tworzenia szkieletu:

```bash
cd firma-signing
```

### Krok 2: Zapisz klucz API jako sekret

Dodaj swój klucz API Firma jako zaszyfrowany sekret. Wrangler poprosi Cię o wklejenie wartości:

```bash
wrangler secret put FIRMA_API_KEY
```

Nigdy nie umieszczaj klucza API w plikach źródłowych. Sekrety są szyfrowane w spoczynku i wstrzykiwane do bindingu `env` Workera w czasie wykonywania.

### Krok 3: Utwórz Worker do wysyłania żądań podpisu

Zastąp zawartość pliku `src/index.js` (lub `src/index.ts`, jeśli wybrałeś TypeScript) poniższym kodem. Ten Worker przyjmuje żądanie POST z danymi podpisującego, wywołuje endpoint Firma `create-and-send` i zwraca ID żądania podpisu.

```javascript
const FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api";

export default {
  async fetch(request, env) {
    if (request.method === "OPTIONS") {
      return new Response(null, {
        headers: {
          "Access-Control-Allow-Origin": "*",
          "Access-Control-Allow-Methods": "POST, OPTIONS",
          "Access-Control-Allow-Headers": "Content-Type",
        },
      });
    }

    if (request.method !== "POST") {
      return Response.json({ error: "Method not allowed" }, { status: 405 });
    }

    const { name, template_id, signer_email, signer_first_name, signer_last_name } =
      await request.json();

    const response = await fetch(
      `${FIRMA_API}/signing-requests/create-and-send`,
      {
        method: "POST",
        headers: {
          Authorization: env.FIRMA_API_KEY,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({
          name,
          template_id,
          recipients: [
            {
              first_name: signer_first_name,
              last_name: signer_last_name,
              email: signer_email,
              designation: "Signer",
              order: 1,
            },
          ],
        }),
      }
    );

    const data = await response.json();

    if (!response.ok) {
      return Response.json({ error: data }, { status: response.status });
    }

    return Response.json({
      signing_request_id: data.id,
      status: "sent",
    });
  },
};
```

<Tip>
  Endpoint `create-and-send` tworzy żądanie podpisu i wysyła je do odbiorców w jednym wywołaniu. Jeśli chcesz przejrzeć lub zmodyfikować żądanie przed wysłaniem, użyj `POST /signing-requests`, aby utworzyć wersję roboczą, a następnie oddzielnie `POST /signing-requests/{id}/send`.
</Tip>

### Krok 4: Wdrożenie

```bash
wrangler deploy
```

Wrangler wyświetli URL Twojego Workera (coś w rodzaju `https://firma-signing.<your-subdomain>.workers.dev`). Twój frontend może teraz przesyłać POST z danymi podpisującego na ten URL.

### Obsługa webhooków

Aby śledzić podpisywanie dokumentów, utwórz drugi Worker, który odbiera zdarzenia webhooka Firma. W panelu Firma w sekcji **Settings → Webhooks** zarejestruj webhook wskazujący na URL tego Workera.

```javascript
export default {
  async fetch(request, env) {
    if (request.method !== "POST") {
      return new Response("Method not allowed", { status: 405 });
    }

    const payload = await request.json();
    const { type, data } = payload;

    if (type === "signing_request.completed") {
      const signingRequestId = data.signing_request.id;
      console.log(`Signing request ${signingRequestId} completed`);
    }

    if (type === "signing_request.recipient.signed") {
      const recipientEmail = data.recipient?.email;
      console.log(`${recipientEmail} signed the document`);
    }

    return Response.json({ received: true });
  },
};
```

<Warning>
  Do użytku produkcyjnego zawsze weryfikuj podpis webhooka przy użyciu swojego sekretu podpisywania webhooków Firma. Zobacz [przewodnik po webhookach](/guides/webhooks), aby poznać szczegóły implementacji.
</Warning>

### Krok 1: Skonfiguruj projekt Pages

Jeśli masz już projekt Cloudflare Pages, przejdź do następnego kroku. W przeciwnym razie utwórz go:

```bash
mkdir firma-app && cd firma-app
npm init -y
mkdir public functions
echo "<h1>Firma App</h1>" > public/index.html
```

Pages Functions znajdują się w katalogu `/functions`. Każdy plik automatycznie mapuje się na ścieżkę URL, więc `functions/api/send-signing-request.js` staje się `/api/send-signing-request`.

### Krok 2: Zapisz klucz API jako sekret

Dodaj klucz API Firma jako zaszyfrowany sekret Pages:

```bash
wrangler pages secret put FIRMA_API_KEY --project-name your-project-name
```

Alternatywnie przejdź do **Cloudflare Dashboard → Pages → your project → Settings → Environment variables** i dodaj `FIRMA_API_KEY` jako zaszyfrowaną zmienną.

### Krok 3: Utwórz Pages Function do wysyłania żądań podpisu

Utwórz nowy plik w `functions/api/send-signing-request.js`:

```javascript
const FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api";

export async function onRequestPost(context) {
  const { name, template_id, signer_email, signer_first_name, signer_last_name } =
    await context.request.json();

  const response = await fetch(
    `${FIRMA_API}/signing-requests/create-and-send`,
    {
      method: "POST",
      headers: {
        Authorization: context.env.FIRMA_API_KEY,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        name,
        template_id,
        recipients: [
          {
            first_name: signer_first_name,
            last_name: signer_last_name,
            email: signer_email,
            designation: "Signer",
            order: 1,
          },
        ],
      }),
    }
  );

  const data = await response.json();

  if (!response.ok) {
    return Response.json({ error: data }, { status: response.status });
  }

  return Response.json({
    signing_request_id: data.id,
    status: "sent",
  });
}
```

<Tip>
  Pages Functions eksportują nazwane handlery per metoda HTTP: `onRequestPost`, `onRequestGet` itd. Jeśli musisz obsłużyć wiele metod, wyeksportuj `onRequest` i sprawdź `context.request.method` wewnątrz.
</Tip>

### Krok 4: Wdrożenie

Jeśli Twój projekt Pages jest połączony z repozytorium Git, wypchnij zmiany, a Cloudflare wdroży automatycznie. Do ręcznych wdrożeń:

```bash
wrangler pages deploy public --project-name your-project-name
```

Twoja funkcja jest teraz dostępna pod `https://your-project.pages.dev/api/send-signing-request`.

### Obsługa webhooków

Utwórz drugą Pages Function w `functions/api/firma-webhook.js`, aby obsłużyć przychodzące zdarzenia webhooka Firma. Zarejestruj ten URL w panelu Firma w sekcji **Settings → Webhooks**.

```javascript
export async function onRequestPost(context) {
  const payload = await context.request.json();
  const { type, data } = payload;

  if (type === "signing_request.completed") {
    const signingRequestId = data.signing_request.id;
    console.log(`Signing request ${signingRequestId} completed`);
  }

  if (type === "signing_request.recipient.signed") {
    const recipientEmail = data.recipient?.email;
    console.log(`${recipientEmail} signed the document`);
  }

  return Response.json({ received: true });
}
```

<Warning>
  Do użytku produkcyjnego zawsze weryfikuj podpis webhooka przy użyciu swojego sekretu podpisywania webhooków Firma. Zobacz [przewodnik po webhookach](/guides/webhooks), aby poznać szczegóły implementacji.
</Warning>

## Osadzone podpisywanie

Dla aplikacji, w których podpisujący dokańczają dokumenty bezpośrednio w Twoim UI, Firma oferuje osadzone środowisko podpisywania. Odpowiedź `create-and-send` zawiera `first_signer.id` (czyli `signing_request_user_id`) oraz gotowy `first_signer.signing_link`. Załaduj go w iframe:

```html
<iframe
  src="https://app.firma.dev/signing/{signing_request_user_id}"
  style="width:100%;height:900px;border:0;"
  allow="camera;microphone;clipboard-write"
  title="Document Signing"
></iframe>
```

Zobacz [przewodnik po osadzonym podpisywaniu](/guides/embeddable-signing), aby uzyskać pełne instrukcje konfiguracji, w tym najlepsze praktyki bezpieczeństwa.

## Bonus: Połączenie MCP dla budowania wspomaganego przez AI

Firma oferuje [serwer MCP dokumentacji](/guides/mcp), do którego mogą się łączyć narzędzia kodujące AI. Jeśli używasz asystenta AI podczas budowania integracji z Cloudflare, podłączenie serwera MCP pozwala mu przeszukiwać dokumentację Firma i generować dokładne wywołania API.

Dodaj URL serwera MCP do konfiguracji swojego narzędzia:

```text
https://docs.firma.dev/mcp
```

Jest to tylko dla środowiska developerskiego i nie wpływa na wdrożoną aplikację.
