# Osadzane podpisywanie

Osadź przepływ podpisywania bezpośrednio w swojej aplikacji, aby odbiorcy mogli podpisywać dokumenty bez opuszczania Twojego produktu. Tworzy to bezproblemowe, white-labelowe środowisko podpisywania.

## Wzorzec URL podpisywania

Interfejs podpisywania jest dostępny pod:

`https://app.firma.dev/signing/{signing_request_user_id}`

## Podstawowe osadzenie w iframe

```html
<iframe 
  src="https://app.firma.dev/signing/{signing_request_user_id}" 
  style="width:100%;height:900px;border:0;" 
  allow="camera;microphone;clipboard-write"
  title="Document Signing"
></iframe>
```

### Wymagane uprawnienia iframe

- `camera` — do weryfikacji tożsamości (jeśli włączona)
- `microphone` — do weryfikacji wideo (jeśli włączona)
- `clipboard-write` — do kopiowania/wklejania treści

## Uzyskiwanie URL-a podpisywania

`signing_request_user_id` jest zwracany podczas pobierania użytkowników żądania podpisu przez API.

### Przykład: pobierz URL-e podpisywania odbiorców

```js
const response = await fetch(
  `https://api.firma.dev/functions/v1/signing-request-api/signing-requests/${signingRequestId}/users`,
  {
    headers: {
      'Authorization': `Bearer ${API_KEY}`
    }
  }
)

const users = await response.json()

// Każdy użytkownik ma własny URL podpisywania
users.forEach(user => {
  const signingUrl = `https://app.firma.dev/signing/${user.id}`
  console.log(`${user.email}: ${signingUrl}`)
})
```

## Kompletny przykład implementacji

### Komponent React

```jsx
import { useState, useEffect } from 'react'

function SigningView({ signingRequestId, recipientEmail }) {
  const [signingUrl, setSigningUrl] = useState(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)
  
  useEffect(() => {
    async function loadSigningUrl() {
      try {
        // Pobierz użytkowników żądania podpisu z backendu
        const response = await fetch(
          `/api/signing-requests/${signingRequestId}/users`,
          { credentials: 'include' }
        )
        
        if (!response.ok) {
          throw new Error('Failed to load signing users')
        }
        
        const users = await response.json()
        
        // Znajdź użytkownika po e-mailu
        const user = users.find(
          u => u.email === recipientEmail
        )
        
        if (!user) {
          throw new Error('User not found')
        }
        
        const url = `https://app.firma.dev/signing/${user.id}`
        setSigningUrl(url)
        setLoading(false)
      } catch (err) {
        setError(err.message)
        setLoading(false)
      }
    }
    
    loadSigningUrl()
  }, [signingRequestId, recipientEmail])
  
  if (loading) return <div>Loading signing interface...</div>
  if (error) return <div>Error: {error}</div>
  
  return (
    <iframe
      src={signingUrl}
      style={{
        width: '100%',
        height: '900px',
        border: 0
      }}
      allow="camera;microphone;clipboard-write"
      title="Sign Document"
    />
  )
}
```

### Endpoint po stronie serwera (Node.js)

```js
import express from 'express'
import fetch from 'node-fetch'

const app = express()
const API_KEY = process.env.FIRMA_API_KEY

app.get('/api/signing-requests/:id/users', async (req, res) => {
  try {
    const response = await fetch(
      `https://api.firma.dev/functions/v1/signing-request-api/signing-requests/${req.params.id}/users`,
      {
        headers: {
          'Authorization': `Bearer ${API_KEY}`
        }
      }
    )
    
    if (!response.ok) {
      const error = await response.text()
      return res.status(response.status).json({ error })
    }
    
    const users = await response.json()
    res.json(users)
  } catch (error) {
    console.error('Failed to fetch signing users:', error)
    res.status(500).json({ error: 'Internal server error' })
  }
})

app.listen(3000)
```

## Zdarzenia postMessage

Iframe podpisywania emituje zdarzenia postMessage do śledzenia postępu podpisywania:

```js
window.addEventListener('message', (event) => {
  // Waliduj origin
  if (event.origin !== 'https://app.firma.dev') return
  
  const data = event.data
  
  switch (data.type) {
    case 'signing.started':
      console.log('User started signing')
      break
    case 'signing.completed':
      console.log('Document signed successfully')
      // Przekieruj lub pokaż komunikat sukcesu
      break
    case 'signing.declined':
      console.log('User declined to sign')
      break
    case 'signing.error':
      console.error('Signing error:', data.error)
      break
  }
})
```

## Najlepsze praktyki bezpieczeństwa

<Warning>
  **Nigdy nie ujawniaj kluczy API w kodzie frontendu.** Zawsze pobieraj ID użytkowników podpisujących przez bezpieczny endpoint backendu.
</Warning>

### ✅ Rób

- ✅ Pobieraj ID użytkowników podpisujących przez backend
- ✅ Waliduj origin zdarzeń postMessage (`https://app.firma.dev`)
- ✅ Używaj HTTPS dla wszystkich żądań API
- ✅ Monitoruj zdarzenia podpisywania przez webhooki (60 zapytań/min)

### ❌ Nie rób

- ❌ Nie ujawniaj kluczy API w kodzie klienta
- ❌ Nie ufaj danym postMessage bez walidacji origin

## Integracja z webhookami

Używaj webhooków do śledzenia zdarzeń podpisywania w czasie rzeczywistym zamiast odpytywania:

```js
// Handler webhooka
app.post('/webhooks/firma', async (req, res) => {
  const event = req.body
  
  switch (event.event_type) {
    case 'signing_request.viewed':
      await notifyUser(event.data.signing_request_id, 'viewed')
      break
    case 'signing_request.recipient.signed':
      await notifyUser(event.data.signing_request_id, 'signed')
      break
    case 'signing_request.completed':
      await markComplete(event.data.signing_request_id)
      break
  }
  
  res.json({ received: true })
})
```

Zobacz [przewodnik po webhookach](webhooks), aby zapoznać się z kompletnymi szczegółami implementacji.

## Rozwiązywanie problemów

### Iframe nie ładuje się

**Możliwe przyczyny**:

- Nieprawidłowy `signing_request_user_id`
- Odbiorca już zakończył podpisywanie
- Żądanie podpisu zostało anulowane lub wygasło

**Rozwiązanie**: zweryfikuj status żądania podpisu przez API

### Zdarzenia postMessage nie są odbierane

**Możliwe przyczyny**:

- Walidacja origin blokuje wiadomości
- Listener zdarzeń nie został dołączony przed załadowaniem iframe

**Rozwiązanie**:

- Sprawdź, czy origin to dokładnie `https://app.firma.dev`
- Dołącz listener przed utworzeniem iframe

### 403 Forbidden podczas pobierania żądania podpisu

**Możliwe przyczyny**:

- Użytkownik nie ma dostępu do żądania podpisu
- Klucz API nie ma wymaganych uprawnień

**Rozwiązanie**: zweryfikuj logikę autoryzacji i uprawnienia klucza API
