Firma udostępnia dwa serwery MCP, które pozwalają asystentom AI współpracować z Firmą na różne sposoby:

- **Data MCP** (`https://mcp.firma.dev/mcp`) - Daje asystentom AI bezpośredni dostęp do API Firmy. Zarządzaj szablonami, twórz żądania podpisu, konfiguruj przestrzenie robocze i więcej, korzystając z rozmów w języku naturalnym. Wymaga uwierzytelniania OAuth.
- **Docs MCP** (`https://docs.firma.dev/mcp`) - Daje asystentom AI dostęp do pełnej dokumentacji i referencji API Firmy. Użyj go, aby generować dokładny kod integracji z prawidłowymi endpointami, kształtami żądań i wzorcami uwierzytelniania. Nie wymaga uwierzytelniania.

Większość deweloperów chce mieć oba podłączone: Data MCP do operowania na danych Firmy i Docs MCP do budowania integracji z Firmą.

## Czym jest MCP?

[Model Context Protocol](https://modelcontextprotocol.io) (MCP) to otwarty standard łączący asystentów AI z zewnętrznymi narzędziami i źródłami danych. Zamiast pisać wywołania API ręcznie, opisujesz, czego potrzebujesz, w języku naturalnym, a asystent AI używa serwerów MCP Firmy, aby wykonać odpowiednie wywołania API lub wyszukać właściwą dokumentację w Twoim imieniu.

## Konfiguracja

<Tabs>

<Tab title="Data MCP">

Serwer Data MCP uwierzytelnia przez OAuth 2.1 z PKCE, używając Twoich danych logowania do konta Firmy. Podłączając się przez dowolnego obsługiwanego klienta, przepływ OAuth jest obsługiwany automatycznie.

**Obsługiwani klienci:**

- **Claude** - Claude Code, Claude Desktop, Claude.ai
- **ChatGPT** - Przez konektory Developer Mode
- **Cursor** - Natywne wsparcie MCP
- **GitHub Copilot** - VS Code 1.101+
- **OpenAI Codex** - Przez CLI lub konfigurację
- **Inni klienci MCP** - Dowolny klient implementujący [transport Streamable HTTP](https://modelcontextprotocol.io/specification/2025-03-26/basic/transports#streamable-http)

### Co udostępnia Docs MCP

Serwer Docs MCP daje asystentom AI dostęp do:

- **Referencji API** - Adresy URL endpointów, kształty żądań/odpowiedzi, szczegóły uwierzytelniania
- **Przewodników integracji** - Instrukcje krok po kroku dla każdej obsługiwanej platformy
- **Przewodników funkcji** - Webhooki, osadzone podpisywanie, white labeling, szablony e-maili i więcej
- **Przykładów kodu** - Działające przykłady w Node.js, Pythonie i innych językach

### Kiedy go używać

Odwołuj się do „the Firma docs" w swoich podpowiedziach, aby powiedzieć asystentowi, aby zapytał ten serwer:

```text
Using the Firma docs, create an Express route that sends a signing request
using the create-and-send endpoint. Use process.env.FIRMA_API_KEY for auth.
```

Bez tego odniesienia Twój asystent może wrócić do ogólnej wiedzy i pominąć szczegóły endpointu lub uwierzytelniania.

## Wybór przestrzeni roboczej

Po zalogowaniu się do Data MCP zaczynasz bez wybranej przestrzeni roboczej. Możesz:

- **Poprosić asystenta o wybór przestrzeni roboczej** - np. „Switch to the Marketing workspace"
- **Pozwolić asystentowi przekazywać ID przestrzeni roboczej dla każdego żądania** - asystent może dołączyć parametr `workspace_id` do poszczególnych wywołań narzędzi

Asystent zazwyczaj najpierw wyświetli listę Twoich przestrzeni roboczych i zapyta, której użyć.

## Przykładowe rozmowy

### Wyświetlanie wszystkich żądań podpisu

> „Show me all pending signing requests in my Marketing workspace"

Asystent wybierze przestrzeń roboczą, wywoła endpoint listy żądań podpisu i przedstawi wyniki.

### Utworzenie i wysłanie żądania podpisu

> „Create a signing request from the NDA template and send it to jane@example.com"

Asystent znajdzie szablon, zduplikuje go do żądania podpisu, ustawi odbiorcę i wyśle.

### Sprawdzenie statusu podpisywania

> „What's the status of the signing request I sent to John last week?"

Asystent przeszuka niedawne żądania podpisu i zwróci aktualny status, w tym którzy odbiorcy podpisali.

### Generowanie kodu integracji

> „Using the Firma docs, create a Next.js API route that sends a signing request using the create-and-send endpoint"

Asystent zapyta Docs MCP o prawidłowy endpoint, kształt żądania i wzorzec uwierzytelniania, a następnie wygeneruje działający kod.

## Limity szybkości

Żądania MCP podlegają tym samym [limitom szybkości](rate-limits) co bezpośrednie wywołania API. Każde wywołanie narzędzia odpowiada jednemu żądaniu API. Serwer Docs MCP nie ma limitów szybkości.

## Rozwiązywanie problemów

### Narzędzia nie ładują się

Jeśli Twój klient pokazuje serwer jako połączony, ale żadne narzędzia się nie pojawiają:

- **Claude.ai / ChatGPT**: Spróbuj odłączyć i ponownie podłączyć konektor z Ustawień
- **Cursor / VS Code**: Uruchom ponownie aplikację po dodaniu konfiguracji
- **Claude Desktop**: Uruchom ponownie aplikację - zmiany w `claude_desktop_config.json` wymagają ponownego uruchomienia

### Nie znaleziono przestrzeni roboczej

Upewnij się, że wybrałeś przestrzeń roboczą po połączeniu z Data MCP. Użyj „List my workspaces", aby zobaczyć dostępne opcje.

### Błędy uwierzytelniania

Spróbuj wylogować się i zalogować ponownie, aby odświeżyć sesję. Jeśli problem się powtarza, odłącz i ponownie podłącz serwer w ustawieniach klienta. Docs MCP nie wymaga uwierzytelniania.
