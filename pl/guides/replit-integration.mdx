Firma pozwala dodać prawnie wiążące podpisy elektroniczne do dowolnej aplikacji Replit. Użyj backendu Node.js lub Python Replit do wywoływania API Firmy, zarządzania szablonami, wysyłania żądań podpisu i śledzenia zakończeń przez webhooki.

Ten przewodnik obejmuje trzy ścieżki integracji:

1. **Node.js + Express** — Dodaj trasę backendową, która wywołuje REST API Firmy. Najlepsze, gdy Twoja aplikacja już używa JavaScriptu lub TypeScriptu.
2. **Python + Flask** — Te same koncepcje, składnia Pythona. Najlepsze, gdy Twoja aplikacja już używa Pythona.
3. **Podpowiedź agenta Replit** — Pozwól agentowi Replit podłączyć integrację Firmy za Ciebie z podpowiedzi w języku naturalnym. Działa dla dowolnej aplikacji.

## Wymagania wstępne

- [Konto Firma](https://app.firma.dev) z kluczem API
- Konto Replit (darmowy plan Starter zawiera dzienne kredyty agenta; wdrożenia wymagają płatnego planu)
- Co najmniej jeden szablon Firmy z skonfigurowanymi polami do podpisu

<Note>
  Firma używa surowego klucza API jako wartości nagłówka `Authorization` — nie poprzedzaj go prefiksem `Bearer`. Różni się to od wielu innych API.
</Note>

## Ścieżka 1: Node.js + Express

Dodajesz trasę backendową, która wywołuje REST API Firmy przy użyciu klucza API przechowywanego jako sekret Replit.

### Krok 1: zapisz klucz API jako sekret Replit

1. Otwórz swój Repl
2. Otwórz narzędzie **Secrets** z lewego doku narzędzi (lub wyszukaj „Secrets" za pomocą paska wyszukiwania)
3. Kliknij **New Secret**
4. Ustaw klucz na `FIRMA_API_KEY` i wklej swój klucz API Firmy jako wartość

<Warning>
  Nigdy nie ujawniaj klucza API w kodzie frontendu. Zawsze wywołuj API Firmy z trasy backendowej, gdzie sekrety są przechowywane bezpiecznie.
</Warning>

<Note>
  Sekrety przestrzeni roboczej Replit nie są automatycznie przenoszone do wdrożeń. Po dodaniu sekretu w przestrzeni roboczej, dodaj go również w panelu **Publish** w jego własnej sekcji Secrets przed publikacją. Jest to najczęstsza przyczyna tego, że integracja Firmy działa w programowaniu, ale zawodzi w produkcji.
</Note>

### Krok 2: utwórz trasę backendową do wysyłania żądań podpisu

Dodaj trasę do swojego serwera Express. Ten przykład używa endpointu `create-and-send`, aby utworzyć żądanie podpisu z szablonu i wysłać je w pojedynczym wywołaniu API:

```javascript
import express from "express";

const app = express();
app.use(express.json());

const FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api";

app.post("/api/send-signing-request", async (req, res) => {
  const { name, template_id, signer_email, signer_first_name, signer_last_name } = req.body;

  try {
    const response = await fetch(`${FIRMA_API}/signing-requests/create-and-send`, {
      method: "POST",
      headers: {
        Authorization: process.env.FIRMA_API_KEY,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        name,
        template_id,
        recipients: [
          {
            first_name: signer_first_name,
            last_name: signer_last_name,
            email: signer_email,
            designation: "Signer",
            order: 1,
          },
        ],
      }),
    });

    const data = await response.json();

    if (!response.ok) {
      return res.status(response.status).json({ error: data });
    }

    res.json({
      signing_request_id: data.id,
      status: "sent",
    });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, "0.0.0.0", () => {
  console.log(`Server running on port ${PORT}`);
});
```

<Note>
  Endpoint `create-and-send` tworzy żądanie podpisu i wysyła je do odbiorców atomowo. Jeśli musisz przejrzeć lub zmodyfikować żądanie przed wysłaniem, użyj `POST /signing-requests` do utworzenia wersji roboczej, a następnie osobno `POST /signing-requests/{id}/send`.
</Note>

### Krok 3: połącz trasę z interfejsem aplikacji

Wywołaj trasę backendową z frontendu:

```javascript
async function sendContract() {
  const response = await fetch("/api/send-signing-request", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      name: "NDA for Alice Johnson",
      template_id: "your-template-id",
      signer_email: "alice@example.com",
      signer_first_name: "Alice",
      signer_last_name: "Johnson",
    }),
  });

  const result = await response.json();
  console.log("Signing request sent:", result.signing_request_id);
}
```

Jeśli budujesz z agentem Replit, możesz również opisać przepływ w czacie i pozwolić agentowi go podłączyć, na przykład: *„When the user clicks the Send Contract button, call /api/send-signing-request with the template ID, signer email, and name from the form."*

### Integracja webhooków

Aby śledzić, kiedy dokumenty są podpisane, dodaj drugą trasę do odbierania zdarzeń webhook Firmy.

1. Dodaj trasę `POST /api/firma-webhook` do swojego serwera Express
2. Opublikuj swój Repl, aby miał publiczny URL HTTPS (na przykład `https://your-app.replit.app`)
3. W panelu Firmy w **Settings → Webhooks** zarejestruj webhook wskazujący na `https://your-app.replit.app/api/firma-webhook`
4. Firma wysyła zdarzenia dla kluczowych zmian stanu — zobacz [przewodnik po webhookach](/guides/webhooks), aby uzyskać wszystkie typy zdarzeń

```javascript
app.post("/api/firma-webhook", async (req, res) => {
  const { type, data } = req.body;

  if (type === "signing_request.completed") {
    const signingRequestId = data.signing_request.id;

    // Update your database or trigger the next step in your workflow.
    console.log(`Signing request ${signingRequestId} completed`);
  }

  if (type === "signing_request.recipient.signed") {
    const recipientEmail = data.recipient?.email;
    console.log(`${recipientEmail} signed the document`);
  }

  res.json({ received: true });
});
```

<Warning>
  W środowisku produkcyjnym zawsze weryfikuj podpis webhooka, używając sekretu podpisu webhooka Firmy. Szczegóły implementacji znajdziesz w [przewodniku po webhookach](/guides/webhooks).
</Warning>

<Note>
  Endpointy webhook działają tylko na opublikowanych Replach, a nie na URL programistycznym. Opublikuj swoją aplikację przez **Autoscale** lub **Reserved VM** przed zarejestrowaniem webhooka. Wdrożenia statyczne nie obsługują tras backendowych.
</Note>

## Ścieżka 2: Python + Flask

Ta sama integracja, składnia Pythona. Przydatne, jeśli Twój Repl już działa na Flask lub FastAPI.

### Krok 1: zapisz klucz API

Tak samo jak w Ścieżce 1: otwórz narzędzie **Secrets**, dodaj `FIRMA_API_KEY` z Twoim kluczem API Firmy jako wartością.

### Krok 2: utwórz trasę Flask

```python
import os
import requests
from flask import Flask, request, jsonify

app = Flask(__name__)

FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api"

@app.route("/api/send-signing-request", methods=["POST"])
def send_signing_request():
    body = request.get_json()
    template_id = body.get("template_id")
    name = body.get("name")
    signer_email = body.get("signer_email")
    signer_first_name = body.get("signer_first_name")
    signer_last_name = body.get("signer_last_name")

    response = requests.post(
        f"{FIRMA_API}/signing-requests/create-and-send",
        headers={
            "Authorization": os.environ["FIRMA_API_KEY"],
            "Content-Type": "application/json",
        },
        json={
            "name": name,
            "template_id": template_id,
            "recipients": [
                {
                    "first_name": signer_first_name,
                    "last_name": signer_last_name,
                    "email": signer_email,
                    "designation": "Signer",
                    "order": 1,
                }
            ],
        },
    )

    data = response.json()

    if not response.ok:
        return jsonify({"error": data}), response.status_code

    return jsonify({
        "signing_request_id": data["id"],
        "status": "sent",
    })


@app.route("/api/firma-webhook", methods=["POST"])
def firma_webhook():
    payload = request.get_json()
    event_type = payload.get("type")
    data = payload.get("data", {})

    if event_type == "signing_request.completed":
        signing_request_id = data["signing_request"]["id"]
        # Update your database or trigger the next step in your workflow

    if event_type == "signing_request.recipient.signed":
        recipient_email = data.get("recipient", {}).get("email")
        # Handle individual recipient signing

    return jsonify({"received": True})


if __name__ == "__main__":
    app.run(host="0.0.0.0", port=int(os.environ.get("PORT", 3000)))
```

Wywołaj to z frontendu dokładnie tak jak wersję Node.js. Kształty żądań i odpowiedzi są identyczne.

## Ścieżka 3: zbuduj integrację z agentem Replit

Jeśli używasz agenta Replit do generowania swojej aplikacji, możesz pominąć samodzielne pisanie szablonowego kodu. Wyślij agentowi podpowiedź taką jak ta:

```text
Add an e-signature feature to this app using the Firma API.

Create a POST /api/send-signing-request route that:
- Reads the API key from process.env.FIRMA_API_KEY
- Accepts name, template_id, signer_email, signer_first_name, signer_last_name in the request body
- Calls POST https://api.firma.dev/functions/v1/signing-request-api/signing-requests/create-and-send
- Sends the Authorization header as the raw API key (no Bearer prefix needed)
- Includes the "name" field in the request body (it is required)
- Returns the signing_request_id

Also create a POST /api/firma-webhook route that handles signing_request.completed events.

Add a "Send for signature" button to the UI that calls the route.
```

Agent utworzy szkielet trasy, połączy ją z UI i poprosi Cię o dodanie `FIRMA_API_KEY` do Secrets. Aby uzyskać dokładne szczegóły API podczas generowania, połącz serwer Firma Docs MCP (patrz poniżej), a agent pobierze najnowsze schematy endpointów bezpośrednio z dokumentacji Firmy.

## Osadzone podpisywanie

W przypadku aplikacji, w których sygnatariusze wypełniają dokumenty bezpośrednio w Twoim interfejsie, a nie na osobnej stronie Firmy, osadź doświadczenie podpisywania w iframe. Odpowiedź `create-and-send` zawiera `first_signer.id` (`signing_request_user_id`) i gotowy `first_signer.signing_link`. Załaduj URL sygnatariusza w swojej aplikacji Replit:

```html
<iframe
  src="https://app.firma.dev/signing/{signing_request_user_id}"
  style="width:100%;height:900px;border:0;"
  allow="camera;microphone;clipboard-write"
  title="Document Signing"
></iframe>
```

Zobacz [przewodnik osadzonego podpisywania](/guides/embeddable-signing), aby uzyskać pełne instrukcje konfiguracji, w tym najlepsze praktyki bezpieczeństwa.

## Bonus: połączenie MCP dla budowania wspomaganego przez AI

Firma zapewnia [serwer Docs MCP](/guides/mcp), który możesz podłączyć do agenta Replit. Po podłączeniu agent może przeszukiwać dokumentację Firmy podczas generowania kodu, dzięki czemu pisze integracje z prawidłowymi endpointami, parametrami i nagłówkami uwierzytelniania zamiast zgadywać.

Aby to skonfigurować:

1. Przejdź do strony **Integrations** w Replit
2. Przewiń do **MCP Servers for Replit Agent** i kliknij **Add MCP server**
3. Nadaj mu nazwę taką jak `firma-docs`
4. Wpisz URL serwera: `https://docs.firma.dev/mcp`
5. Kliknij **Test & Save**

Agent będzie teraz automatycznie pobierał dokumentację Firmy, gdy poprosisz go o zbudowanie czegokolwiek, co dotyka e-podpisów.

## Wielotenantowe SaaS: przestrzenie robocze klientów

Jeśli budujesz wielotenantową aplikację SaaS na Replit, w której każdy z Twoich klientów potrzebuje własnego izolowanego środowiska podpisywania, użyj [przestrzeni roboczych klientów Firmy](/guides/creating-workspaces). Każdy klient otrzymuje prywatną przestrzeń roboczą z własnymi szablonami, żądaniami podpisu i śledzeniem użycia, bez wycieku danych między tenantami. Utwórz nową przestrzeń roboczą ze swojego backendu, gdy klient się zarejestruje, i przechowuj jego ID przestrzeni roboczej i klucz API razem z resztą jego rekordu tenanta.
