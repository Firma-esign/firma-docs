<Info>
  **Pré-requisitos**:
  - Node.js versão 19 ou superior
  - Um repositório de documentação com um arquivo `docs.json`
</Info>

Siga estas etapas para instalar e executar o Mintlify no seu sistema operacional.

<Steps>
<Step title="Instalar o CLI do Mintlify">

```bash
npm i -g mint
```
</Step>

<Step title="Visualização local">

Navegue até o diretório dos seus docs onde o arquivo `docs.json` está localizado e execute o seguinte comando:

```bash
mint dev
```

Uma visualização local da sua documentação estará disponível em `http://localhost:3000`.

</Step>
</Steps>

## Portas personalizadas

Por padrão, o Mintlify usa a porta 3000. Você pode personalizar a porta em que o Mintlify roda usando a flag `--port`. Por exemplo, para executar o Mintlify na porta 3333, use este comando:

```bash
mint dev --port 3333
```

Se você tentar executar o Mintlify em uma porta que já está em uso, ele usará a próxima porta disponível:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Versões do Mintlify

Observe que cada lançamento do CLI está associado a uma versão específica do Mintlify. Se sua visualização local não corresponder à versão de produção, atualize o CLI:

```bash
npm mint update
```

## Validando links

O CLI pode ajudar na validação de links da sua documentação. Para identificar quaisquer links quebrados, use o seguinte comando:

```bash
mint broken-links
```

## Formatação de código

Sugerimos usar extensões na sua IDE para reconhecer e formatar MDX. Se você é usuário de VSCode, considere a [extensão MDX do VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) para realce de sintaxe e o [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) para formatação de código.

## Solução de problemas

<AccordionGroup>
  <Accordion title='Erro: não foi possível carregar o módulo "sharp" usando o runtime darwin-arm64'>

    Isso pode ocorrer devido a uma versão desatualizada do node. Tente o seguinte:
    1. Remova a versão atualmente instalada do CLI: `npm remove -g mint`
    2. Atualize para o Node v19 ou superior.
    3. Reinstale o CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Problema: encontrando um erro desconhecido">
  
    Solução: vá até a raiz do seu dispositivo e exclua a pasta `~/.mintlify`. Em seguida, execute `mint dev` novamente.
  </Accordion>
</AccordionGroup>

Curioso sobre o que mudou na versão mais recente do CLI? Confira o [changelog do CLI](https://www.npmjs.com/package/mintlify?activeTab=versions).
