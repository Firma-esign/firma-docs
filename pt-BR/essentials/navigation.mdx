O menu de navegação é a lista de links em cada site.

Você provavelmente atualizará o `docs.json` toda vez que adicionar uma nova página. As páginas não aparecem automaticamente.

## Sintaxe de navegação

Nossa sintaxe de navegação é recursiva, o que significa que você pode criar grupos de navegação aninhados. Você não precisa incluir `.mdx` nos nomes das páginas.

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": [
            "quickstart",
            {
              "group": "Nested Reference Pages",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Pastas

Basta colocar seus arquivos MDX em pastas e atualizar os caminhos em `docs.json`.

Por exemplo, para ter uma página em `https://seusite.com/sua-pasta/sua-pagina`, você criaria uma pasta chamada `sua-pasta` contendo um arquivo MDX chamado `sua-pagina.mdx`.

<Warning>

Você não pode usar `api` como nome de pasta, a menos que a aninhe dentro de outra pasta. O Mintlify usa Next.js, que reserva a pasta `api` de nível superior para chamadas internas de servidor. Um nome de pasta como `api-reference` seria aceito.

</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## Páginas ocultas

Arquivos MDX não incluídos em `docs.json` não aparecerão na barra lateral, mas são acessíveis pela barra de pesquisa e vinculando-se diretamente a eles.
