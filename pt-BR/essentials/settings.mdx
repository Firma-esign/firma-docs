description: 'O Mintlify oferece controle total sobre a aparência da sua documentação por meio do arquivo docs.json'

---

Todo site Mintlify precisa de um arquivo `docs.json` com as configurações principais. Saiba mais sobre as [propriedades](#properties) abaixo.

## Propriedades

<ResponseField name="name" type="string" required>
Nome do seu projeto. Usado para o título global.

Exemplo: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Um array de grupos com todas as páginas dentro daquele grupo
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    O nome do grupo.

    Exemplo: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    Os caminhos relativos dos arquivos markdown que servirão como páginas.

    Exemplo: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Caminho para a imagem do logo ou objeto com caminho para as imagens do logo em modo "light" e "dark"
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Caminho para o logo no modo claro
    </ResponseField>
    <ResponseField name="dark" type="string">
      Caminho para o logo no modo escuro
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Para onde clicar no logo leva você
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Caminho para a imagem do favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de cor em hexadecimal para seu tema global
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      A cor primária. Usada mais frequentemente para conteúdo destacado, cabeçalhos
      de seção e acentos no modo claro
    </ResponseField>
    <ResponseField name="light" type="string">
      A cor primária para o modo escuro. Usada mais frequentemente para conteúdo
      destacado, cabeçalhos de seção e acentos no modo escuro
    </ResponseField>
    <ResponseField name="dark" type="string">
      A cor primária para botões importantes
    </ResponseField>
    <ResponseField name="background" type="object">
      A cor do plano de fundo nos modos claro e escuro
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          O código de cor hexadecimal do plano de fundo no modo claro
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          O código de cor hexadecimal do plano de fundo no modo escuro
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array de `name`s e `url`s dos links que você quer incluir na barra superior
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    O nome do botão.

    Exemplo: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    A url ao clicar no botão. Exemplo: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link exibe um botão. GitHub exibe as informações do repositório na url fornecida, incluindo o número de estrelas do GitHub.
    </ResponseField>
    <ResponseField name="url" type="string">
    Se `link`: para onde o botão leva.
    
    Se `github`: link para o repositório do qual carregar as informações do GitHub.
    </ResponseField>
    <ResponseField name="name" type="string">
    Texto dentro do botão. Só é obrigatório se `type` for `link`.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array com nomes das versões. Use isto apenas se quiser exibir diferentes versões
  dos docs com um dropdown na barra de navegação.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Um array de anchors, inclui `icon`, `color` e `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    O ícone [Font Awesome](https://fontawesome.com/search?q=heart) usado para destacar a âncora.

    Exemplo: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    O nome do rótulo da âncora.

    Exemplo: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      O início da URL que marca quais páginas vão para a âncora. Geralmente, é o nome da pasta em que você colocou suas páginas.
    </ResponseField>
    <ResponseField name="color" type="string">
      A cor hexadecimal do fundo do ícone da âncora. Pode também ser um gradiente se você passar um objeto com as propriedades `from` e `to`, cada uma sendo uma cor hexadecimal.
    </ResponseField>
    <ResponseField name="version" type="string">
      Usado se você quiser ocultar uma âncora até que a versão correta dos docs seja selecionada.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Passe `true` se você quiser ocultar a âncora até que você direcione alguém diretamente para os docs dentro dela.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Um dos seguintes: "brands", "duotone", "light", "sharp-solid", "solid" ou "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Substitua as configurações padrão para a âncora superior.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      O nome da âncora superior
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Ícone Font Awesome.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Um dos seguintes: "brands", "duotone", "light", "sharp-solid", "solid" ou "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Um array de abas de navegação.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      O nome do rótulo da aba.
    </ResponseField>
    <ResponseField name="url" type="string">
      O início da URL que marca quais páginas vão para a aba. Geralmente, é o nome
      da pasta em que você colocou suas páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuração para as definições de API. Saiba mais sobre páginas de API em [Componentes de API](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      A URL base para todos os endpoints da API. Se `baseUrl` for um array, permitirá
      múltiplas opções de URL base que o usuário pode alternar.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          A estratégia de autenticação usada para todos os endpoints da API.
        </ResponseField>
        <ResponseField name="name" type="string">
        O nome do parâmetro de autenticação usado no playground da API.

        Se o método for `basic`, o formato deve ser `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        O valor padrão projetado para ser um prefixo do campo de entrada de autenticação.

        Ex.: um `inputPrefix` de `AuthKey` herdaria o resultado padrão do campo de autenticação como `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configurações para o playground da API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Se o playground está sendo exibido, oculto ou exibindo apenas o endpoint sem interatividade adicional do usuário `simple`

          Saiba mais nos [guias do playground](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Habilitar esta flag garante que a ordenação de chaves em páginas OpenAPI corresponda à ordenação de chaves definida no arquivo OpenAPI.

      <Warning>Este comportamento em breve será habilitado por padrão, momento em que este campo será descontinuado.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Uma string ou um array de strings de URL(s) ou caminho(s) relativo(s) apontando para
  seu arquivo OpenAPI.
  
  Exemplos:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Um objeto de contas de mídias sociais onde o par chave:propriedade representa a plataforma de mídia social e a URL da conta.
  
  Exemplo: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Um dos seguintes valores `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Exemplo: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    A URL para a plataforma social.
    
    Exemplo: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configurações para habilitar botões de feedback

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Habilita um botão para permitir que os usuários sugiram edições via pull requests
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Habilita um botão para permitir que os usuários registrem um problema sobre a documentação
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personalize o alternador de modo escuro.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Defina se você sempre quer exibir o modo claro ou escuro para novos usuários. Quando não
      definido, o padrão é o mesmo modo do sistema operacional do usuário.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Defina como true para ocultar o alternador de modo claro/escuro. Você pode combinar `isHidden` com `default` para forçar seus docs a usarem apenas modo claro ou escuro. Por exemplo:
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Uma imagem de plano de fundo a ser exibida atrás de cada página. Veja exemplo com
  [Infisical](https://infisical.com/docs) e [FRPC](https://frpc.io).
</ResponseField>
