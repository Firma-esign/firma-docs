O Claude Code é a ferramenta CLI oficial da Anthropic. Este guia irá ajudá-lo a configurar o Claude Code para o auxiliar a escrever e manter a sua documentação.

## Pré-requisitos

- Subscrição ativa do Claude (Pro, Max ou acesso à API)

## Configuração

1. Instale o Claude Code globalmente:

  ```bash
  npm install -g @anthropic-ai/claude-code
```

2. Navegue até ao diretório da sua documentação.
3. (Opcional) Adicione o ficheiro `CLAUDE.md` abaixo ao seu projeto.
4. Execute `claude` para iniciar.

## Criar `CLAUDE.md`

Crie um ficheiro `CLAUDE.md` na raiz do seu repositório de documentação para treinar o Claude Code sobre as suas normas específicas de documentação:

````markdown
# Documentação do Mintlify

## Relação de trabalho
- Pode contestar ideias — isto pode levar a melhor documentação. Cite fontes e explique o seu raciocínio quando o fizer
- Pergunte SEMPRE para obter esclarecimentos em vez de fazer suposições
- NUNCA minta, adivinhe ou invente informação

## Contexto do projeto
- Formato: ficheiros MDX com frontmatter YAML
- Configuração: docs.json para navegação, tema, definições
- Componentes: componentes do Mintlify

## Estratégia de conteúdo
- Documente apenas o suficiente para o sucesso do utilizador — nem demasiado, nem pouco
- Priorize a precisão e a usabilidade da informação
- Torne o conteúdo perene sempre que possível
- Pesquise informação existente antes de adicionar novo conteúdo. Evite duplicação, salvo se for feito por uma razão estratégica
- Verifique os padrões existentes para manter consistência
- Comece por fazer as menores alterações razoáveis

## Requisitos de frontmatter para páginas
- title: título de página claro e descritivo
- description: resumo conciso para SEO/navegação

## Normas de escrita
- Voz na segunda pessoa ("you")
- Pré-requisitos no início do conteúdo procedimental
- Teste todos os exemplos de código antes de publicar
- Corresponda ao estilo e formatação das páginas existentes
- Inclua casos de uso básicos e avançados
- Etiquetas de linguagem em todos os blocos de código
- Texto alternativo em todas as imagens
- Caminhos relativos para ligações internas

## Fluxo de trabalho Git
- NUNCA use --no-verify ao fazer commit
- Pergunte como lidar com alterações não submetidas antes de começar
- Crie um novo ramo quando não existir um ramo claro para as alterações
- Faça commits frequentes durante o desenvolvimento
- NUNCA salte ou desative pre-commit hooks

## Não fazer
- Omitir frontmatter em qualquer ficheiro MDX
- Utilizar URLs absolutos para ligações internas
- Incluir exemplos de código não testados
- Fazer suposições — peça sempre esclarecimentos
````
