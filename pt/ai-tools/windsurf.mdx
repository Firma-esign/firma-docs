Configure o assistente de IA Cascade do Windsurf para o ajudar a escrever e manter documentação. Este guia mostra como configurar o Windsurf especificamente para o seu fluxo de trabalho de documentação do Mintlify.

## Pré-requisitos

- Editor Windsurf instalado
- Acesso ao seu repositório de documentação

## Regras do espaço de trabalho

Crie regras de espaço de trabalho que forneçam ao Windsurf contexto sobre o seu projeto de documentação e normas.

Crie `.windsurf/rules.md` na raiz do seu projeto:

````markdown
# Regra de escrita técnica do Mintlify

## Contexto do projeto

- Este é um projeto de documentação na plataforma Mintlify
- Utilizamos ficheiros MDX com frontmatter YAML  
- A navegação é configurada em `docs.json`
- Seguimos as boas práticas de escrita técnica

## Normas de escrita

- Utilize a segunda pessoa ("you") para instruções
- Escreva na voz ativa e no tempo presente
- Comece os procedimentos com pré-requisitos
- Inclua resultados esperados para os passos principais
- Utilize cabeçalhos descritivos e ricos em palavras-chave
- Mantenha as frases concisas mas informativas

## Estrutura de página obrigatória

Todas as páginas devem começar com frontmatter:

```yaml
---
title: "Título claro e específico"
description: "Descrição concisa para SEO e navegação"
---
```

## Componentes do Mintlify

### Callouts

- `<Note>` para informação suplementar útil
- `<Warning>` para cuidados importantes e alterações incompatíveis
- `<Tip>` para boas práticas e conselhos de especialista  
- `<Info>` para informação contextual neutra
- `<Check>` para confirmações de sucesso

### Exemplos de código

- Quando apropriado, inclua exemplos completos e executáveis
- Utilize `<CodeGroup>` para exemplos em múltiplas linguagens
- Especifique etiquetas de linguagem em todos os blocos de código
- Inclua dados realistas, não preenchimentos
- Utilize `<RequestExample>` e `<ResponseExample>` para documentação de API

### Procedimentos

- Utilize o componente `<Steps>` para instruções sequenciais
- Inclua passos de verificação com componentes `<Check>` quando relevante
- Divida procedimentos complexos em passos menores

### Organização de conteúdo

- Utilize `<Tabs>` para conteúdo específico de plataforma
- Utilize `<Accordion>` para divulgação progressiva
- Utilize `<Card>` e `<CardGroup>` para destacar conteúdo
- Envolva imagens em componentes `<Frame>` com texto alternativo descritivo

## Requisitos de documentação de API

- Documente todos os parâmetros com `<ParamField>` 
- Mostre a estrutura de resposta com `<ResponseField>`
- Inclua exemplos de sucesso e de erro
- Utilize `<Expandable>` para propriedades de objetos aninhados
- Inclua sempre exemplos de autenticação

## Normas de qualidade

- Teste todos os exemplos de código antes de publicar
- Utilize caminhos relativos para ligações internas
- Inclua texto alternativo em todas as imagens
- Assegure a hierarquia adequada dos cabeçalhos (comece com h2)
- Verifique os padrões existentes para manter consistência
````
