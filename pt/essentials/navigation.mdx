O menu de navegação é a lista de ligações em cada site.

Provavelmente irá atualizar o `docs.json` sempre que adicionar uma nova página. As páginas não aparecem automaticamente.

## Sintaxe de navegação

A nossa sintaxe de navegação é recursiva, o que significa que pode criar grupos de navegação aninhados. Não precisa de incluir `.mdx` nos nomes das páginas.

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": [
            "quickstart",
            {
              "group": "Nested Reference Pages",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Pastas

Basta colocar os seus ficheiros MDX em pastas e atualizar os caminhos em `docs.json`.

Por exemplo, para ter uma página em `https://yoursite.com/your-folder/your-page`, criaria uma pasta chamada `your-folder` contendo um ficheiro MDX chamado `your-page.mdx`.

<Warning>

Não pode utilizar `api` como nome de uma pasta a menos que a aninhe dentro de outra pasta. O Mintlify utiliza Next.js, que reserva a pasta `api` de nível superior para chamadas internas de servidor. Um nome de pasta como `api-reference` seria aceite.

</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## Páginas ocultas

Os ficheiros MDX não incluídos em `docs.json` não aparecerão na barra lateral, mas estão acessíveis através da barra de pesquisa e por ligações diretas.
