Todos os sites do Mintlify precisam de um ficheiro `docs.json` com as definições de configuração principais. Saiba mais sobre as [propriedades](#properties) abaixo.

## Propriedades

<ResponseField name="name" type="string" required>
Nome do seu projeto. Utilizado para o título global.

Exemplo: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Uma matriz de grupos com todas as páginas dentro desse grupo
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    O nome do grupo.

    Exemplo: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    Os caminhos relativos para os ficheiros markdown que servirão como páginas.

    Exemplo: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Caminho para a imagem do logótipo ou objeto com o caminho para as imagens do logótipo nos modos "light" e "dark"
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Caminho para o logótipo no modo claro
    </ResponseField>
    <ResponseField name="dark" type="string">
      Caminho para o logótipo no modo escuro
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Para onde a ligação do logótipo o leva
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Caminho para a imagem do favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de cor hex para o seu tema global
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      A cor primária. Utilizada com maior frequência para conteúdo destacado, cabeçalhos
      de secção e acentos, no modo claro
    </ResponseField>
    <ResponseField name="light" type="string">
      A cor primária para o modo escuro. Utilizada com maior frequência para conteúdo
      destacado, cabeçalhos de secção e acentos, no modo escuro
    </ResponseField>
    <ResponseField name="dark" type="string">
      A cor primária para botões importantes
    </ResponseField>
    <ResponseField name="background" type="object">
      A cor do fundo tanto no modo claro como no escuro
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          O código hex da cor de fundo no modo claro
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          O código hex da cor de fundo no modo escuro
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Matriz de `name`s e `url`s de ligações que pretende incluir na barra superior
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    O nome do botão.

    Exemplo: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    O URL para onde a ligação leva ao clicar no botão. Exemplo: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link mostra um botão. GitHub mostra a informação do repositório no URL fornecido, incluindo o número de estrelas do GitHub.
    </ResponseField>
    <ResponseField name="url" type="string">
    Se `link`: para onde o botão liga.
    
    Se `github`: ligação para o repositório do qual carregar a informação do GitHub.
    </ResponseField>
    <ResponseField name="name" type="string">
    Texto dentro do botão. Só é obrigatório se `type` for `link`.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Matriz de nomes de versões. Utilize apenas se pretender mostrar diferentes versões
  da documentação com uma lista pendente na barra de navegação.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Uma matriz das âncoras, inclui o `icon`, `color` e `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    O ícone [Font Awesome](https://fontawesome.com/search?q=heart) utilizado para destacar a âncora.

    Exemplo: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    O nome do rótulo da âncora.

    Exemplo: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      O início do URL que marca quais as páginas que pertencem à âncora. Geralmente é o nome da pasta onde colocou as suas páginas.
    </ResponseField>
    <ResponseField name="color" type="string">
      A cor hex do fundo do ícone da âncora. Também pode ser um gradiente se passar um objeto com as propriedades `from` e `to`, cada uma sendo uma cor hex.
    </ResponseField>
    <ResponseField name="version" type="string">
      Utilizado se pretender ocultar uma âncora até que a versão correta da documentação esteja selecionada.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Passe `true` se pretender ocultar a âncora até que ligue diretamente alguém à documentação dentro dela.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Um de: "brands", "duotone", "light", "sharp-solid", "solid" ou "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Substitua as configurações por defeito da âncora superior.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      O nome da âncora superior
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Ícone Font Awesome.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Um de: "brands", "duotone", "light", "sharp-solid", "solid" ou "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Uma matriz de separadores de navegação.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      O nome do rótulo do separador.
    </ResponseField>
    <ResponseField name="url" type="string">
      O início do URL que marca quais as páginas que pertencem ao separador. Geralmente,
      é o nome da pasta onde colocou as suas páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuração das definições de API. Saiba mais sobre as páginas de API em [Componentes de API](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      O URL base para todos os endpoints da API. Se `baseUrl` for uma matriz, permite várias
      opções de URL base entre as quais o utilizador pode alternar.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          A estratégia de autenticação utilizada em todos os endpoints da API.
        </ResponseField>
        <ResponseField name="name" type="string">
        O nome do parâmetro de autenticação utilizado no playground da API.

        Se o método for `basic`, o formato deve ser `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        O valor por defeito concebido para ser um prefixo do campo de introdução da autenticação.

        Por exemplo, um `inputPrefix` de `AuthKey` faria com que o campo de autenticação herdasse o resultado por defeito `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configurações para o playground da API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Se o playground é mostrado, oculto ou apenas apresenta o endpoint sem interatividade adicional do utilizador (`simple`).

          Saiba mais em [guias do playground](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Ativar esta flag assegura que a ordem das chaves nas páginas OpenAPI corresponde à ordem definida no ficheiro OpenAPI.

      <Warning>Este comportamento será em breve ativado por defeito, altura em que este campo será descontinuado.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Uma cadeia ou uma matriz de cadeias com o(s) URL(s) ou caminho(s) relativo(s) que apontam para
  o seu ficheiro OpenAPI.
  
  Exemplos:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Um objeto de contas de redes sociais em que o par chave:propriedade representa a plataforma de rede social e o URL da conta.
  
  Exemplo: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Um dos seguintes valores `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Exemplo: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    O URL da plataforma social.
    
    Exemplo: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configurações para ativar botões de feedback

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Ativa um botão que permite aos utilizadores sugerir edições através de pull requests
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Ativa um botão que permite aos utilizadores abrir um problema sobre a documentação
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personalize o alternador de modo escuro.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Defina se pretende mostrar sempre modo claro ou escuro para novos utilizadores. Quando
      não estiver definido, adotamos o mesmo modo do sistema operativo do utilizador.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Defina como true para ocultar o alternador entre modo escuro/claro. Pode combinar `isHidden` com `default` para forçar a sua documentação a utilizar apenas o modo claro ou escuro. Por exemplo:
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Uma imagem de fundo apresentada atrás de cada página. Veja exemplos com
  [Infisical](https://infisical.com/docs) e [FRPC](https://frpc.io).
</ResponseField>
