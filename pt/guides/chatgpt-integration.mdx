O Firma permite-lhe adicionar assinaturas eletrónicas com validade legal a tudo o que é construído sobre a OpenAI. Utilize function calling da Responses API para permitir aos seus agentes de IA enviar signing requests a pedido, ligue os servidores MCP do Firma ao Codex CLI para que a IA gere código de integração preciso, ou adicione o Firma como conector do ChatGPT para gerir signing requests a partir da aplicação de ambiente de trabalho do ChatGPT.

## Pré-requisitos

- Uma [conta Firma](https://app.firma.dev) com uma chave de API
- Uma chave de API OpenAI (para function calling), ou uma conta ChatGPT Plus/Pro/Enterprise (para Codex CLI e conectores)
- Pelo menos um template Firma com campos de assinatura configurados

<Note>
  O Firma utiliza a chave de API em bruto como valor do cabeçalho `Authorization` — não a prefixe com `Bearer`. Isto difere da API OpenAI em si, que utiliza tokens `Bearer`.
</Note>

## Começar

<Tabs>

<Tab title="Function calling">

O function calling permite que um modelo GPT decida quando chamar o seu código. Declara uma função que o modelo pode chamar e, quando o utilizador lhe pedir para enviar um contrato, o modelo devolve uma chamada de ferramenta estruturada que a sua aplicação executa contra a API do Firma.

Este caminho é adequado para agentes de IA, chatbots e qualquer aplicação onde um pedido do utilizador em linguagem natural se deva transformar num signing request.

### Passo 1: guardar as suas chaves de API

Mantenha tanto a sua chave de API OpenAI como a sua chave de API Firma em variáveis de ambiente. Nunca as coloque em código frontend.

```bash
OPENAI_API_KEY=your_openai_key
FIRMA_API_KEY=your_firma_key
```

### Passo 2: declarar uma função e lidar com chamadas de ferramenta

Utilizando a Responses API da OpenAI (recomendada para novos projetos):

```javascript
import OpenAI from "openai";

const FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api";
const openai = new OpenAI();

const tools = [
  {
    type: "function",
    name: "send_signing_request",
    description:
      "Send a document for e-signature via Firma. Uses a pre-configured template and one signer.",
    parameters: {
      type: "object",
      properties: {
        name: {
          type: "string",
          description: "A descriptive name for the signing request.",
        },
        template_id: {
          type: "string",
          description: "The ID of the Firma template to send.",
        },
        signer_email: { type: "string" },
        signer_first_name: { type: "string" },
        signer_last_name: { type: "string" },
      },
      required: [
        "name",
        "template_id",
        "signer_email",
        "signer_first_name",
        "signer_last_name",
      ],
      additionalProperties: false,
    },
    strict: true,
  },
];

async function sendSigningRequest(args) {
  const response = await fetch(
    `${FIRMA_API}/signing-requests/create-and-send`,
    {
      method: "POST",
      headers: {
        Authorization: process.env.FIRMA_API_KEY,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        name: args.name,
        template_id: args.template_id,
        recipients: [
          {
            first_name: args.signer_first_name,
            last_name: args.signer_last_name,
            email: args.signer_email,
            designation: "Signer",
            order: 1,
          },
        ],
      }),
    }
  );

  const data = await response.json();
  if (!response.ok) return JSON.stringify({ error: data });

  return JSON.stringify({
    signing_request_id: data.id,
    status: "sent",
    signing_link: data.first_signer?.signing_link,
  });
}

export async function handleUserMessage(userText) {
  let input = [{ role: "user", content: userText }];

  let response = await openai.responses.create({
    model: "gpt-4.1",
    tools,
    input,
  });

  input.push(...response.output);

  for (const item of response.output) {
    if (item.type !== "function_call") continue;
    if (item.name === "send_signing_request") {
      const args = JSON.parse(item.arguments);
      const result = await sendSigningRequest(args);
      input.push({
        type: "function_call_output",
        call_id: item.call_id,
        output: result,
      });
    }
  }

  response = await openai.responses.create({
    model: "gpt-4.1",
    tools,
    input,
  });

  return { text: response.output_text };
}
```

<Note>
  O endpoint `create-and-send` cria o signing request e envia-o aos destinatários numa única chamada de API. Se precisar que o modelo redija um pedido para revisão antes do envio, utilize `POST /signing-requests` para criar um rascunho e depois `POST /signing-requests/{id}/send` após aprovação.
</Note>

### Passo 3: integrar na sua aplicação

Chame `handleUserMessage` a partir do seu backend sempre que o utilizador envia uma mensagem de chat:

```javascript
app.post("/chat", async (req, res) => {
  const reply = await handleUserMessage(req.body.message);
  res.json(reply);
});
```

Uma mensagem do utilizador como "Envia o acordo de consultoria para alice@acme.com" acionará a chamada de função.

### Alternativa em Python

O mesmo fluxo com o SDK Python:

```python
import os
import json
import requests
from openai import OpenAI

FIRMA_API = "https://api.firma.dev/functions/v1/signing-request-api"
client = OpenAI()

tools = [
    {
        "type": "function",
        "name": "send_signing_request",
        "description": "Send a document for e-signature via Firma.",
        "parameters": {
            "type": "object",
            "properties": {
                "name": {"type": "string", "description": "A descriptive name for the signing request."},
                "template_id": {"type": "string"},
                "signer_email": {"type": "string"},
                "signer_first_name": {"type": "string"},
                "signer_last_name": {"type": "string"},
            },
            "required": [
                "name",
                "template_id",
                "signer_email",
                "signer_first_name",
                "signer_last_name",
            ],
            "additionalProperties": False,
        },
        "strict": True,
    },
]

def execute(args):
    r = requests.post(
        f"{FIRMA_API}/signing-requests/create-and-send",
        headers={
            "Authorization": os.environ["FIRMA_API_KEY"],
            "Content-Type": "application/json",
        },
        json={
            "name": args["name"],
            "template_id": args["template_id"],
            "recipients": [
                {
                    "first_name": args["signer_first_name"],
                    "last_name": args["signer_last_name"],
                    "email": args["signer_email"],
                    "designation": "Signer",
                    "order": 1,
                }
            ],
        },
    )
    return json.dumps(r.json())

input_list = [{"role": "user", "content": "Send the NDA to bob@example.com (Bob Smith)."}]

response = client.responses.create(
    model="gpt-4.1",
    tools=tools,
    input=input_list,
)

input_list += response.output

for item in response.output:
    if item.type == "function_call":
        args = json.loads(item.arguments)
        result = execute(args)
        input_list.append({
            "type": "function_call_output",
            "call_id": item.call_id,
            "output": result,
        })

response = client.responses.create(
    model="gpt-4.1",
    tools=tools,
    input=input_list,
)
print(response.output_text)
```

<Tab title="Codex CLI via MCP">

O Codex CLI da OpenAI é um agente de codificação baseado em terminal que suporta servidores MCP. Ligar os servidores MCP do Firma permite ao agente construir integrações Firma e operar sobre os seus dados do Firma.

### Passo 1: instalar o Codex CLI

```bash
npm i -g @openai/codex
```

Ou via Homebrew:

```bash
brew install codex
```

### Passo 2: adicionar os servidores MCP do Firma

Edite a sua configuração global em `~/.codex/config.toml` ou crie um `.codex/config.toml` delimitado ao projeto:

```toml
[mcp_servers.firma-api]
url = "https://mcp.firma.dev/mcp"

[mcp_servers.firma-docs]
url = "https://docs.firma.dev/mcp"
```

Ou adicione-os via CLI:

```bash
codex mcp add firma-api --url https://mcp.firma.dev/mcp
codex mcp add firma-docs --url https://docs.firma.dev/mcp
```

Na primeira utilização de uma ferramenta da API Firma, o Codex CLI guiá-lo-á pelo início de sessão via OAuth.

<Note>
  Quando utilizar cada servidor: `firma-api` é para fazer coisas (enviar signing requests, gerir templates). `firma-docs` é para construir coisas (gerar código de integração com detalhes de API precisos). A maioria dos programadores quer ambos ligados.
</Note>

### Passo 3: utilizar o Firma a partir do Codex CLI

Uma vez ligado, peça ao Codex CLI para construir integrações ou operar sobre os seus dados:

```text
Utilizando a documentação da API Firma, cria uma rota Express que:
1. Aceite um name, template_id, o e-mail do signatário e o nome do signatário
2. Utilize o endpoint create-and-send para enviar um signing request
3. Devolva o ID do signing request e o signing link do signatário
Guarda a chave de API em process.env.FIRMA_API_KEY.
```

Referir explicitamente "a documentação Firma" indica ao agente para consultar o servidor MCP `firma-docs` antes de escrever código.

<Tab title="Conectores do ChatGPT">

A aplicação de ambiente de trabalho do ChatGPT suporta servidores MCP via conectores. Isto permite-lhe gerir signing requests, templates e workspaces do Firma diretamente a partir da interface de chat do ChatGPT.

### Passo 1: ativar o Developer Mode

Abra as definições do ChatGPT e navegue para **Settings > Connectors > Advanced > Developer Mode**. O Developer Mode está disponível nos planos Plus, Pro, Business, Enterprise e Education.

### Passo 2: adicionar os servidores MCP do Firma

Clique em **Add custom connector** e adicione:

- **Firma API** — URL: `https://mcp.firma.dev/mcp`
- **Firma Docs** — URL: `https://docs.firma.dev/mcp`

Na primeira ligação ao servidor da API Firma, ser-lhe-á pedida a autenticação com a sua conta Firma via OAuth.

### Passo 3: utilizar o Firma a partir do ChatGPT

Com os conectores ativos, pode pedir ao ChatGPT para gerir a sua conta Firma:

- "Lista todos os meus templates Firma"
- "Envia o template NDA para alice@example.com para assinatura"
- "Que signing requests estão pendentes no meu workspace Sales?"
- "Mostra-me o audit trail do signing request X"

## Integração com webhooks

Para acompanhar eventos de assinatura em tempo real, registe um webhook Firma a apontar para um endpoint na sua aplicação:

```javascript
app.post("/api/firma-webhook", async (req, res) => {
  const { type, data } = req.body;

  if (type === "signing_request.completed") {
    const signingRequestId = data.signing_request.id;
    // Atualize a sua BD, acione automatização a jusante, etc.
  }

  res.json({ received: true });
});
```

No painel Firma, em **Settings > Webhooks**, registe o URL do seu endpoint. O Firma envia eventos para todas as principais alterações de estado. Consulte o [guia de webhooks](/guides/webhooks) para a lista completa de eventos e verificação de assinatura.

<Warning>
  Verifique sempre a assinatura do webhook utilizando o segredo de assinatura de webhook do Firma em produção. Consulte o [guia de webhooks](/guides/webhooks) para detalhes de implementação.
</Warning>

## Assinatura incorporada

Para aplicações onde os signatários completam documentos dentro da sua UI em vez de abrirem uma página alojada pelo Firma, a resposta de `create-and-send` inclui `first_signer.id` (o `signing_request_user_id`) e um `first_signer.signing_link` pronto a utilizar. Carregue o URL do signatário num iframe:

```html
<iframe
  src="https://app.firma.dev/signing/{signing_request_user_id}"
  style="width:100%;height:900px;border:0;"
  allow="camera;microphone;clipboard-write"
  title="Document Signing"
></iframe>
```

Consulte o [guia de assinatura incorporada](/guides/embeddable-signing) para configuração completa, incluindo boas práticas de segurança.

## Dicas

- **Utilize function calling para agentes em runtime, MCP para tempo de construção.** O function calling é o que a sua aplicação implementada usa para enviar documentos. Os servidores MCP são o que você e o Codex CLI utilizam para construir essa integração.
- **Passe `template_id` como argumento de ferramenta, não como cadeia livre.** Os templates são a forma mais segura de restringir o que o modelo pode enviar.
- **Valide antes de enviar.** Para documentos de maior risco, utilize `POST /signing-requests` para criar um rascunho, apresente-o ao utilizador e chame `POST /signing-requests/{id}/send` apenas após confirmação.
- **Workspaces para aplicações multi-tenant.** Se está a construir um produto SaaS sobre a API OpenAI, atribua a cada cliente final o seu próprio workspace Firma para que templates e utilização se mantenham isolados.
