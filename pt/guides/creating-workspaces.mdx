- **Acesso particionado:** utilizadores, templates e atividade estão delimitados a um único workspace.
- **Acompanhamento de utilização:** veja os envelopes enviados por workspace para um reporte limpo.
- **Segurança por design:** sem exposição entre workspaces de documentos ou dados.

Isto torna simples dar a cada cliente ou equipa o seu próprio ambiente de assinatura privado, enquanto gere tudo através da mesma API.

## Endpoints essenciais

Endpoints essenciais (consulte a referência da API para todos os detalhes):

- POST `/workspaces` — criar um workspace
- GET `/workspaces/{workspace_id}` — obter detalhes do workspace
- PUT `/workspaces/{workspace_id}` — atualizar workspace
- GET `/workspaces` — listar workspaces (paginação)
- GET `/workspace/{workspace_id}/settings` — obter definições do workspace
- PUT `/workspace/{workspace_id}/settings` — atualizar definições do workspace

<Note>
  Consulte o guia sobre [Limites de taxa](rate-limits).
</Note>


Também pode encontrar páginas de referência dedicadas em `api-reference/v01.15.00/workspaces`:

- `api-reference/v01.15.00/workspaces/create-a-new-workspace`
- `api-reference/v01.15.00/workspaces/list-workspaces`
- `api-reference/v01.15.00/workspaces/update-a-workspace`

Autenticação

Todos os pedidos devem incluir um cabeçalho Authorization utilizando a sua chave de API:

```
Authorization: Bearer YOUR_API_KEY
```

Nota: nunca exponha a sua chave de API principal em navegadores de utilizador final. Para operações por workspace que precisem de ser iniciadas a partir do frontend, chame o seu backend, que detém a chave.

## Criar um workspace

Endpoint: POST `/workspaces`

Corpo obrigatório: `{ "name": "string" }`

Exemplo (curl):

```bash
curl -X POST "https://api.firma.dev/functions/v1/signing-request-api/workspaces" \
  -H "Authorization: Bearer $FIRMA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "name": "Acme Corp Workspace" }'
```

Uma resposta bem-sucedida (201) devolve o recurso `Workspace`, incluindo o `api_key` (live) e o `test_api_key` do workspace. Exemplo:

```json
{
  "id": "workspace_123",
  "name": "Acme Corp Workspace",
  "protected": false,
  "api_key": "firma_live_…",
  "test_api_key": "firma_test_…",
  "created_date": "2025-09-04T12:00:00Z",
  "updated_date": "2025-09-04T12:00:00Z"
}
```

O `api_key` é a chave live do workspace e `test_api_key` é a chave de teste. Consulte [Autenticação](authentication) para saber como funciona o modo de teste.

Exemplo de servidor — Node (fetch):

```js
import fetch from 'node-fetch'

export async function createWorkspace(name) {
  const resp = await fetch('https://api.firma.dev/functions/v1/signing-request-api/workspaces', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.FIRMA_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ name })
  })
  if (!resp.ok) throw new Error(`create workspace failed: ${resp.status}`)
  return resp.json()
}
```

Exemplo de servidor — Python (requests):

```py
import os
import requests

def create_workspace(name):
    url = 'https://api.firma.dev/functions/v1/signing-request-api/workspaces'
    resp = requests.post(url, headers={
        'Authorization': f"Bearer {os.environ['FIRMA_API_KEY']}",
        'Content-Type': 'application/json'
    }, json={ 'name': name })
    resp.raise_for_status()
    return resp.json()
```

## Obter um workspace

Endpoint: GET `/workspaces/{workspace_id}`

Exemplo (curl):

```bash
curl "https://api.firma.dev/functions/v1/signing-request-api/workspaces/workspace_123" \
  -H "Authorization: Bearer $FIRMA_API_KEY"
```

Resposta: o recurso `Workspace` (ver exemplo acima).

## Atualizar um workspace

Endpoint: PUT `/workspaces/{workspace_id}` — substituição total do recurso. Corpo obrigatório idêntico ao `CreateWorkspace` (por exemplo, `{ "name": "New name" }`).

Exemplo (curl):

```bash
curl -X PUT "https://api.firma.dev/functions/v1/signing-request-api/workspaces/workspace_123" \
  -H "Authorization: Bearer $FIRMA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "name": "Acme Corp - East Coast" }'
```

Exemplo de servidor — Node (fetch):

```js
export async function updateWorkspace(workspaceId, name) {
  const resp = await fetch(`https://api.firma.dev/functions/v1/signing-request-api/workspaces/${workspaceId}`, {
    method: 'PUT',
    headers: {
      'Authorization': `Bearer ${process.env.FIRMA_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ name })
  })
  if (!resp.ok) throw new Error(`update workspace failed: ${resp.status}`)
  return resp.json()
}
```

## Listar workspaces

Endpoint: GET `/workspaces`

Suporta parâmetros de query `page` e `page_size`. Exemplo:

```bash
curl "https://api.firma.dev/functions/v1/signing-request-api/workspaces?page=1&page_size=20" \
  -H "Authorization: Bearer $FIRMA_API_KEY"
```

O formato de resposta contém `results` (matriz de `Workspace`) e metadados de `pagination`.

## Boas práticas de RBAC e chaves de API

- Utilize chaves de API delimitadas sempre que possível: mantenha uma chave de API master apenas no servidor e crie chaves de API por workspace para parceiros de integração se a sua plataforma o suportar.
- Nunca devolva chaves de API ao navegador. Para operações iniciadas por utilizadores finais, chame o seu backend, que executa a chamada de API para o Firma.
- Utilize idempotência onde for suportada (para endpoints que aceitem `Idempotency-Key`) para evitar duplicação de criação a partir de retentativas.

## Casos limite e resolução de problemas

- Nomes duplicados: os nomes de workspace são amigáveis para apresentação e podem não ser únicos; utilize o `id` para referências programáticas.
- Workspaces protegidos: a flag `protected` indica workspaces pertencentes ao sistema, que não podem ser eliminados ou modificados por fluxos normais.
- Limites de taxa e erros: inspecione respostas não-2xx à procura de `{ error, code }` e apresente mensagens significativas aos operadores.
- **Limite de taxa excedido (429)**: monitorize o cabeçalho `X-RateLimit-Remaining` e implemente exponential backoff para retentativas.
