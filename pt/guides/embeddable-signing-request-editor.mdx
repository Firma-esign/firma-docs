# Editor de signing request incorporável

Incorpore o editor de signing request do Firma dentro da sua aplicação utilizando autenticação JWT para acesso seguro e limitado no tempo. É ideal para integrações white-label e aplicações multi-tenant.

## Casos de uso

- **Fluxos de assinatura white-label**: gerir signing requests sob a sua marca
- **Aplicações multi-tenant**: acesso seguro por signatário sem expor chaves de API
- **Fluxos de trabalho de documentos incorporados**: gestão contínua de signing request dentro do seu produto
- **Acesso limitado no tempo**: os tokens expiram automaticamente por segurança (expiração de 7 dias)

## Como funciona

1. O seu servidor pede um token JWT à API Firma utilizando a sua chave de API
2. O Firma devolve um token JWT de curta duração com o ID do signing request
3. O seu frontend incorpora o editor com o token JWT
4. O token expira automaticamente após 7 dias

<Note>
**Limite de taxa**: os endpoints JWT suportam **100 pedidos por minuto por chave de API** para aplicações de grande volume.
</Note>

## Autenticação JWT

### Gerar token JWT

Gere um token JWT para um signing request específico utilizando o endpoint `/jwt/generate-signing-request`.

**Endpoint**: `POST /jwt/generate-signing-request`

**Corpo do pedido**:
```json
{
  "companies_workspaces_signing_requests_id": "123e4567-e89b-12d3-a456-426614174000"
}
```

**Resposta (200 OK)**:
```json
{
  "jwt": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "jwt_id": "jwt123-e89b-12d3-a456-426614174000",
  "expires_at": "2024-04-27T10:00:00Z",
  "signing_request_id": "123e4567-e89b-12d3-a456-426614174000",
  "created_at": "2024-04-20T10:00:00Z"
}
```

**Cabeçalhos de limite de taxa**:
```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 99
X-RateLimit-Reset: 1729180800
```

## Guia de implementação

<Warning>
**Segurança**: nunca exponha a sua chave de API em código do lado do cliente. Gere sempre tokens JWT a partir do seu backend seguro.
</Warning>

### Backend: gerar token JWT

Chame a API do Firma a partir do seu backend para gerar um token JWT. O seu endpoint de backend deve aceitar um ID de signing request e devolver o JWT ao seu frontend.

**Exemplo Node.js**:
```js
// Exemplo: função de backend simples para gerar JWT
async function generateSigningRequestToken(signingRequestId) {
  const response = await fetch('https://api.firma.dev/functions/v1/signing-request-api/jwt/generate-signing-request', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.FIRMA_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      companies_workspaces_signing_requests_id: signingRequestId
    })
  })
  
  const data = await response.json()
  return data.jwt
}
```

**Exemplo Python**:
```py
import os
import requests

def generate_signing_request_token(signing_request_id):
    response = requests.post(
        'https://api.firma.dev/functions/v1/signing-request-api/jwt/generate-signing-request',
        headers={
            'Authorization': f'Bearer {os.getenv("FIRMA_API_KEY")}',
            'Content-Type': 'application/json'
        },
        json={
            'companies_workspaces_signing_requests_id': signing_request_id
        }
    )
    data = response.json()
    return data['jwt']
```

### Implementação no frontend — HTML / JavaScript puro

```html
<!-- Carregue a biblioteca do editor de signing request Firma -->
<script src="https://api.firma.dev/functions/v1/embed-proxy/signing-request-editor.js"></script>

<!-- Crie um contentor para o editor -->
<div id="signing-request-editor-container"></div>

<script>
  // Assumindo que já obteve o JWT do seu backend
  const jwt = 'eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...';
  const signingRequestId = '08f72e3b-79a8-4eac-a268-b3f9efaf6573';
  
  // Inicialize o editor de signing request
  const editor = new FirmaSigningRequestEditor({
    container: '#signing-request-editor-container',
    jwt: jwt,
    signingRequestId: signingRequestId,
    showCloseButton: true, // Mostrar botão de fechar no cabeçalho
    onSave: (data) => {
      console.log('Signing request saved:', data);
    },
    onSend: (data) => {
      console.log('Signing request sent:', data);
    },
    onClose: (data) => {
      if (data.hasUnsavedChanges) {
        // Pedir ao utilizador que guarde antes de sair
      }
      console.log('Editor closed, unsaved changes:', data.hasUnsavedChanges);
    },
    onError: (error) => {
      console.error('Editor error:', error);
    },
    onLoad: (signingRequest) => {
      console.log('Editor loaded successfully:', signingRequest);
    }
  });
</script>
```

### Implementação no frontend — React

```jsx
import { useEffect, useRef, useState } from 'react';

function SigningRequestEditorComponent({ signingRequestId, jwt }) {
  const containerRef = useRef<HTMLDivElement>(null);
  const editorRef = useRef<any>(null);
  const [isLoaded, setIsLoaded] = useState(false);

  // Carregar a biblioteca do editor de signing request Firma
  useEffect(() => {
    const script = document.createElement('script');
    script.src = 'https://api.firma.dev/functions/v1/embed-proxy/signing-request-editor.js';
    script.async = true;
    
    script.onload = () => setIsLoaded(true);
    document.body.appendChild(script);
    
    return () => {
      if (document.body.contains(script)) {
        document.body.removeChild(script);
      }
    };
  }, []);

  // Inicializar o editor quando a biblioteca e o JWT estiverem prontos
  useEffect(() => {
    if (!isLoaded || !containerRef.current || !jwt) return;

    editorRef.current = new window.FirmaSigningRequestEditor({
      container: containerRef.current,
      jwt: jwt,
      signingRequestId: signingRequestId,
      showCloseButton: true, // Mostrar botão de fechar no cabeçalho
      onSave: (data) => console.log('Saved:', data),
      onSend: (data) => console.log('Sent:', data),
      onClose: (data) => {
        if (data.hasUnsavedChanges) {
          // Pedir ao utilizador que guarde antes de sair
        }
      },
      onError: (error) => console.error('Error:', error),
      onLoad: (signingRequest) => console.log('Loaded:', signingRequest)
    });

    return () => {
      if (editorRef.current?.destroy) {
        editorRef.current.destroy();
      }
    };
  }, [isLoaded, jwt, signingRequestId]);

  return <div ref={containerRef} className="w-full h-full" />;
}

export default SigningRequestEditorComponent;
```

### Opções de configuração

| Opção | Tipo | Descrição |
|-------|------|-----------|
| `container` | HTMLElement | Elemento DOM onde montar o editor (obrigatório) |
| `jwt` | string | Token de autenticação do seu backend (obrigatório) |
| `signingRequestId` | string | Identificador do signing request (obrigatório) |
| `theme` | `'dark'` \| `'light'` | Tema do editor (por defeito: `'dark'`) |
| `readOnly` | boolean | Ativar modo apenas de leitura (por defeito: `false`) |
| `height` | string | Altura do contentor (por defeito: `'100vh'`) |
| `width` | string | Largura do contentor (por defeito: `'100%'`) |
| `showCloseButton` | boolean | Mostrar um botão de fechar no cabeçalho do editor (por defeito: `false`) |
| `onSave` | function | Callback quando o signing request é guardado |
| `onSend` | function | Callback quando o signing request é enviado |
| `onClose` | function | Callback quando o botão de fechar é clicado ou `triggerClose()` é invocado. Recebe `{ hasUnsavedChanges: boolean }` |
| `onError` | function | Callback para tratamento de erros |
| `onLoad` | function | Callback quando o editor carrega com os dados do signing request |

### Métodos de instância

| Método | Descrição |
|--------|-----------|
| `triggerClose()` | Aciona o fluxo de fecho programaticamente. Verifica alterações não guardadas e invoca `onClose` com `{ hasUnsavedChanges: boolean }` |

Utilize `triggerClose()` quando a sua aplicação anfitriã precisar de fechar o editor a partir da própria UI — por exemplo, um evento de navegação pai ou um botão "voltar":

```js
// Fechar o editor a partir da UI da sua aplicação
document.getElementById('my-back-button').addEventListener('click', () => {
  editor.triggerClose();
});
```

## Eventos postMessage (editor → anfitrião)

O editor de signing request do Firma emitirá eventos postMessage para ações importantes do ciclo de vida. Abaixo está um esquema de eventos mínimo recomendado que pode implementar para reagir a saves e envios do editor.

<Tip>
Utilize estes eventos para acompanhar a atividade do editor sem fazer chamadas de API adicionais, ajudando-o a manter-se dentro dos limites de taxa.
</Tip>

Envelope de evento (payload de window.postMessage):

```json
{
	"type": "editor.event",
	"event": "signing_request.saved", // ou signing_request.sent, signing_request.closed
	"payload": {
		"signing_request_id": "sr_123",
		"updated_at": "2025-09-04T12:34:56Z",
		"status": "draft" // ou "sent"
	}
}
```

### Exemplo de listener cliente (JS simples)

```js
window.addEventListener('message', (ev) => {
	// 1) Validar origem
	if (ev.origin !== 'https://app.firma.dev') return
	// 2) Validar formato
	const data = ev.data || {}
	if (data.type !== 'editor.event') return

	switch (data.event) {
		case 'signing_request.saved':
			console.log('Signing request saved', data.payload)
			// Opcionalmente obter o signing request mais recente via API
			break
		case 'signing_request.sent':
			console.log('Signing request sent', data.payload)
			// Acionar webhook ou notificação
			break
		case 'signing_request.closed':
			console.log('Editor closed')
			break
		default:
			break
	}
})
```

## Gestão do ciclo de vida do token

Os tokens JWT são gerados com um tempo de expiração de 7 dias para fluxos de trabalho de edição típicos. O editor tratará automaticamente da expiração do token.

### Expiração automática

Os tokens JWT expiram automaticamente após 7 dias com base no timestamp `expires_at`. Após a expiração:
- O editor incorporado rejeitará o token
- Terá de ser pedido um novo token para continuar
- Sem necessidade de chamada de API — os tokens expiram passivamente

### Atualização de token (opcional)

Para sessões de edição de longa duração, pode atualizar o token antes da expiração utilizando o método `updateJWT()`:

```js
// Gerar um novo JWT a partir do seu backend
const newJwt = await fetch('/your-backend/generate-token').then(r => r.json());

// Atualizar o editor com o novo token
editor.updateJWT(newJwt.jwt);
```

## Limitação de taxa

Os endpoints JWT suportam **100 pedidos por minuto por chave de API**.

**Cabeçalhos de limite de taxa**:
```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1729180800
```

Se exceder o limite de taxa (resposta 429):
- Coloque tokens em cache e reutilize-os até à expiração (7 dias)
- Implemente exponential backoff para retentativas
- Monitorize o cabeçalho `X-RateLimit-Remaining`
- Gere tokens a pedido, não preventivamente

## Boas práticas de segurança

<Warning>
**Nunca exponha a sua chave de API em código do lado do cliente.** Gere sempre tokens JWT a partir de um endpoint de servidor seguro.
</Warning>

### ✅ O que fazer

- ✅ Gerar tokens a partir do servidor backend
- ✅ Monitorizar limites de taxa (100 pedidos/minuto)
- ✅ Utilizar HTTPS para todos os pedidos de API
- ✅ Utilizar `readOnly: true` para acesso apenas de visualização

### ❌ O que não fazer

- ❌ Não exponha chaves de API em código frontend
- ❌ Não reutilize tokens entre sessões ou signatários
- ❌ Não registe tokens JWT (risco de segurança)
- ❌ Não partilhe tokens entre signing requests diferentes

## Resolução de problemas

### Erro Token expired

**Sintoma**: o editor mostra "Token expired" ou erro de autenticação

**Solução**: 
- Implemente atualização de token antes da expiração (janela de 7 dias)
- Gere um novo token e chame `editor.updateJWT(newToken)`
- Verifique a sincronização do relógio do sistema

### 401 Unauthorized

**Sintoma**: a geração de JWT falha com 401

**Causas possíveis**:
- Chave de API inválida ou em falta
- A chave de API não tem as permissões necessárias
- A chave de API está desativada

**Solução**: verifique a chave de API no painel e verifique as permissões

### 404 Not Found

**Sintoma**: signing request não encontrado ao gerar JWT

**Causas possíveis**:
- O ID do signing request não existe
- O signing request pertence a um workspace diferente
- O signing request foi eliminado

**Solução**: verifique o ID do signing request e o acesso ao workspace

### Limite de taxa excedido

**Sintoma**: 429 Too Many Requests

**Solução**:
- Implemente cache de tokens (a janela de expiração de 7 dias é generosa)
- Aguarde a reposição do limite de taxa (verifique o cabeçalho `X-RateLimit-Reset`)
- Implemente lógica de retentativa com exponential backoff

### O editor não carrega

**Sintoma**: o contentor está vazio ou mostra um spinner de carregamento indefinidamente

**Causas possíveis**:
- Script não carregado (verifique o evento `script.onload`)
- JWT ou ID do signing request inválido
- Problemas de CORS (verifique a consola do navegador)

**Solução**:
- Verifique se o script é carregado de `https://app.firma.dev/embed/signing-request-editor.js`
- Verifique se o JWT é válido e não expirou
- Assegure-se de que o backend devolve os cabeçalhos CORS corretos
