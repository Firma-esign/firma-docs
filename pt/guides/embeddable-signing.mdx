# Assinatura incorporável

Incorpore o fluxo de assinatura diretamente na sua aplicação para que os destinatários possam assinar documentos sem sair do seu produto. Isto cria uma experiência de assinatura contínua e white-label.

## Padrão do URL de assinatura

A interface de assinatura está disponível em:

`https://app.firma.dev/signing/{signing_request_user_id}`

## Incorporação básica em iframe

```html
<iframe 
  src="https://app.firma.dev/signing/{signing_request_user_id}" 
  style="width:100%;height:900px;border:0;" 
  allow="camera;microphone;clipboard-write"
  title="Document Signing"
></iframe>
```

### Permissões de iframe necessárias

- `camera` — para verificação de identidade (se ativada)
- `microphone` — para verificação por vídeo (se ativada)
- `clipboard-write` — para copiar/colar conteúdo

## Obter o URL de assinatura

O `signing_request_user_id` é devolvido quando obtém os utilizadores do signing request através da API.

### Exemplo: obter URLs de assinatura de destinatários

```js
const response = await fetch(
  `https://api.firma.dev/functions/v1/signing-request-api/signing-requests/${signingRequestId}/users`,
  {
    headers: {
      'Authorization': `Bearer ${API_KEY}`
    }
  }
)

const users = await response.json()

// Cada utilizador tem o seu próprio URL de assinatura
users.forEach(user => {
  const signingUrl = `https://app.firma.dev/signing/${user.id}`
  console.log(`${user.email}: ${signingUrl}`)
})
```

## Exemplo completo de implementação

### Componente React

```jsx
import { useState, useEffect } from 'react'

function SigningView({ signingRequestId, recipientEmail }) {
  const [signingUrl, setSigningUrl] = useState(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)
  
  useEffect(() => {
    async function loadSigningUrl() {
      try {
        // Obter os utilizadores do signing request a partir do seu backend
        const response = await fetch(
          `/api/signing-requests/${signingRequestId}/users`,
          { credentials: 'include' }
        )
        
        if (!response.ok) {
          throw new Error('Failed to load signing users')
        }
        
        const users = await response.json()
        
        // Encontrar utilizador por e-mail
        const user = users.find(
          u => u.email === recipientEmail
        )
        
        if (!user) {
          throw new Error('User not found')
        }
        
        const url = `https://app.firma.dev/signing/${user.id}`
        setSigningUrl(url)
        setLoading(false)
      } catch (err) {
        setError(err.message)
        setLoading(false)
      }
    }
    
    loadSigningUrl()
  }, [signingRequestId, recipientEmail])
  
  if (loading) return <div>Loading signing interface...</div>
  if (error) return <div>Error: {error}</div>
  
  return (
    <iframe
      src={signingUrl}
      style={{
        width: '100%',
        height: '900px',
        border: 0
      }}
      allow="camera;microphone;clipboard-write"
      title="Sign Document"
    />
  )
}
```

### Endpoint do lado do servidor (Node.js)

```js
import express from 'express'
import fetch from 'node-fetch'

const app = express()
const API_KEY = process.env.FIRMA_API_KEY

app.get('/api/signing-requests/:id/users', async (req, res) => {
  try {
    const response = await fetch(
      `https://api.firma.dev/functions/v1/signing-request-api/signing-requests/${req.params.id}/users`,
      {
        headers: {
          'Authorization': `Bearer ${API_KEY}`
        }
      }
    )
    
    if (!response.ok) {
      const error = await response.text()
      return res.status(response.status).json({ error })
    }
    
    const users = await response.json()
    res.json(users)
  } catch (error) {
    console.error('Failed to fetch signing users:', error)
    res.status(500).json({ error: 'Internal server error' })
  }
})

app.listen(3000)
```

## Eventos postMessage

O iframe de assinatura emite eventos postMessage para acompanhar o progresso da assinatura:

```js
window.addEventListener('message', (event) => {
  // Validar origem
  if (event.origin !== 'https://app.firma.dev') return
  
  const data = event.data
  
  switch (data.type) {
    case 'signing.started':
      console.log('User started signing')
      break
    case 'signing.completed':
      console.log('Document signed successfully')
      // Redirecionar ou mostrar mensagem de sucesso
      break
    case 'signing.declined':
      console.log('User declined to sign')
      break
    case 'signing.error':
      console.error('Signing error:', data.error)
      break
  }
})
```

## Boas práticas de segurança

<Warning>
  **Nunca exponha chaves de API em código frontend.** Obtenha sempre os IDs de utilizador de assinatura através de um endpoint de backend seguro.
</Warning>

### ✅ O que fazer

- ✅ Obtenha IDs de utilizador de assinatura através do seu backend
- ✅ Valide origens de postMessage (`https://app.firma.dev`)
- ✅ Utilize HTTPS para todos os pedidos de API
- ✅ Monitorize eventos de assinatura via webhooks (60 req/min)

### ❌ O que não fazer

- ❌ Não exponha chaves de API em código cliente
- ❌ Não confie em dados de postMessage sem validação de origem

## Integração com webhooks

Utilize webhooks para acompanhar eventos de assinatura em tempo real em vez de fazer polling:

```js
// Handler de webhook
app.post('/webhooks/firma', async (req, res) => {
  const event = req.body
  
  switch (event.event_type) {
    case 'signing_request.viewed':
      await notifyUser(event.data.signing_request_id, 'viewed')
      break
    case 'signing_request.recipient.signed':
      await notifyUser(event.data.signing_request_id, 'signed')
      break
    case 'signing_request.completed':
      await markComplete(event.data.signing_request_id)
      break
  }
  
  res.json({ received: true })
})
```

## Resolução de problemas

### O iframe não carrega

**Causas possíveis**:

- `signing_request_user_id` inválido
- O destinatário já concluiu a assinatura
- O signing request foi cancelado ou expirou

**Solução**: verifique o estado do signing request via API

### Eventos postMessage não recebidos

**Causas possíveis**:

- Validação de origem a bloquear mensagens
- Listener de eventos não anexado antes do iframe carregar

**Solução**:

- Verifique se a origem é exatamente `https://app.firma.dev`
- Anexe o listener antes de criar o iframe

### 403 Forbidden ao obter signing request

**Causas possíveis**:

- O utilizador não tem acesso ao signing request
- A chave de API não tem as permissões necessárias

**Solução**: verifique a lógica de autorização e as permissões da chave de API
