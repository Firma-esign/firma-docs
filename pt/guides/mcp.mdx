O Firma fornece dois servidores MCP que permitem aos assistentes de IA trabalhar com o Firma de formas diferentes:

- **Data MCP** (`https://mcp.firma.dev/mcp`) — dá aos assistentes de IA acesso direto à API do Firma. Gira templates, cria signing requests, configura workspaces e mais através de conversas em linguagem natural. Requer autenticação OAuth.
- **Docs MCP** (`https://docs.firma.dev/mcp`) — dá aos assistentes de IA acesso à documentação completa e à referência da API do Firma. Utilize-o para gerar código de integração preciso com endpoints, formatos de pedido e padrões de autenticação corretos. Não requer autenticação.

A maioria dos programadores quer ambos ligados: o Data MCP para operar sobre dados do Firma, e o Docs MCP para construir integrações Firma.

## O que é o MCP?

O [Model Context Protocol](https://modelcontextprotocol.io) (MCP) é uma norma aberta que liga assistentes de IA a ferramentas e fontes de dados externas. Em vez de escrever chamadas de API manualmente, descreve o que precisa em linguagem simples, e o assistente de IA utiliza os servidores MCP do Firma para executar as chamadas de API corretas ou consultar a documentação correta em seu nome.

## Configuração

<Tabs>

<Tab title="Data MCP">

O servidor Data MCP autentica-se via OAuth 2.1 com PKCE utilizando as credenciais da sua conta Firma. Quando se liga através de qualquer cliente suportado, o fluxo OAuth é tratado automaticamente.

**Clientes suportados:**

- **Claude** — Claude Code, Claude Desktop, Claude.ai
- **ChatGPT** — via conectores em Developer Mode
- **Cursor** — suporte MCP nativo
- **GitHub Copilot** — VS Code 1.101+
- **OpenAI Codex** — via CLI ou config
- **Outros clientes MCP** — qualquer cliente que implemente o [Streamable HTTP transport](https://modelcontextprotocol.io/specification/2025-03-26/basic/transports#streamable-http)

<Tabs>

<Tab title="Claude Code">

Adicione o servidor Firma Data MCP via CLI:

```bash
claude mcp add --transport http firma-api https://mcp.firma.dev/mcp
```

Ou adicione-o à sua configuração do Claude Code (`.claude/settings.json`):

```json
{
  "mcpServers": {
    "firma-api": {
      "type": "url",
      "url": "https://mcp.firma.dev/mcp"
    }
  }
}
```

Ser-lhe-á pedido para iniciar sessão na sua conta Firma na primeira utilização.

### O que o Docs MCP fornece

O servidor Docs MCP dá aos assistentes de IA acesso a:

- **Referência da API** — URLs de endpoint, formatos de pedido/resposta, detalhes de autenticação
- **Guias de integração** — instruções passo a passo para cada plataforma suportada
- **Guias de funcionalidades** — webhooks, assinatura incorporada, white labeling, templates de e-mail e mais
- **Exemplos de código** — exemplos funcionais em Node.js, Python e outras linguagens

### Quando utilizar

Referencie "a documentação Firma" nos seus prompts para indicar ao assistente que consulte este servidor:

```text
Utilizando a documentação Firma, cria uma rota Express que envie um signing request
utilizando o endpoint create-and-send. Utiliza process.env.FIRMA_API_KEY para auth.
```

Sem esta referência, o seu assistente pode recorrer a conhecimento geral e falhar detalhes de endpoint ou autenticação.

## Seleção de workspace

Depois de iniciar sessão no Data MCP, começa sem um workspace selecionado. Pode:

- **Pedir ao assistente que selecione um workspace** — por exemplo, "Muda para o workspace Marketing"
- **Deixar o assistente passar um ID de workspace por pedido** — o assistente pode incluir um parâmetro `workspace_id` em chamadas de ferramenta individuais

O assistente normalmente listará os seus workspaces primeiro e perguntará qual utilizar.

## Conversas de exemplo

### Listar todos os signing requests

> "Mostra-me todos os signing requests pendentes no meu workspace Marketing"

O assistente selecionará o workspace, chamará o endpoint de lista de signing requests e apresentará os resultados.

### Criar e enviar um signing request

> "Cria um signing request a partir do template NDA e envia-o para jane@example.com"

O assistente encontrará o template, duplicá-lo-á para um signing request, definirá o destinatário e enviá-lo-á.

### Verificar estado de assinatura

> "Qual é o estado do signing request que enviei para o João na semana passada?"

O assistente pesquisará signing requests recentes e devolverá o estado atual, incluindo que destinatários assinaram.

### Gerar código de integração

> "Utilizando a documentação Firma, cria uma rota de API Next.js que envie um signing request utilizando o endpoint create-and-send"

O assistente consultará o Docs MCP para o endpoint correto, formato de pedido e padrão de autenticação, e gerará código funcional.

## Limites de taxa

Os pedidos MCP estão sujeitos aos mesmos [limites de taxa](rate-limits) que as chamadas diretas de API. Cada chamada de ferramenta corresponde a um pedido de API. O servidor Docs MCP não tem limites de taxa.

## Resolução de problemas

### Ferramentas não carregam

Se o seu cliente mostra o servidor como ligado mas não aparecem ferramentas:

- **Claude.ai / ChatGPT**: tente desligar e voltar a ligar o conector nas Definições
- **Cursor / VS Code**: reinicie a aplicação depois de adicionar a configuração
- **Claude Desktop**: reinicie a aplicação — alterações em `claude_desktop_config.json` requerem reinício

### Workspace não encontrado

Certifique-se de que selecionou um workspace após ligar ao Data MCP. Utilize "Lista os meus workspaces" para ver as opções disponíveis.

### Erros de autenticação

Tente terminar sessão e voltar a iniciar sessão para atualizar a sua sessão. Se o problema persistir, desligue e volte a ligar o servidor nas definições do seu cliente. O Docs MCP não requer autenticação.
