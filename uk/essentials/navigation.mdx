Меню навігації — це список посилань на кожному вебсайті.

Ви, ймовірно, оновлюватимете `docs.json` кожного разу, коли додаєте нову сторінку. Сторінки не з’являються автоматично.

## Синтаксис навігації

Наш синтаксис навігації рекурсивний, що означає, що ви можете створювати вкладені групи навігації. Не потрібно включати `.mdx` в імена сторінок.

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": [
            "quickstart",
            {
              "group": "Nested Reference Pages",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Папки

Просто розмістіть ваші MDX-файли в папках і оновіть шляхи в `docs.json`.

Наприклад, щоб мати сторінку за адресою `https://yoursite.com/your-folder/your-page`, ви маєте створити папку під назвою `your-folder`, що містить MDX-файл під назвою `your-page.mdx`.

<Warning>

Ви не можете використовувати `api` як назву папки, якщо тільки не вкладите її в іншу папку. Mintlify використовує Next.js, який резервує папку верхнього рівня `api` для внутрішніх серверних викликів. Ім’я папки на кшталт `api-reference` буде прийнятим.

</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## Приховані сторінки

MDX-файли, які не включені в `docs.json`, не з’являться в бічній панелі, але доступні через рядок пошуку та за прямими посиланнями на них.
