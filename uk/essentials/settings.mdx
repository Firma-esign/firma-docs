---

Кожен сайт Mintlify потребує файлу `docs.json` з основними налаштуваннями конфігурації. Дізнайтеся більше про [властивості](#properties) нижче.

## Властивості

<ResponseField name="name" type="string" required>
Назва вашого проєкту. Використовується для глобального заголовка.

Приклад: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Масив груп з усіма сторінками всередині цієї групи
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    Назва групи.

    Приклад: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    Відносні шляхи до markdown-файлів, які будуть слугувати сторінками.

    Приклад: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Шлях до зображення логотипу або об’єкт зі шляхом до зображень логотипу для «світлого» та «темного» режимів
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Шлях до логотипу у світлому режимі
    </ResponseField>
    <ResponseField name="dark" type="string">
      Шлях до логотипу у темному режимі
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Куди веде клік на логотипі
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Шлях до зображення favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Hex-коди кольорів для вашої глобальної теми
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      Основний колір. Найчастіше використовується для виділеного вмісту, заголовків
      розділів, акцентів у світлому режимі
    </ResponseField>
    <ResponseField name="light" type="string">
      Основний колір для темного режиму. Найчастіше використовується для виділеного
      вмісту, заголовків розділів, акцентів у темному режимі
    </ResponseField>
    <ResponseField name="dark" type="string">
      Основний колір для важливих кнопок
    </ResponseField>
    <ResponseField name="background" type="object">
      Колір фону як у світлому, так і в темному режимі
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          Hex-код кольору фону у світлому режимі
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          Hex-код кольору фону у темному режимі
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Масив `name` і `url` посилань, які ви хочете включити в верхню панель
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    Назва кнопки.

    Приклад: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    URL-адреса, за якою переходить кнопка при натисканні. Приклад: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link показує кнопку. GitHub показує інформацію про репозиторій за наданою URL-адресою, включаючи кількість зірочок GitHub.
    </ResponseField>
    <ResponseField name="url" type="string">
    Якщо `link`: На що посилається кнопка.
    
    Якщо `github`: Посилання на репозиторій для завантаження інформації GitHub.
    </ResponseField>
    <ResponseField name="name" type="string">
    Текст усередині кнопки. Обов’язково, лише якщо `type` — `link`.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Масив назв версій. Використовуйте це лише якщо ви хочете показати різні версії
  документації через випадаючий список у панелі навігації.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Масив якорів, включає `icon`, `color` і `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    Іконка [Font Awesome](https://fontawesome.com/search?q=heart), яка використовується для якоря.

    Приклад: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    Назва мітки якоря.

    Приклад: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      Початок URL-адреси, що визначає, які сторінки потрапляють до якоря. Зазвичай це назва папки, в яку ви розмістили ваші сторінки.
    </ResponseField>
    <ResponseField name="color" type="string">
      Hex-колір фону іконки якоря. Може також бути градієнтом, якщо ви передасте об’єкт із властивостями `from` та `to`, кожна з яких є hex-кольором.
    </ResponseField>
    <ResponseField name="version" type="string">
      Використовується, якщо ви хочете приховати якір, доки не буде вибрана відповідна версія документації.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Передайте `true`, якщо ви хочете приховати якір, доки ви не направите когось безпосередньо на документацію всередині нього.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Одне з: "brands", "duotone", "light", "sharp-solid", "solid" або "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Перевизначає стандартні налаштування для найвищого якоря.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      Назва найвищого якоря
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Іконка Font Awesome.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Одне з: "brands", "duotone", "light", "sharp-solid", "solid" або "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Масив навігаційних вкладок.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      Назва мітки вкладки.
    </ResponseField>
    <ResponseField name="url" type="string">
      Початок URL-адреси, що визначає, які сторінки потрапляють до вкладки. Зазвичай це
      назва папки, в яку ви розмістили ваші сторінки.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Конфігурація для налаштувань API. Дізнайтеся більше про сторінки API у [Компонентах API](/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      Базова URL-адреса для всіх кінцевих точок API. Якщо `baseUrl` є масивом, це увімкне кілька варіантів
      базових URL-адрес, між якими користувач може перемикатися.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          Стратегія автентифікації, яка використовується для всіх кінцевих точок API.
        </ResponseField>
        <ResponseField name="name" type="string">
        Назва параметра автентифікації, який використовується в пісочниці API.

        Якщо метод `basic`, формат має бути `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        Значення за замовчуванням, яке призначено бути префіксом для поля введення автентифікації.

        Наприклад, `inputPrefix` зі значенням `AuthKey` успадкує результат введення за замовчуванням поля автентифікації як `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Конфігурації для пісочниці API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Чи показується пісочниця, чи прихована, чи лише відображає кінцеву точку без додаткової інтерактивності користувача (`simple`)

          Дізнайтеся більше в [посібниках з пісочниці](/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Увімкнення цього прапорця забезпечує, щоб порядок ключів на сторінках OpenAPI відповідав порядку ключів, визначеному у файлі OpenAPI.

      <Warning>Ця поведінка незабаром буде увімкнена за замовчуванням, після чого це поле буде застаріле.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Рядок або масив рядків з URL-адресами або відносними шляхами, що вказують на ваш
  файл OpenAPI.
  
  Приклади:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Об’єкт облікових записів соціальних мереж, де пара ключ:властивість представляє платформу соціальних мереж та URL-адресу облікового запису.
  
  Приклад: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Одне з наступних значень: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Приклад: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    URL-адреса соціальної платформи.
    
    Приклад: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Конфігурації для увімкнення кнопок зворотного зв’язку

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Вмикає кнопку, яка дозволяє користувачам пропонувати правки через pull request
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Вмикає кнопку, яка дозволяє користувачам повідомляти про проблему з документацією
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Налаштуйте перемикач темного режиму.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Встановіть, якщо ви завжди хочете показувати світлий або темний режим для нових користувачів. Якщо не
      встановлено, ми за замовчуванням використовуємо той самий режим, що й операційна система користувача.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Встановіть true, щоб приховати перемикач темного/світлого режиму. Ви можете поєднати `isHidden` з `default`, щоб змусити вашу документацію використовувати лише світлий або темний режим. Наприклад:
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Фонове зображення, яке відображатиметься за кожною сторінкою. Дивіться приклад з
  [Infisical](https://infisical.com/docs) та [FRPC](https://frpc.io).
</ResponseField>
