Кожен запит на підпис у Firma має повний аудиторський слід. Кожен перегляд документа, взаємодія з полем, підпис, відмова та дія адміністратора записуються з часовими мітками, ідентифікацією актора та IP-адресами.

Поширені випадки використання:

- **Відповідність вимогам та юридичні докази** — доведіть, хто, що, коли і звідки підписав
- **Налагодження проблем підписантів** — побачте точно, що робив одержувач під час сесії підпису
- **Внутрішня звітність** — відстежуйте дії адміністраторів, такі як створення, редагування та скасування

Аудиторський слід фіксує дві категорії подій:

- **Події підписанта** — перегляди документа, взаємодії з полями, підписання, відмова, завантаження сертифіката
- **Події адміністратора** — створення, редагування, надсилання, скасування, повторне надсилання (через панель керування або API)

Кожна подія містить часову мітку, читабельний опис, ідентифікацію актора та (для подій підписанта) IP-адресу підписанта.

## Перегляд аудиторського сліду в панелі керування

Відкрийте будь-який запит на підпис у панелі керування Firma та натисніть вкладку **Audit Trail**. Ви побачите хронологічну шкалу всіх подій, де дії підписанта та адміністратора візуально розрізняються. Послідовні повторювані події (як прокрутка чи повторні взаємодії з полями) автоматично згортаються з індикатором кількості.

## Отримання аудиторського сліду через API

Використайте ендпоінт audit, щоб отримати повну історію подій запиту на підпис:

```
GET /signing-requests/{id}/audit
```

### curl

```bash
curl -X GET "https://api.firma.dev/functions/v1/signing-request-api/signing-requests/{signing_request_id}/audit" \
  -H "Authorization: YOUR_API_KEY"
```

### Node.js

```js
const response = await fetch(
  `https://api.firma.dev/functions/v1/signing-request-api/signing-requests/${signingRequestId}/audit`,
  {
    headers: {
      'Authorization': process.env.FIRMA_API_KEY,
    },
  }
);

const { results } = await response.json();
console.log(results);
```

### Python

```python
import os
import requests

response = requests.get(
    f"https://api.firma.dev/functions/v1/signing-request-api/signing-requests/{signing_request_id}/audit",
    headers={
        "Authorization": os.environ["FIRMA_API_KEY"],
    },
)

results = response.json()["results"]
```

### Поля відповіді

| Поле | Тип | Опис |
|-------|------|-------------|
| `id` | `uuid` | ID події |
| `timestamp` | `datetime` | Коли відбулася подія |
| `source` | `string` | `"signer"` або `"admin"` |
| `event` | `string` | Ідентифікатор типу події (див. [довідник подій](#event-reference) нижче) |
| `description` | `string` | Читабельний опис події |
| `actor` | `object \| null` | Хто виконав дію (див. нижче) |
| `ip_address` | `string \| null` | IP-адреса підписанта (лише події підписанта) |
| `details` | `object \| null` | Метадані, специфічні для події |
| `condensed_count` | `integer` | Присутній, коли послідовні однакові події від того самого актора згруповано |

## Розуміння джерел подій

Поле `source` вказує, чи подія від підписанта, чи від адміністратора, а поле `actor` має різну структуру для кожного.

### Події підписанта

Події з `source: "signer"` — це дії, виконані одержувачами під час процесу підпису. Об’єкт `actor` містить `name` та `email` підписанта. Поле `ip_address` заповнене.

```json
{
  "source": "signer",
  "actor": { "name": "Alice Johnson", "email": "alice@example.com" },
  "ip_address": "203.0.113.42"
}
```

### Події адміністратора

Події з `source: "admin"` — це дії, виконані через панель керування або API. Об’єкт `actor` містить поле `type` (`"api_key"` або `"user"`) і, опційно, `user_id`. Поле `ip_address` дорівнює `null`.

```json
{
  "source": "admin",
  "actor": { "type": "api_key" },
  "ip_address": null
}
```

```json
{
  "source": "admin",
  "actor": { "type": "user", "user_id": "usr_abc123" },
  "ip_address": null
}
```

## Відстеження в реальному часі через webhook-и

Якщо вам потрібне повідомлення про події підписанта в реальному часі, а не опитування ендпоінта audit, Firma підтримує webhook-и. Дивіться [посібник з Webhooks](webhooks) для інструкцій з налаштування.

## Довідник подій

### Життєвий цикл документа

| Подія | Опис | Має `details` |
|-------|-------------|---------------|
| `document_opened` | Відкрив документ | Ні |
| `document_viewed` | Переглянув документ | Ні |
| `document_saved` | Зберіг значення полів | Ні |
| `document_finished` | Завершив підписання | Ні |
| `document_signed` | Підписав документ | Ні |
| `document_hidden` | Залишив сторінку | Ні |
| `document_visible` | Повернувся на сторінку | Ні |
| `document_closed` | Закрив документ | Ні |

### Підпис і верифікація

| Подія | Опис | Має `details` |
|-------|-------------|---------------|
| `signature_finalized` | Підпис фіналізовано | Ні |
| `signing_declined` | Відмовився підписувати | Ні |
| `terms_accepted` | Прийняв умови та положення | Ні |
| `otp_verified` | Верифікував особу через OTP | Ні |
| `certificate_download` | Завантажив сертифікат | Ні |

### Взаємодії з полями

Взаємодії з полями відстежуються як події `field_interaction`. Об’єкт `details` містить конкретну дію та метадані поля.

| Дія | Опис |
|--------|-------------|
| `field_focused` | Обрано поле |
| `field_completed` | Поле заповнено вперше |
| `field_modified` | Змінено існуюче значення поля |
| `field_cleared` | Очищено значення поля |
| `field_blur` | Завершено редагування поля |

Об’єкт `details` для взаємодій з полями включає:

| Поле | Тип | Опис |
|-------|------|-------------|
| `field_type` | `string` | Тип поля (signature, text, date тощо) |
| `action` | `string` | Конкретна дія з таблиці вище |
| `interaction_count` | `integer` | Кількість разів взаємодії з цим полем |
| `value_length` | `integer` | Довжина значення поля (за застосовністю) |
| `time_spent_ms` | `integer` | Час, витрачений на поле в мілісекундах (за застосовністю) |

### Навігація

| Подія | Опис | Має `details` |
|-------|-------------|---------------|
| `page_viewed` | Переглянуто конкретну сторінку | Так — `page_number` |
| `document_scrolled` | Прокручено до сторінки | Так — `page_number` |
| `navigation_action` | Навігація документом | Так — `action` (`next_page`, `prev_page`, `go_to_page`) |
| `zoom_changed` | Змінено рівень масштабу | Ні |
| `modal_interaction` | Відкрив або закрив діалог | Так — `modal_type`, `action` |

### Активність адміністратора

Події адміністратора використовують `event: "activity"` з читабельним полем `description`, що описує виконану дію (наприклад, "Created signing request via API", "Sent signing request", "Cancelled signing request").
